# 部署管理规范

## 1. scripts/ 目录分类体系

所有脚本按**使用场景**分到四个子目录，禁止直接放根目录。

```
scripts/
├── dev/    # 本地开发辅助 —— 适配器启动、WSL 初始化
├── ci/     # CI/CD 质量门禁 —— GitHub Actions 调用或本地模拟 CI
├── ops/    # 生产运维 —— K8s 部署、发布、监控、回滚（高危操作集中于此）
└── tools/  # 杂项工具 —— E2E 测试、知识库导入、功能联调
```

| 目录 | 文件 | 用途 |

## 2. 脚本头部规范

所有脚本文件（`.sh`）**必须**在 shebang 之后包含标准化头部注释：

```bash
#!/usr/bin/env bash
# =============================================================================
# 脚本名称: <文件名>
# 职责: <简短描述，一句话说清楚这个脚本做什么>
# 使用场景: <手动/CI/定时任务 - 说明什么时候/谁会调用它>
# 影响范围: <说明脚本操作的系统/环境范围>
# 高危级别: 🔴 高危 / 🟡 中危 / 🟢 低危
#   - 🔴 高危: 影响生产，不可快速回滚，需要显式确认
#   - 🟡 中危: 影响测试/暂存，操作可逆
#   - 🟢 低危: 只读、本地、可完全重放
# =============================================================================
set -euo pipefail
```

**高危脚本（🔴）额外要求**：
- 脚本开头打印三行红色警告（使用 ANSI 转义码）
- 明确输出目标环境（主机名/Namespace）
- 提供 5 秒倒计时供用户取消（`sleep 5`）
- Makefile target 额外打印三行警告 + `sleep 5`

**示例（高危脚本开头保护段）**：
```bash
echo -e "\033[0;31m============================================================\033[0m"
echo -e "\033[0;31m 🔴  高危操作 - 即将修改生产环境！\033[0m"
echo -e "\033[0;31m============================================================\033[0m"
echo "目标环境: $(hostname) / Namespace: hci-troubleshoot"
echo "5 秒后执行，Ctrl+C 取消..."
sleep 5
```

## 3. deploy/ 配置分层说明

```
deploy/
├── docker/
│   └── docker-compose.yml          # ← 本地开发唯一入口（含健康检查）
├── env/
│   └── platform.env                # ← 非敏感配置基准（提交 git）
├── helm/
│   └── hci-platform/
│       ├── values.yaml             # ← 生产非敏感配置默认值（提交 git）
│       └── values-*.yaml           # ← 各环境覆盖（staging/prod/local）
├── claw-configs/                   # ← AI 适配器静态配置（提交 git）
├── gitops/                         # ← ArgoCD Application 定义（指向环境仓库）
└── observability/                  # ← 可观测性栈（Loki/Tempo/Grafana/Promtail）
```

### 3.1 配置来源优先级

| 运行环境 | 配置来源（优先级从高到低） |
|---------|--------------------------|
| 本地开发 | `deploy/docker/docker-compose.yml` → `.env` 文件 |
| K3s 测试 | `helm/values.yaml` → `values-staging.yaml` → `--set` 参数 |
| K3s 生产 | `helm/values.yaml` → `values-prod.yaml` → ArgoCD 注入 |

**单一来源原则**：
- 开发环境：`deploy/docker/docker-compose.yml` 是唯一来源，禁止在根目录保留副本
- 生产环境：`helm/values.yaml` 是唯一来源，`platform.env` 仅作参考文档，不直接 source 使用

### 3.2 platform.env 与 Helm values.yaml 变量映射

`deploy/env/platform.env` 中的 `HCI_` 前缀变量与 Helm `values.yaml` 的 `config.*` 字段一一对应：

| platform.env 变量 | values.yaml 字段 | 说明 |
|-------------------|-----------------|------|
| `HCI_POSTGRES_DB` | `config.postgresDb` | 数据库名 |
| `HCI_POSTGRES_USER` | `config.postgresUser` | 数据库用户名 |
| `HCI_REDIS_URL` | `config.redisUrl` | Redis 连接串 |
| `HCI_LOG_LEVEL` | `config.logLevel` | 日志级别 |
| `HCI_OTEL_ENDPOINT` | `config.otelEndpoint` | OTel Collector 地址 |
| `HCI_K8S_NAMESPACE` | `config.k8sNamespace` | K8s 主业务命名空间 |
| `HCI_K8S_IMAGE_PULL_SECRET` | `global.imagePullSecretName` | GHCR 镜像拉取 Secret 名称 |
| `HCI_WARM_POOL_SIZE` | `config.warmPoolSize` | 热备池初始大小 |
| `HCI_MAX_POOL_SIZE` | `config.maxPoolSize` | 热备池最大大小 |
| `HCI_POD_IDLE_TIMEOUT` | `config.podIdleTimeout` | Pod 空闲超时（秒） |

> **platform.env 的定位**：面向人类阅读的配置基准参考，用于文档化非敏感配置的合理默认值。
> 生产部署时，通过 Helm values 文件或 `--set` 参数传入，而非 `source platform.env`。

## 4. 密钥管理三条规则

1. **`.example` 文件进 git，真实文件 gitignore**  
   敏感配置文件（`.env`、密钥文件）必须在 `.gitignore` 中排除，对应 `.example` 模板提交 git。

2. **CI/CD 通过 GitHub Secrets 注入**  
   GitHub Actions 使用 `${{ secrets.XXX }}`，ArgoCD 使用 K8s Secret（由 SealedSecrets 或手动创建）。

3. **代码中禁止硬编码任何凭证**  
   任何 API Key / Token / Password 不得出现在源码文件中。`values.yaml` 中的 `dev_password_123` 类占位符仅供本地开发，生产部署时**必须**通过 `--set secrets.postgresPassword=xxx` 覆盖。

## 5. 高危脚本三重保护

| 层次 | 机制 | 说明 |
|------|------|------|
| **脚本层** | 头部警告 + 5 秒倒计时 | 脚本内置，直接调用时生效 |
| **Makefile 层** | target 打印三行红色警告 + `sleep 5` | `make k3s-deploy-prod` 时触发 |
| **头部注释层** | `🔴 高危` 标识 + 影响范围说明 | 代码审查时可见，阻止轻率修改 |

涉及脚本：`ops/k3s-deploy-prod.sh`、`ops/k3s-release.sh`、`ops/rollback-drill.sh`

## 6. 变更规则

| 操作 | 规则 |
|------|------|
| 新增脚本 | 必须放入正确子目录，必须有标准化头部 |
| 修改脚本路径 | 同步更新：Makefile + `.github/workflows/` + AGENTS.md |
| 新增配置变量 | 同步更新：`platform.env`（非敏感） + `values.yaml`（Helm 默认值）+ 本文档映射表 |
| 删除脚本 | 检查所有引用（`grep -r "脚本名"` 全项目搜索） |
