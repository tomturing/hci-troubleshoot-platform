# Admin 管理台 API 路由修复部署

## 部署范围

本次变更涉及以下服务，需重新构建镜像并重启：

| 服务 | 端口 | 变更内容 |
|------|------|---------|
| api-gateway | 8000 | 新增 categories_router + kbd_router |
| kb-service | 8004 | 新增 KBD 列表 + 拒绝端点 |
| admin-ui | 80 | 修复分类解析、删除知识审核、补充 KBD 鉴权头 |

## 部署步骤

### 1. 构建后端镜像

```bash
# api-gateway
cd backend/api-gateway
docker build -t api-gateway:latest .

# kb-service
cd backend/kb-service
docker build -t kb-service:latest .
```

### 2. 构建前端镜像

```bash
cd frontend/admin
pnpm install && pnpm build
docker build -t admin-ui:latest .
```

### 3. 滚动更新

```bash
# 按依赖顺序重启：先后端再前端
kubectl rollout restart deployment kb-service -n hci
kubectl rollout status deployment kb-service -n hci --timeout=120s

kubectl rollout restart deployment api-gateway -n hci
kubectl rollout status deployment api-gateway -n hci --timeout=120s

kubectl rollout restart deployment admin-ui -n hci
kubectl rollout status deployment admin-ui -n hci --timeout=120s
```

### 4. 使用 Helm 部署（推荐）

```bash
cd deploy/helm
helm upgrade hci-platform ./hci-platform -n hci -f ../../hci-platform-env/environments/dev/values.yaml
```

## 数据库变更

无。本次修复不涉及数据库 schema 变更。

## 回滚方案

回滚至上一版本镜像即可，无数据库迁移需要撤销：

```bash
kubectl rollout undo deployment api-gateway -n hci
kubectl rollout undo deployment kb-service -n hci
kubectl rollout undo deployment admin-ui -n hci
```

## 注意事项

- api-gateway 新增了两个独立 router（categories_router、kbd_router），与原 kb.router 不冲突
- KBD 列表使用内部 Token 鉴权（`INTERNAL_API_TOKEN`），确保 kb-service 环境变量已正确配置
