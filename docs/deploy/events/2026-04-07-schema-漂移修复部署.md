# Schema 漂移修复 — 部署操作手册

## 前置条件

- 已阅读 [方案文档](../../solution/events/2026-04-07-schema-漂移修复方案.md)
- 拥有目标 namespace 的 kubectl exec 权限
- 项目根目录有最新代码（含 `database/migrations/20260407001_schema_repair.sql`）

---

## Phase 1: Dev 环境（已完成 ✅）

```bash
# 直接在 postgres-0 执行修复 SQL
kubectl cp database/migrations/20260407001_schema_repair.sql hci-dev/postgres-0:/tmp/
kubectl exec postgres-0 -n hci-dev -- psql -U hci_admin -d hci_troubleshoot \
  -f /tmp/schema_repair.sql

# 注册版本
kubectl exec postgres-0 -n hci-dev -- psql -U hci_admin -d hci_troubleshoot \
  -c "INSERT INTO schema_migrations (version) VALUES ('20260407001') ON CONFLICT DO NOTHING;"
```

**结果**：23 张表，case-service HTTP 201 ✅

## Phase 2: Staging 环境

### 步骤 1：诊断

```bash
kubectl exec postgres-0 -n hci-staging -- psql -U hci_admin -d hci_troubleshoot -c "\dt+"
kubectl exec postgres-0 -n hci-staging -- psql -U hci_admin -d hci_troubleshoot \
  -c "SELECT version FROM schema_migrations ORDER BY version;"
```

### 步骤 2：使用一键脚本修复

```bash
./scripts/ops/db-repair-env.sh hci-staging
```

脚本自动完成：复制 SQL → 执行修复 → 注册版本 → 验证表数量。

### 步骤 3：启用 dbMigrate

修改 `hci-platform-env/environments/staging/values.yaml`：

```yaml
dbMigrate:
  enabled: true
```

通过 PR 提交到环境仓库并审批合并。

### 步骤 4：验证

```bash
# ArgoCD sync 后检查 db-migrate Job（应为 no-op）
kubectl logs job/db-migrate-<revision> -n hci-staging

# 服务验证
kubectl exec -n hci-staging deploy/case-service -- python3 -c "
import urllib.request, json
req = urllib.request.Request('http://localhost:8001/api/cases/',
    data=json.dumps({'title':'staging迁移验证','description':'test','assistant_type':'openclaw','client_id':'staging-verify'}).encode(),
    headers={'Content-Type':'application/json'}, method='POST')
resp = urllib.request.urlopen(req)
print(f'HTTP {resp.status}: {json.loads(resp.read())[\"case_id\"]}')
"
```

## Phase 3: Prod 环境

### 额外前提

1. **staging 观察窗口 ≥ 72 小时**无迁移相关问题
2. **维护窗口已通知**相关团队
3. **人工审批**：修改 `dbMigrate.enabled: true` 必须通过 PR + 审批

### 步骤 1：备份

```bash
kubectl exec postgres-0 -n hci-prod -- \
  pg_dump -U hci_admin -d hci_troubleshoot > /tmp/prod_backup_$(date +%Y%m%d).sql
```

### 步骤 2：执行修复

```bash
./scripts/ops/db-repair-env.sh hci-prod
```

### 步骤 3：启用 + 验证

同 staging 步骤 3、4，但需等待 PR 审批。

### 回滚计划

如出问题：
1. 恢复 `dbMigrate.enabled: false`
2. 从备份恢复 DB：`psql -U hci_admin -d hci_troubleshoot < /tmp/prod_backup_YYYYMMDD.sql`
