# SOP 导入操作指南（2026-04-11）

## 背景

本文档记录将 SOP `.docx` 文件导入知识库（kb-service）的完整流程，含 CLI 方式和 Admin UI 方式。

---

## 前置条件

### K3s 环境端口转发

在本地终端执行（需保持窗口开启）：

```bash
# 转发 API Gateway（SOP 导入接口在 API Gateway）
kubectl -n hci port-forward svc/api-gateway 8080:80 &

# 如需访问 Admin UI
kubectl -n hci port-forward svc/admin-ui 3000:80 &
```

### 获取内部 API Token

```bash
kubectl -n hci get secret hci-secrets -o jsonpath='{.data.INTERNAL_API_TOKEN}' | base64 -d
```

默认 dev 环境 token：`hci-dev-internal-token`

---

## 方式一：CLI 批量导入（推荐）

适用于批量处理多个 `.docx` 文件。

### 安装依赖

```bash
cd /mnt/d/aihci/hci-troubleshoot-platform
uv sync
```

### 执行导入

```bash
uv run PYTHONPATH=data-pipeline python -m kbd.import_sop \
  --file /path/to/your.docx \
  --category-id "分类编码" \
  --token hci-dev-internal-token \
  --api-url http://localhost:8080
```

**参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `--file` | ✓ | `.docx` 文件路径 |
| `--category-id` | ✗ | 分类编码，如 `虚拟机-001`（可后续在 Admin UI 修改） |
| `--token` | ✓ | 内部 API Token |
| `--api-url` | ✗ | 默认 `http://localhost:8080` |

### 输出示例

```
✓ 文档解析完成: my_sop.docx
✓ 导入成功: document_id=5, chunks_created=42, status=draft
```

## 方式二：Admin UI 上传

1. 打开 Admin UI：`http://localhost:3000`
2. 进入 **知识库管理 → SOP 文档**
3. 点击 **导入 .docx**
4. 选择文件并指定分类
5. 点击确认，等待解析完成

## 导入后操作

### 发布 SOP（使 AI 可搜索）

导入成功后文档状态为 `draft`，需要**发布**才能被 AI 检索到。

```bash
# 通过 API 发布（替换 {document_id}）
curl -X POST http://localhost:8080/api/admin/sop/{document_id}/approve \
  -H "Authorization: Bearer hci-dev-internal-token"
```

或在 Admin UI 中点击 **发布** 按钮。

### 验证向量已生成

```bash
# 查看 chunks_embedded 是否 > 0
curl http://localhost:8080/api/admin/sop/{document_id} \
  -H "Authorization: Bearer hci-dev-internal-token" | jq '.chunks_embedded'
```

## 幂等规则（重要）

### 同文件重复导入

- 若文件内容未变（SHA256 哈希相同），重复导入**不会**新增或覆盖文档
- 若同时传入了不同的 `--category-id`，系统会**更新** category_id（文档内容不变）
- 返回的仍是已有 `document_id`

### 更新文件内容

- 修改文件内容后，哈希变化，系统视为**新文档**，会创建新记录
- 旧记录不会自动删除，需要在 Admin UI 手动归档

## 常见问题

### Q: 导入报 "服务器未安装 python-docx"

**原因**：kb-service 镜像缺少 `python-docx` 依赖（已在 requirements.txt 中修复，需重新构建镜像）。

**临时方案**：先将 `.docx` 手动转换为 `.md`，再用 CLI 直接传 Markdown 内容。

---

### Q: 发布后 chunks_embedded=0（生成了 0 个向量）

**根因（三层级联）**：

1. `ZAI_API_KEY` 未注入容器 → z.ai API 返回 401
2. `EMBEDDING_DIM=512` 与 DB `Vector(1536)` 不一致 → numpy fallback 向量维度被拒绝
3. `sentence_transformers` 未安装 → BGE 本地模型不可用

**已修复**：
- `ZAI_API_KEY` 已通过 `hci-secrets` 自动注入
- `EMBEDDING_DIM` 已从 512 改为 1536

**注意**：当前 `ZAI_BASE_URL=http://openclaw:18789` 是 chat gateway，不直接支持 embedding 模型名 `embedding-3`。短期降级方案为 numpy hash fallback（1536 维哈希向量），语义搜索精度有限，建议后续配置独立的 z.ai embedding 端点。

---

### Q: 导入时 category_id 没有生效

查看是否是重复导入（相同哈希），查看日志：

```bash
kubectl -n hci logs deployment/kb-service | grep sop_ingest
```

若出现 `event=sop_ingest_category_updated`，说明已更新分类；
若出现 `event=sop_ingest_duplicate`，说明 category_id 相同，未变更。
