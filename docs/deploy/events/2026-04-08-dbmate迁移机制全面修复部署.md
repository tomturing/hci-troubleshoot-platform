# dbmate 迁移机制全面修复部署指南

## 部署概述

本次变更属于基础设施改造，无应用逻辑变更，部署风险低。变更合并到 main 后，ArgoCD 自动触发 PreSync Hook 执行 dbmate，新增迁移 `20260408001` 会被执行。

## 前置条件

- [ ] PR 已合并到 main
- [ ] ArgoCD 可正常连接目标环境
- [ ] 目标环境 PostgreSQL 可访问

## 自动化部署流程（正常路径）

```
PR 合并 main
    ↓
ArgoCD 检测到 Git 变更，触发 Sync
    ↓
PreSync Hook：db-migrations-configmap（weight=-10）
    — 更新包含 20260408001 的 ConfigMap
    ↓
PreSync Hook：db-migrate-job（weight=-5）
    — dbmate up 执行 20260408001_sop_tables_fix_version.sql
    — 创建 sop_document / sop_chunk 表（IF NOT EXISTS，已存在则跳过）
    ↓
应用 Pod 正常滚动更新
```

## 手动验证步骤（部署后）

```bash
# 1. 确认迁移已执行
kubectl exec -n hci -it <postgres-pod> -- psql -U hci_admin -d hci_troubleshoot \
    -c "SELECT version FROM schema_migrations WHERE version >= '20260408' ORDER BY version;"
# 期望输出：20260408001

# 2. 确认 sop_document / sop_chunk 表已创建
kubectl exec -n hci -it <postgres-pod> -- psql -U hci_admin -d hci_troubleshoot \
    -c "\dt sop_*"
# 期望输出：sop_document, sop_chunk

# 3. 确认废弃表不存在
kubectl exec -n hci -it <postgres-pod> -- psql -U hci_admin -d hci_troubleshoot \
    -c "SELECT tablename FROM pg_tables WHERE schemaname='public' \
        AND tablename IN ('kb_document','kb_chunk','kb_sop_node','kb_synonym',\
                          'raw_cases','knowledge_atoms','prompt_audit');"
# 期望输出：0 rows

# 4. 确认总表数为 17
kubectl exec -n hci -it <postgres-pod> -- psql -U hci_admin -d hci_troubleshoot \
    -c "SELECT COUNT(*) FROM pg_tables WHERE schemaname='public' AND tablename != 'schema_migrations' AND tablename != 'alembic_version';"
# 期望输出：17

# 5. 确认 Alembic Job 未运行（hci-platform-data）
kubectl get job -n hci | grep db-migrate
# 期望：只有 db-migrate-<revision>（dbmate），无 db-migrate-alembic-*
```

## 存量环境说明

如果目标环境的 `schema_migrations` 中已有 `20260408001` 记录（不太可能，但防御），dbmate 会跳过该迁移，无副作用。

如果 `sop_document` 表已通过手动 SQL 创建，`CREATE TABLE IF NOT EXISTS` 会跳过，无副作用。

## 回滚方案

本次变更无破坏性操作（只新增表，不删除数据）。若需回滚：

1. 在 ArgoCD 中将 main 回退到前一个版本
2. 新建迁移文件执行 `DROP TABLE IF EXISTS sop_chunk; DROP TABLE IF EXISTS sop_document;`（不推荐，仅极端情况）
3. 从 `schema_migrations` 删除 `20260408001` 记录（人工操作）

## Alembic Job 确认

合并后确认 hci-platform-data 的 ArgoCD Application 中 Alembic Job 显示为 `Suspended/Disabled`（`enabled: false`），不会触发任何 Alembic 迁移。
