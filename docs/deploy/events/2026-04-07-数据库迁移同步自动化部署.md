# 数据库迁移同步自动化部署

## 部署概述

部署数据库迁移同步自动化机制，包括同步脚本和 CI 检查。

## 部署清单

| 组件 | 文件 | 说明 |
|------|------|------|
| 同步脚本 | `scripts/ci/sync-db-migrations.sh` | 将迁移同步到 ConfigMap |
| 检查脚本 | `scripts/ci/check-db-migrations-sync.sh` | 验证同步状态 |
| CI 配置 | `.github/workflows/ci.yml` | 添加 check-db-migrations-sync job |

## 部署步骤

### 1. 合并 PR

```bash
gh pr merge 114 --squash --delete-branch
```

### 2. 验证 CI

确认 CI 中 `check-db-migrations-sync` job 运行成功。

### 3. 验证 ArgoCD 同步

```bash
# 检查 ConfigMap 已更新
kubectl get configmap db-migrations -n hci-dev -o jsonpath='{.metadata.resourceVersion}'

# 检查迁移 Job 状态
kubectl get jobs -n hci-dev -l app.kubernetes.io/component=db-migrate
```

## 回滚方案

如需回滚：

```bash
# 删除新增脚本
git revert <commit-sha>

# 手动移除 CI job
# 编辑 .github/workflows/ci.yml 删除 check-db-migrations-sync job
```

## 后续操作

添加新迁移时：

```bash
# 1. 创建迁移文件
vim database/migrations/YYYYMMDDNN_description.sql

# 2. 运行同步脚本
scripts/ci/sync-db-migrations.sh

# 3. 提交
git add database/migrations/ deploy/helm/hci-platform/templates/hooks/db-migrations-configmap.yaml
git commit -m "feat(db): 添加迁移 XXX"
```
