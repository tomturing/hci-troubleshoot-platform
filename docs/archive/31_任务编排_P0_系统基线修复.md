# Task 01：Prompt 手术——重写对话核心约束（P0）

```
你是一名负责 hci-troubleshoot-platform conversation-service 的 agent。

【仓库】
git clone https://github.com/tomturing/hci-troubleshoot-platform.git
cd hci-troubleshoot-platform

【背景】
当前系统最核心的 AI 排障逻辑在：
  backend/conversation-service/app/services/conversation_service.py

该文件中存在一个致命约束 _SYSTEM_BASE，内容大意为：
  - "只能根据下方参考资料回答，不得使用训练知识"
  - "如果知识库无内容，必须直接拒绝回答"

这导致系统在知识库为空、或未命中 RAG 时，对用户的任何 HCI 故障描述都只能返回
"知识库暂无相关内容"——无法发挥 GLM 的 HCI 领域知识。这是系统最根本的功能缺陷。

HCI 排障的专家工作方式是：
  1. 以客户描述为基准，提取关键实体（虚拟机/集群/存储/网络）
  2. 同时调用告警日志和操作日志工具，拿到系统侧证据
  3. 提出 1-3 个确认问题，定位到最精确的故障分类
  4. 即使没有相关 SOP，也能用机制知识推理，明确标注"基于领域知识推理"

【任务目标】
1. 重写 _SYSTEM_BASE，改为 5 段式专家 Prompt 结构
2. 修改空知识库时的处理逻辑：拒绝 → 机制推理 + 明确标注
3. 在 ConversationSession 模型中预留 diagnostic_stage 字段（为 Phase 2 状态机预留）
4. 确认改动后 GLM 能对 HCI 故障做有意义的回答（不再直接拒绝）

【涉及服务 / 文件范围】
允许修改：
  - backend/conversation-service/app/services/conversation_service.py（核心修改文件）
  - backend/shared/models/（如果 ConversationSession 模型在此定义）
只读参考：
  - docs/architecture/各层最优设计.md § Layer 2（5段式 Prompt 设计）
  - docs/architecture/完整技术方案.md § 三、Phase 0
禁止修改：
  - 任何数据库迁移文件（Phase 2 任务负责）
  - 其他微服务代码

当前诊断阶段：{stage_desc}

标准诊断流程：
S0 意图识别：从客户描述提取关键实体（虚拟机名/集群/时间点），

编写修复脚本（Python 最稳定）：
```python
#!/usr/bin/env python3
# scripts/dev/fix_sop_format.py
"""修复 SOP 文件中的 docx 导入 artifact：命令文本重复、多余换行"""
import re
import sys
from pathlib import Path

def fix_sop_file(path: Path) -> int:
    """返回修复的行数"""
    content = path.read_text(encoding='utf-8')
    original = content

    # 修复1：去除 acli 命令重复（"acli xxx\nacli xxx" → "acli xxx"）
    # 匹配模式：某段 acli 命令后紧跟同一命令
    lines = content.split('\n')
    fixed_lines = []
    prev_line = None
    for line in lines:
        stripped = line.strip()
        # 跳过与前一行完全相同的 acli 命令行（排除代码块注释行）
        if stripped and stripped == (prev_line or '').strip() and stripped.startswith('acli'):
            continue
        fixed_lines.append(line)
        prev_line = line

    content = '\n'.join(fixed_lines)

    # 修复2：去除行内重复（同一行出现两次相同的 acli 命令）
    # 如 "acli task get -v ${vmid}acli task get -v ${vmid}"
    def deduplicate_inline(m):
        half = len(m.group(0)) // 2
        first_half = m.group(0)[:half]
        if m.group(0) == first_half * 2:
            return first_half
        return m.group(0)

    # 匹配 30-200 字符的命令重复
    content = re.sub(r'(acli [^\n]{15,100})\1', r'\1', content)

    if content != original:
        path.write_text(content, encoding='utf-8')
        return content.count('\n') - original.count('\n')  # 减少的行数
    return 0

if __name__ == '__main__':
    sop_dir = Path('data-pipeline/sop_skills')
    files = [sop_dir / 'vm_power_failure.md'] + \
            list((sop_dir / 'vm_power_failure' / 'chapters').glob('*.md'))
    for f in files:
        n = fix_sop_file(f)
        print(f'修复 {f.name}: {n} 处改动')
```
