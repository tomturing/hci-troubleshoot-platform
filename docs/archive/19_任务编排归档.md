在 save_user_message()（或等效的保存用户消息方法）中增加重复提问检测逻辑：

1. 获取当前 case 下最近 N 条（建议 N=10）用户消息
2. 计算新消息与历史消息的 Jaccard 相似度（token 级，中英文分词用 split() 即可，不需要 jieba）
3. 若任意一条历史消息 Jaccard ≥ 0.6，判定为重复提问
4. 重复时：UPDATE conversation SET repeat_question_count = repeat_question_count + 1 WHERE id = {conversation_id}

Jaccard 相似度参考实现：
def jaccard(a: str, b: str) -> float:
    sa, sb = set(a.lower().split()), set(b.lower().split())
    if not sa or not sb:
        return 0.0
    return len(sa & sb) / len(sa | sb)

【约束】
- 检测逻辑应轻量，不阻塞消息保存（可先保存再异步检测，或直接同步但要控制查询范围）

  async def _write_prompt_audit(
      self,
      conversation_id: uuid.UUID,
      case_id: str,
      assistant_type: str,
      trace_id: str,
      message_count: int,
      audit_meta: dict,
      sample_payload: list | None,
  ) -> None:
      """写入 prompt_audit 记录（后台任务，失败不影响主流程）"""
      try:
          async with self.session_factory() as session:
              await ConversationRepository(session).insert_prompt_audit(
                  conversation_id=conversation_id,
                  case_id=case_id,
                  assistant_type=assistant_type,
                  trace_id=trace_id,
                  message_count=message_count,
                  has_sop=audit_meta["has_sop"],
                  kb_chunks_count=audit_meta["kb_chunks_count"],
                  kb_top_score=audit_meta["kb_top_score"],
                  messages=sample_payload,
              )
              await session.commit()
          logger.info(
              event="prompt_audit_written",
              conversation_id=str(conversation_id),
              case_id=case_id,
              has_sop=audit_meta["has_sop"],
              kb_chunks_count=audit_meta["kb_chunks_count"],
              sampled=sample_payload is not None,
          )
      except Exception as e:
          # 审计失败不影响业务流程，只记录 warning
          logger.warning(
              event="prompt_audit_write_error",
              message=str(e),
              conversation_id=str(conversation_id),
          )

              "kb_top_score": kb_top_score,

          },
      )

# Task 37：SSH 代理与终端交互后端能力（API + WS）（P1）

```
