# Task 30-D：重复提问统计

```
你是一名负责 hci-troubleshoot-platform conversation-service 开发的 agent。

【仓库】
git clone https://github.com/tomturing/hci-troubleshoot-platform.git
cd hci-troubleshoot-platform

【前置条件】
Task 30-A（数据库迁移）已执行，conversation 表存在 repeat_question_count INTEGER 列。

【任务目标】
文件：backend/conversation-service/app/services/conversation_service.py

在 save_user_message()（或等效的保存用户消息方法）中增加重复提问检测逻辑：

1. 获取当前 case 下最近 N 条（建议 N=10）用户消息
2. 计算新消息与历史消息的 Jaccard 相似度（token 级，中英文分词用 split() 即可，不需要 jieba）
3. 若任意一条历史消息 Jaccard ≥ 0.6，判定为重复提问
4. 重复时：UPDATE conversation SET repeat_question_count = repeat_question_count + 1 WHERE id = {conversation_id}

Jaccard 相似度参考实现：
def jaccard(a: str, b: str) -> float:
    sa, sb = set(a.lower().split()), set(b.lower().split())
    if not sa or not sb:
        return 0.0
    return len(sa & sb) / len(sa | sb)

【约束】
- 检测逻辑应轻量，不阻塞消息保存（可先保存再异步检测，或直接同步但要控制查询范围）
- 代码注释使用中文
- 不修改与本任务无关的代码

【验收标准】
- 在同一 case 发送两条措辞相近的消息（如 "虚拟机磁盘IO异常" 和 "虚拟机 磁盘IO 异常排查"）后，conversation.repeat_question_count = 1
- 发送内容完全不同的消息，repeat_question_count 不增加
- 单元测试：jaccard("虚拟机 磁盘 异常", "虚拟机 磁盘 IO 异常") ≥ 0.6 → True

完成后提交 PR，等待 Claude 审核。
```
