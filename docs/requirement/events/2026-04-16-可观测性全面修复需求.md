# 可观测性全面修复需求

## 变更历史
| 日期 | 版本 | 变更内容 | 关联事件文档 |
|------|------|---------|------------|
| 2026-04-16 | v1.0 | 初版，基于全面审查发现14项缺陷 | 本文档 |

## 背景与问题

通过对现有可观测性实现的深度代码审查，发现三大支柱（Traces/Logs/Metrics）均存在严重缺陷，
导致多数告警永远不触发、Trace→Log 联动失效、历史数据不持久，实际可观测能力远低于文档描述。

核心痛点：
1. **告警静默**：大量告警规则引用的指标名称与代码实际注册名不一致，永远不触发
2. **链路断裂**：`X-Trace-Id` 响应头使用 `uuid4()` 生成，与 OTel Trace ID 完全无关，用户无法从 F12 反查链路
3. **数据丢失**：Tempo 存储在 `/tmp`，Loki 无持久化卷，容器重启后历史数据全部丢失
4. **联动失效**：Promtail 无 JSON 解析 pipeline，`trace_id` 字段无法提取，Trace→Log 一键跳转看到的是所有日志而非当前请求日志

## 需求描述

对可观测性三大支柱进行全面修复，使其从"文档存在但功能失效"转为"真实可用的生产级可观测性"。

## 功能需求

### P0（核心功能失效）
1. 修复 Tempo 存储路径，挂载持久化卷，保证历史 Trace 数据不丢失
2. 修复 Promtail pipeline，解析 JSON 日志提取 `trace_id` 字段，使 Trace→Log 联动真实有效
3. 修复告警规则中 Pod 池指标名称（`hci_warm_pool_size` → `hci_pod_pool_idle/active`），并修复 scheduler-service 的指标上报
4. 为所有服务添加 HTTP 请求延迟/错误率指标，使 `HighApiLatency`/`HighErrorRate` 告警真实有效

### P1（可用性保障缺失）
5. 修复 `TraceIDMiddleware`，使 `X-Trace-Id` 响应头返回真实 OTel Trace ID
6. 为 kb-service 添加 KB 检索耗时指标上报（`KB_SEARCH_DURATION_SECONDS`）
7. 添加 node-exporter 和 kube-state-metrics scrape 配置（可选，注释形式提供）
8. 为 Loki 添加持久化卷

### P2（配置错误/安全）
9. 修复 datasources.yml 中 `serviceMap.datasourceUid` 从 `loki` 改为 `prometheus`
10. 为 Tempo metrics_generator 添加 `remote_write` 配置，使 span-metrics 落地到 Prometheus
11. 修复 Grafana 安全配置（弱密码、匿名访问改为环境变量注入）

### P3（代码质量）
12. 为 `StructuredLogger` 添加 `exception()` 方法，支持完整 traceback 记录
13. 为 `LoggingInstrumentor` 添加清晰注释，说明与 `StructuredLogger` 的关系
14. 提供 Grafana Dashboard 代码化预置（provisioning/dashboards/）

## 验收标准

- [ ] 重启 Tempo 容器后，历史 Trace 数据仍可查询
- [ ] 重启 Loki 容器后，历史日志仍可查询
- [ ] Grafana Tempo 瀑布图点击 Span → "Logs for this span" 只显示该请求的日志（而非全部日志）
- [ ] F12 看到的 `X-Trace-Id` 响应头值可以直接粘贴到 Tempo 搜索框得到结果
- [ ] Prometheus 告警 `WarmPoolExhausted` 和 `WarmPoolLow` 在 scheduler-service 状态变化时触发
- [ ] Prometheus 告警 `HighApiLatency` 在 P95 延迟超过 2s 时触发
- [ ] Grafana serviceMap（服务拓扑图）页面正常渲染
- [ ] `logger.exception()` 调用能记录完整的 Python traceback

## 约束条件

- 不引入新的基础设施组件（在现有 Tempo/Loki/Prometheus/Grafana 体系上修复）
- 所有 Python 代码变更须通过 Ruff lint 检查
- 不破坏现有业务功能
