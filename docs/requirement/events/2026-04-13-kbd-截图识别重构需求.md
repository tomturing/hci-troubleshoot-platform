# KBD 截图识别重构需求

**日期**：2026-04-13  
**状态**：已完成  
**迭代**：feature/kbd-screenshot-ocr-redesign

---

## 背景

KBD 截图识别链路原有方案依赖 PaddleOCR，在生产/开发环境（WSL2/Python 3.13）下因 OneDNN 框架层 `NotImplementedError` 无法正常运行，导致截图内容无法入库。

---

## 需求目标

1. **重建截图 OCR 链路**：放弃 PaddleOCR，改用 Vision LLM（qwen3.5-plus）作为主路径
2. **结构化分析**：新增分析 LLM（qwen3-max-2026-01-23）从截图中提取 TYPE/KEY/TIPS
3. **v2 desc 格式**：引入 BACKGROUND/TYPE/FULL_TEXT/KEY/TIPS 五字段格式
4. **前端差异化展示**：KbdReviewView 根据截图类型（任务截图/日志截图/其他截图）差异化渲染

---

## 功能需求

### 1. Vision OCR 主路径（scripts/kbd/image_proc.py）

- Step 1：Vision LLM 文字照录（全量提取截图中所有文字，不过滤）
- Step 2：Pillow 背景色采样（白/黑/其他）
- Step 3：分析 LLM（TYPE/KEY/TIPS 结构化提取）
- Step 4：组装 v2 desc.txt 输出

### 2. v2 desc 格式（scripts/kbd/config.py、scripts/kbd/analyzer.py）

```
BACKGROUND: 白色
TYPE: 任务截图
FULL_TEXT:
- 行1
- 行2
KEY:
- 关键条目1
TIPS:
- 分析建议1
```

TYPE 枚举：任务截图 | 日志截图 | 配置截图 | 其他截图

### 3. converter v2 解析（scripts/kbd/converter.py）

`_desc_to_screenshot_block()` 同时兼容 v1（结构化字段）和 v2（原始文本）格式，生成 Markdown 引用块。

### 4. 前端展示（frontend/admin/src/views/KbdReviewView.vue）

- 解析 `> **【截图说明】**` 块内的 BACKGROUND/TYPE/FULL_TEXT/KEY/TIPS
- 根据 TYPE 差异化显示：任务截图显示进度表格，日志截图显示代码块，其他截图显示原始文字
- 折叠长文本（>10行），新增 TIPS 建议区

---

## 非功能需求

- 超时容错：Vision LLM 超时时降级为"其他截图"
- 速率限制：openai client 自动重试
- 性能：单图处理不超过 120s（含重试）
