# 环境同步并发竞争修复需求

## 背景

2026-04-07 发现 hci-troubleshoot-platform 的 PR #102、#103 合入后，ArgoCD 未部署最新代码。排查发现根本原因为 **环境仓库并发推送竞争**：

1. CI workflow 同时触发 dev 和 staging 环境同步
2. 两个 workflow 并发向环境仓库 main 分支推送 commit
3. staging 先推送成功，dev 推送时远程已有新提交，导致 `git push` 被拒绝
4. dev 环境镜像标签未更新，ArgoCD 无法获取新镜像

## 问题现象

- 镜像已构建并推送（`20260407-0352-d999647`）
- 环境仓库 dev values.yaml 仍为旧标签（`20260407-0310-59ae7be`）
- ArgoCD 显示 Synced/Healthy，但实际部署的是旧镜像

## 根因分析

CI workflow 中的 `auto-deploy-non-prod` job 使用 `for` 循环同时触发两个环境同步：

```yaml
for env in dev staging; do
  gh workflow run env-repo-sync.yml ... -f target_env=${env}
done
```

两个 `env-repo-sync.yml` 运行实例并发向同一 main 分支推送，产生竞争条件。

## 需求

1. **dev 环境优先部署**：dev 是开发测试环境，应最快获得最新代码
2. **staging 不阻塞 dev**：staging 的任何问题（审批、网络、权限）不应影响 dev 部署
3. **避免并发竞争**：环境同步改为串行执行，每个环境推送完成后再触发下一个

## 验收标准

- [ ] CI 构建后 dev 环境镜像标签自动更新
- [ ] staging 环境同步失败不影响 dev 环境部署
- [ ] 有明确的同步顺序日志，可追溯每个环境的同步状态
