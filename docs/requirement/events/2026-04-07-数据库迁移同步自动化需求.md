# 数据库迁移同步自动化需求

## 背景与问题

### 问题描述

PR #113 合并了废弃表清理迁移 `20260407002_cleanup_deprecated_tables.sql`，但 ConfigMap 没有同步更新。导致 ArgoCD 部署后迁移不会执行，数据库中废弃表仍然存在。

### 根因分析

1. **ConfigMap 是静态文件**：`deploy/helm/hci-platform/templates/hooks/db-migrations-configmap.yaml` 手动维护
2. **缺少同步机制**：添加新迁移文件后，需要手动将内容复制到 ConfigMap
3. **缺少 CI 检查**：没有自动化验证迁移是否已同步

### 影响

- 数据库迁移不会执行，废弃表残留
- 环境状态与代码不一致
- 需要人工排查和修复

## 需求

### 功能需求

| 需求ID | 描述 | 优先级 |
|--------|------|--------|
| REQ-001 | 提供自动化脚本同步迁移到 ConfigMap | P0 |
| REQ-002 | CI 中添加检查，防止迁移遗漏同步 | P0 |
| REQ-003 | 同步脚本支持增量更新 | P1 |

### 非功能需求

- 同步脚本幂等，可重复执行
- CI 检查快速（< 5秒）
- 错误信息清晰，提示修复方法

## 验收标准

- [ ] 运行 `scripts/ci/sync-db-migrations.sh` 可同步所有迁移
- [ ] CI 中 `check-db-migrations-sync` job 验证同步状态
- [ ] 迁移未同步时 CI 失败，提示修复命令
