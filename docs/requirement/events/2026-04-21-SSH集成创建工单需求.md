## 背景与问题

### 业务背景

当前 PR181 已合入 Environment API 后端实现，但前端缺少 UI 入口，用户无法触发环境数据采集。用户需要：
1. 打开 SSH 终端面板
2. 手动填写 SSH 连接信息
3. 连接成功后，没有自动触发采集

### 核心痛点

1. **流程割裂**：创建工单和 SSH 连接是两个独立操作，用户需要多次点击
2. **采集入口缺失**：SSH 连接成功后没有自动采集环境数据，AI 首轮诊断缺少上下文
3. **无 SSH 场景未支持**：某些情况下用户暂时无法连接 SSH，但仍需创建工单

### 影响范围

- 所有使用客户端对话界面的工程师
- AI 首轮诊断准确率（缺少环境上下文）

## 需求描述

### 功能概述

将 SSH 连接流程集成到创建工单对话框中，实现「一次操作完成工单创建 + SSH 连接 + 环境数据采集」。

### 用户场景

**场景 A：有 SSH 连接**
1. 工程师点击「新建工单」
2. 弹框显示，自动检测 Bridge 是否运行
3. 填写工单基本信息 + 选择 AI 助手 + 填写 SSH 信息（自动填充上次成功的信息）
4. 点击「连接并创建工单」
5. SSH 连接成功 → 检查 acli → 秒级采集环境数据
6. 终端面板自动打开（抽屉式）
7. 展示采集结果（折叠卡片）
8. 点击「完成创建」进入对话界面

**场景 B：SSH 连接失败**
1. 连接失败时显示错误信息
2. 提示检查 SSH 配置
3. 用户可重新填写并重试，或选择「无 SSH 创建」

**场景 C：无 SSH 创建**
1. 点击「无 SSH 创建工单」（灰色按钮，悬浮显示告警）
2. 弹框提示自动化能力降低
3. 点击「继续创建」完成工单创建
4. 进入对话界面后可随时通过终端面板连接 SSH

**场景 D：Bridge 未运行**
1. 弹框弹出时检测到 Bridge 未运行
2. SSH 连接按钮禁用
3. 提示下载并启动 terminal_bridge.exe
4. 「无 SSH 创建」按钮仍可用

## 功能需求

### F1：创建工单对话框改造

1. **基本信息区**：标题、描述、AI 助手选择器
2. **SSH 连接信息区**：主机、端口、用户名、密码（仅支持密码认证）
3. **Bridge 检测**：弹框弹出时自动检测 terminal_bridge.exe 是否运行
4. **按钮区域**：
   - 「连接并创建工单」（蓝色背景）- 上方
   - 「取消」（白色背景）- 中间
   - 「无 SSH 创建工单」（灰色背景，悬浮告警）- 下方

### F2：SSH 信息持久化

- 使用 localStorage 存储上次成功连接的 SSH 信息
- 下次弹框自动填充
- 仅支持密码认证方式

### F3：SSH 连接流程

1. 点击「连接并创建工单」→ 连接 SSH
2. 连接成功 → 检查 acli 工具（执行 `acli`）
3. acli 可用 → 秒级采集（无进度条）：
   - `acli platform info get` → 集群信息
   - `acli --formatter json alert list` → 告警列表
   - `acli --formatter json task list` → 任务状态
4. 采集完成 → 终端面板自动打开（抽屉式，与现有行为一致）
5. 展示采集结果（折叠卡片格式）

### F4：错误处理

1. **Bridge 未运行**：
   - SSH 连接按钮禁用
   - 显示「下载 SSH Bridge」链接
   - 「无 SSH 创建」按钮可用

2. **SSH 连接失败**：
   - 显示错误类型和详情
   - 提示检查项（地址、端口、用户名、密码、SSH 服务）
   - 可重新填写并重试

3. **acli 未安装**：
   - 提示 acli 未安装
   - 环境采集跳过
   - 可选择「跳过，继续创建」

### F5：无 SSH 创建工单

1. 按钮灰色背景，不凸显
2. 悬浮提示：「若无 SSH，自动化能力将大大降低」
3. 点击后弹框确认：
   - 列出失去的能力（无法自动采集、无法执行命令、需手动提供信息）
   - 可返回填写 SSH 或继续创建

### F6：环境数据展示

创建成功后，对话界面顶部显示折叠卡片：
- 集群信息摘要
- 告警数量
- 任务状态摘要
- 点击展开显示完整内容
- 提供「刷新数据」「发送到助手」按钮

## 非功能需求

### 性能要求

- SSH 连接超时：15 秒
- 环境数据采集：秒级完成（< 5 秒）
- Bridge 检测：< 2 秒

### 可用性要求

- SSH 信息自动填充，减少用户输入
- 错误信息清晰，指导用户如何修复
- 无 SSH 流程不推荐但可用，满足临时场景

### 安全要求

- 密码明文存储在 localStorage（仅本地开发环境，生产环境需加密）
- SSH 会话通过 WebSocket 本地转发，不暴露到网络

### 可维护性

- 前端采集逻辑统一封装，便于扩展新采集类型
- Bridge 检测逻辑复用现有 checkBridgeRunning 函数

## 约束条件

### 技术约束

- 仅支持密码认证，不支持密钥认证
- 采集通过 acli 命令执行，需目标主机安装 acli
- Bridge 检测逻辑复用现有实现

### 资源约束

- 需在 1-2 个开发日内完成

### 依赖约束

- 依赖 Environment API（PR181 已合入）
- 依赖 terminal_bridge.exe（已实现）

---

## 风险与假设

### 已知风险

1. localStorage 存储密码有安全风险（生产环境需加密）
2. acli 命令执行失败可能导致采集不完整

### 假设条件

1. 用户目标主机已安装 acli 工具
2. terminal_bridge.exe 已分发到用户
3. 用户网络可访问目标 SSH 端口
