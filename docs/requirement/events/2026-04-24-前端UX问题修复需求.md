## 背景与问题

用户在使用 HCI 故障排查平台创建工单、采集环境数据、以及 AI 问诊全流程中，发现了 5 个影响体验和正确性的问题。

- **业务背景**：SSH 流程创建工单后需采集环境数据，采集完成后进入 AI 对话问诊。
- **核心痛点**：采集命令被静默丢弃、工单状态不实时刷新、AI 不响应、有数据却提示未采集。
- **影响范围**：所有通过 SSH 流程创建工单的用户，以及通过纯文字创建工单的用户。

---

## 需求描述

### Bug 1：采集命令被静默丢弃（显示超时）

- **现象**：SSH 采集流程中，命令发出后一直等待到超时，但实际后台手动执行同一命令 2 秒即返回。
- **根因**：`CaseCreateDialog.vue` 的 `runCommand()` 在 SSH 建立后，发送 `ssh_input` 消息时使用 `case_id: createdCaseId.value || 'temp'`（如 `Q2026042497781`），但 Terminal Bridge 中 SSH session 是以 `'ssh-create-temp'` 为键存储的，Key 不匹配导致命令被静默丢弃。
- **附带问题**：采集失败时会将错误占位符（`{ parse_error: ..., raw: '' }`）写入数据库，干扰后续 AI 推理。
- **超时时间**：原本 10s 对某些复杂集群可能不够，需要调整。

### Bug 2：顶部工单 ID 不实时显示

- **现象**：通过 SSH 流程创建并确认工单后，页面顶部不显示工单 ID badge，刷新页面后才出现。
- **根因**：`CaseCreateDialog.vue` 确认工单后，只更新了本地 `createdCaseId.value`，没有同步写入 `chatStore.currentCase`。`App.vue` 用 `v-if="chatStore.currentCase"` 控制 badge 渲染，因此不显示。

### Bug 3：AI 助手无反应（纯文字创建流程）

- **现象**：工单显示"工单已确认，正在连接 AI 助手..."后，AI 一直没有响应。
- **根因**：`createConversation()` 的异常被 catch 块吞掉，`conversationId.value` 保持 null，后续 `streamAIResponse()` 函数第一行 `if (!conversationId.value) return` 静默退出。

### Bug 4（非 Bug）：工单状态更新时机

- **结论**：非真实 bug。`confirm` 在 AI 开始前同步完成，数据库 status 已是 `confirmed`。意图识别是对话层（conversation）的状态变化，与 case status 无关。

### Bug 5：有数据也显示"未采集环境数据"

- **现象**：工单 Q2026042496150 数据库中已有环境数据，但前端展示"未采集环境数据"。
- **根因**：`environmentContext` store 状态只在 SSH 采集完成时填充，以下路径设置了 `currentCase` 但从未调用 `collectEnvironmentData()`：
  - `resumePendingCase()` — 刷新后恢复工单
  - `switchToCase()` — 切换历史工单
  - `confirmCreateCase()` — 纯文字创建（无 SSH）

---

## 功能需求

1. **修复 Bug 1**：`runCommand()` 发送 `ssh_input` 时使用 `case_id: 'ssh-create-temp'`，与 Bridge session key 对齐。
2. **修复 Bug 1 附带**：采集超时时间 10s → 30s；采集失败时不入库错误占位符。
3. **修复 Bug 2**：SSH 确认工单后同步写入 `chatStore.currentCase`。
4. **修复 Bug 3**：`createConversation()` catch 块向上重新抛出异常。
5. **修复 Bug 5**：在 `resumePendingCase`、`switchToCase`、`confirmCreateCase` 中补调 `collectEnvironmentData`。

---

## 验收标准

- [ ] SSH 采集流程能在 30s 内成功返回集群信息、告警、任务数据
- [ ] 采集失败时数据库中无错误占位符（`parse_error` 字段）
- [ ] SSH 流程创建工单后顶部立即显示工单 ID badge，无需刷新
- [ ] 纯文字创建工单后 AI 助手正常响应
- [ ] 已有环境数据的工单（通过刷新/切换恢复）能正确显示环境摘要卡片
- [ ] 切换工单时，旧工单的环境数据不错误显示在新工单上

---

## 约束条件

- **技术约束**：Terminal Bridge 以 `case_id` 字符串为键存储 SSH session，创建工单前统一使用 `'ssh-create-temp'` 作为临时 key
- **资源约束**：仅修改前端代码（`CaseCreateDialog.vue`、`chat.ts`），不涉及后端变更

---

## 风险与假设

- **假设**：后端 `GET /environments/case/{id}/context` 始终返回对象（即使无数据），不会 404
- **已知风险**：`collectEnvironmentData` 失败时采用 fire-and-forget（`.catch(() => {})`），不影响主流程但会静默失败
