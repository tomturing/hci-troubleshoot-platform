# KBD 截图展示与数据管道重构

## 变更历史
| 日期 | 版本 | 变更内容 | 关联事件文档 |

## 背景与问题
- **业务背景**：KBD 管理页面用于审核从深信服技术支持门户抓取的知识条目，每个条目包含多张截图及其 OCR/Vision 描述
- **核心痛点**：
  1. 截图展开内容字段名与后端设计不匹配（前端显示"可见内容/错误日志/排障建议"，但 v3 格式只有"截图中文字/语义描述")
  2. 图片语义描述（DESCRIPTION）未展示在截图展开框内，直接显示在页面上
  3. "fail_slow_domain_mp" 显示为 "fail\_slow_domain_mp"（Markdown 转义问题）
  4. `_extract_image_urls_with_positions()` 与 `fetcher._extract_image_urls()` 重复实现图片 URL 提取逻辑，后续任一侧规则调整可能导致序号不一致，上下文映射错位
- **影响范围**：KBD 管理页面（admin-ui）、数据管道（data-pipeline/kbd）

## 需求描述
- **功能概述**：
  1. 修复前端截图展开框的字段渲染逻辑，正确展示 v3 格式（DESCRIPTION 字段）
  2. 修复 Markdown 下划线转义问题
  3. 抽取公共图片 URL 提取函数，确保 fetcher 和 image_proc 的序号一致性
- **用户场景**：管理员在 KBD 管理页面点击条目详情，查看截图说明时能看到正确的字段内容和位置
- **预期收益**：提升审核体验，避免上下文错位导致的排障误导

## 功能需求
1. ✅ 前端 DESCRIPTION 字段渲染位置修正（展开框内）
2. ✅ `inlineRender` 函数处理 `\_` → `_` 转义

4. ✅ fetcher.py 保留向后兼容封装
5. ✅ image_proc.py 移除重复实现
6. ✅ 单元测试覆盖公共模块
