# 终端操作接口与交互优化需求

**日期**：2026-04-25  
**类型**：缺陷修复 + 功能增强  
**优先级**：P0（线上 500 错误）

## 背景

上线终端操作录制功能（Task 42）后，发现以下四处问题影响用户体验：

1. **`GET /api/terminal/operations` 返回 500**：api-gateway Helm Deployment 未注入 `DATABASE_URL`，导致 `TerminalService.list_operations` 在 `DatabaseManager` 初始化时因空连接串报错。
2. **集群信息显示"未知"**：环境数据卡片展开后集群版本/名称/节点数均显示"未知"占位，无法区分"已采集但值为空"与"未采集"两种状态，用户体验混乱。
3. **SSH 登录成功后只恢复 IP，端口/用户名未恢复**：`SshFormSection.vue` 中用 `if (!localForm.value.port)` 判断是否填充，由于初始值为 `'22'`（truthy），永远不会覆盖 saved port，用户重开窗口需要重填。
4. **S0/S6 候选选项纯文字展示，不支持点击**：AI 消息中的 `①②③` 候选故障/操作选项需要人工手打序号，不够直观，希望渲染为可点击按钮。

## 需求列表

| # | 需求 | 优先级 |
|---|------|--------|
| R1 | api-gateway Helm Deployment 补充 `DATABASE_URL` 环境变量注入 | P0 |
| R2 | 环境数据卡片：隐藏值为"未知"的字段，全为"未知"时显示"待采集"提示 | P1 |
| R3 | `SshFormSection` 加载 localStorage 时始终覆盖（不限空值检查） | P1 |
| R4 | `MessageBubble` 自动识别包含圆圈数字选项的助手消息，渲染可点击按钮 | P1 |
