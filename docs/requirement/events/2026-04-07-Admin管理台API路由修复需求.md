# Admin 管理台 API 路由修复需求

## 背景

Admin 管理台三个页面（分类基线、知识审核、KBD 审核）均报错"加载失败，请刷新重试"，数据库中有数据但前端无法获取。

## 问题清单

| 页面 | 前端请求路径 | 根因 |
|------|------------|------|
| 分类基线 | `GET /api/kb/categories` | api-gateway KB 路由前缀为 `/api/v1/kb`，没有 `/api/kb/categories` 代理；响应格式与前端解析不匹配 |
| 知识审核 | `GET /api/v1/atoms/pending` | `knowledge_atoms` 表已废弃并删除，后端无对应 API，前端组件应删除 |
| KBD 审核 | `GET /api/v1/kbd/pending` | kb-service 有 `kbd_router`（前缀 `/api/admin/kbd`）但缺少列表端点；api-gateway 无代理 |

## 需求

1. **分类基线**：api-gateway 新增分类管理代理路由，修复前端响应解析
2. **知识审核**：删除废弃的 `KnowledgeReviewView.vue` 组件和路由配置
3. **KBD 审核**：
   - kb-service 新增列表查询端点（分页 + 状态/分类过滤）和拒绝端点
   - api-gateway 新增 KBD 代理路由
   - 前端补充鉴权头

## 优先级

P1 — 管理台核心功能不可用
