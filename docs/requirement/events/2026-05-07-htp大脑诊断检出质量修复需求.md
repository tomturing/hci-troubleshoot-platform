# htp 大脑诊断检出质量修复需求

## 背景与问题

基于 2026-04-28 的代码精读分析（见 `docs/solution/agent/大脑可选-集成重设计方案.md`），
结合 ops-agent feature-hci 分支的参考实现，发现 htp 大脑存在以下影响排障检出效果的缺陷。

## 问题优先级分级

### P0：排障功能直接失效（本次必修）

| 编号 | 问题 | 影响 |
|------|------|------|
| N-1 | Jaccard 相似度按空格分词，中文完全失效 | 重复提问检测对中文用户 100% 无效 |
| N-3 | BrainRouter 路径 env_context 硬编码 None | ops-agent 大脑永远看不到实时告警数据 |
| D-3 | ReactExecutor 调用 `_build_system_prompt` 未传 `diagnostic_stage` | 永远以 S0 意图识别 Prompt 执行工具调用 |

### P1：知识路由错误，降低检出率（本次必修）

| 编号 | 问题 | 影响 |
|------|------|------|
| N-2 | S1+ 阶段 KB 检索忽略 S0 确认的分类，用当前用户消息重新意图识别 | 可能漂移到错误知识轨道，且每轮额外 LLM 分类调用 |
| N-4 | ops-agent 路径下 htp 状态机仍对 ops-agent 输出做后验正则检测 | 可能误触发 htp S0-S6 阶段推进 |

### P2：架构改进，提升检出质量（本次完成核心项）

| 编号 | 改进 | 价值 |
|------|------|------|
| D-2 | S3 阶段 acli 命令自动提取执行（ReactExecutor 局部激活） | 消除 S3 阶段用户手动执行的瓶颈，提升排障速度 |

### P3：稳定性修复（本次一并修复）

| 编号 | 问题 | 影响 |
|------|------|------|
| N-5 | `_increment_s0_candidate_rounds` 非原子 read-modify-write | 并发下计数器可能少计，兜底轮次判断失准 |
| N-6 | OpsAgentBrainAdapter httpx 客户端从不关闭 | 长期运行后 TCP 连接池资源泄漏 |

## 核心需求

### 功能需求

1. **中文分词修复**：Jaccard 相似度改用 bigram 字符级分词，支持中文重复提问检测
2. **env_context 传递修复**：BrainRouter 路径将预取的 `context_info` 正确传入 `env_context`
3. **ReactExecutor 阶段同步**：`send_message_react_stream` 使用当前 `diagnostic_stage` 构建 Prompt
4. **KB 检索分类锁定**：S1+ 阶段优先使用 S0 确认的 `category_id`，跳过重复意图识别
5. **ops-agent 状态机隔离**：ops-agent 路径跳过 htp 状态机后验检测
6. **S3 自动工具执行**：从 LLM 回复中提取 acli 命令，自动执行并返回结果（需用户确认高危命令）
7. **原子计数器**：`s0_candidate_rounds` 改用 PostgreSQL 原生 JSON 更新避免竞态
8. **httpx 客户端生命周期**：`OpsAgentBrainAdapter` 在 FastAPI lifespan 中注册关闭

### 非功能需求

- 所有 P0/P1 修复向后兼容，不改变 API 接口
- P2 的 S3 自动执行：只读命令（`acli * get *`）自动执行，写操作命令仍需 `confirm_service` 确认
- 可观测性：每个修复点需有对应日志事件 key

## 验收标准

- [ ] 发送中文重复提问（相同语义不同表述），`repeat_question_count` 能正确递增
- [ ] ops-agent 模式下，S0 阶段 LLM 回复中包含活跃告警数据
- [ ] ReactExecutor 模式下 S3 阶段，system_prompt 包含 `S3-验证执行` 方法论
- [ ] S1 阶段 KB 检索直接使用 `conversation.category_id`，Grafana 看不到 `classify_intent` 调用
- [ ] ops-agent 输出包含"开始验证"关键词时，`diagnostic_stage` 不发生变化
- [ ] S3 阶段 LLM 回复包含 `acli` 命令时，自动执行结果追加在回复后

## 约束条件

- Python 环境管理使用 `uv`
- 不引入新的外部依赖（Redis 等 P3 N-7 改进推迟）
- 所有代码注释使用中文
