# 命令自动执行需求

## 变更历史
| 日期 | 版本 | 变更内容 | 关联事件文档 |
|------|------|---------|------------|
| 2026-05-11 | v1.0 | 初版 | — |

## 背景与问题

### 业务背景
HCI 排障助手当前命令执行链路存在**两个手动授权点**：
1. CommandBlock 上的「发送到终端」按钮（填充输入框）
2. 终端内用户手动按回车（确认执行）

AI 在诊断阶段通常需要连续执行多条诊断命令，工程师必须反复完成：
**看命令 → 点发送 → 按回车 → 看输出 → 切回对话 → 贴结果** 的循环。

### 核心痛点
- 连续多步诊断时，手动操作摩擦成本高
- 工程师需要在终端和对话窗口之间频繁切换
- 命令输出无法自动回填 AI 上下文，需要工程师手动贴结果

### 影响范围
所有通过 SSH 连接到客户 HCI 环境进行排障的工程师。

## 需求描述

### 功能概述
新增「命令自动执行」功能开关：**打开后，AI 建议的命令可自动通过 terminal_bridge 发送到已登录 SSH 对端执行，执行结果自动注入对话上下文，触发 AI 继续分析。**

### 用户场景
工程师已通过 SSH 连接到客户 HCI 环境后，在终端面板开启「安全模式」自动执行开关。AI 输出只读诊断命令后，无需工程师介入，命令自动执行并将结果追加到对话，AI 基于结果继续分析，直到给出最终诊断结论。

### 预期收益
- 消除连续诊断场景中大量重复的手动操作
- 对话上下文更完整，AI 诊断质量提升
- 排障时间缩短，提升工程师效率

## 功能需求

1. **自动执行开关**：在终端面板提供三级开关（关闭 / 安全模式 / 激进模式），默认关闭；仅 SSH 已连接时可见
2. **风险分级控制**：`danger` 级命令无论何种模式均需强制手动确认；`caution` 级命令在安全模式下拦截
3. **结果回填对话**：命令执行完成后，输出自动作为 `role:user` 消息追加到对话，触发 AI 继续分析
4. **串行执行**：多命令严格串行，同一时刻只有一条命令处于执行状态
5. **取消窗口**：命令自动执行前提供 3 秒倒计时，工程师可随时中断单条命令
6. **前端风险推断**：命令块根据 acli 命令前缀静态推断风险级别并展示对应标签
7. **执行状态可见**：CommandBlock 展示执行状态（等待 / 执行中 / 成功 / 失败）

## 非功能需求

- **安全**：`danger` 级命令强制拦截，无任何后门；默认模式为关闭
- **可靠**：串行锁防止命令并发；自动执行计数上限（10条/会话）防止 Agent Loop
- **可观测**：自动执行的命令在终端显示区域有视觉区分；开关状态持久化到 localStorage
- **降级**：SSH 断连时自动降级为手动模式

## 验收标准

- [ ] SSH 已连接时，终端面板头部显示自动执行开关（下拉：关闭/安全模式/激进模式）
- [ ] SSH 未连接时，开关不可见
- [ ] `safe-only` 模式下，`none`/`readonly` 级命令自动执行，`caution`/`danger` 级命令拦截
- [ ] `aggressive` 模式下，`none`/`readonly`/`caution` 自动执行，`danger` 强制确认弹框
- [ ] 任意模式下，`danger` 命令均弹出确认对话框
- [ ] 自动执行命令前显示 3 秒倒计时，工程师可点 × 取消
- [ ] 命令执行完成后，结果自动作为用户消息注入对话并触发 AI 继续分析
- [ ] 一次 AI 回复包含多个命令块时，仅自动执行第一条
- [ ] 连续 3 次命令失败时自动熔断（关闭自动执行 + 系统提示）
- [ ] 单次会话自动执行超过 10 条后自动关闭（+ 系统提示）
- [ ] 页面切到后台时暂停命令队列派发

## 约束条件

- **技术约束**：前端 Vue 3 + TypeScript；命令执行通过现有全局 `sshWebSocket`（`chat.ts`）完成，不新建连接
- **SSH 依赖**：功能强依赖 SSH 连接，无 SSH 时功能不可用
- **marker 协议**：沿用现有 `buildBridgeMarker` / `parseBridgeCommandResult` 协议

## 风险与假设

- **假设**：bridge 端已支持 `ssh_input` 命令流；marker 协议已验证可靠
- **已知风险**：命令风险推断依赖前端静态白名单，存在误判可能（宁可多拦截）
- **长期风险**：AI Agent Loop 需计数保护，否则可能无限循环
