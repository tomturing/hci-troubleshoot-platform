## 背景与问题

### 业务背景

HCI智能排障平台支持工程师通过SSH终端执行诊断命令。当前终端采用 **Bridge 模式**：
- SSH连接完全在 Windows 本地发起（通过 terminal_bridge.exe）
- 公网服务器不参与 SSH 流量
- 终端交互数据仅存在于浏览器内存中，页面刷新或关闭后数据丢失

### 核心痛点

1. **排障复盘困难**：工程师无法追溯诊断过程中执行了哪些命令、获得了什么输出
2. **知识沉淀缺失**：典型案例的操作序列无法沉淀为 SOP 参考
3. **审计合规空白**：缺少"谁在什么时候执行了什么命令"的审计记录
4. **诊断报告不完整**：生成的诊断报告缺少命令执行证据

### 影响范围

- **工程师**：无法复盘自己的排障过程，无法引用历史输出继续对话
- **管理员**：无法审计工程师的操作行为，无法从典型案例提取操作序列
- **AI Agent**：无法获取历史命令上下文辅助诊断

## 需求描述

### 功能概述

将终端上的操作记录（输入命令）和回显（输出结果）上传到远端数据库，以便后续在 custom-ui 和 admin-ui 中回放所有终端操作。

### 用户场景

#### 场景1：工程师排障复盘（custom-ui）

工程师完成一个工单诊断后，打开工单详情页 → 点击"终端历史" Tab → 查看本次诊断过程中执行的命令序列 → 回放终端输出 → 点击"引用到对话"将历史命令/输出插入当前对话。

#### 场景2：管理员审计（admin-ui）

管理员在工单管理界面查看所有工单 → 打开某个工单详情 → 查看该工单的终端操作记录 → 搜索关键词定位问题 → 导出审计报告。

#### 场景3：知识库沉淀

管理员从典型案例中提取有效的命令序列 → 作为 SOP 参考文档 → 新工程师学习排障方法。

### 预期收益

1. 排障过程可追溯、可复盘
2. 审计合规有据可查
3. 知识沉淀形成闭环
4. 诊断报告更完整可信

## 功能需求

### F1: 终端操作录制

- **F1.1 输入记录**：每条用户执行的命令（包括 AI 注入的命令）写入数据库
- **F1.2 输出记录**：命令执行的输出结果（含 ANSI 转义码）写入数据库
- **F1.3 时间戳**：记录每条操作的精确时间
- **F1.4 关联信息**：与 case_id、conversation_id、诊断阶段（S0-S6）关联
- **F1.5 序号维护**：每个工单的操作按序号递增，便于回放时排序

### F2: 数据存储

- **F2.1 数据库表**：新建 `terminal_operation` 表存储操作记录
- **F2.2 原始内容**：存储原始输出（含 ANSI 码），用于回放渲染
- **F2.3 纯文本索引**：同时存储剥离 ANSI 的纯文本，用于搜索
- **F2.4 大输出处理**：单条输出超过 64KB 时截断或分页

### F3: 前端回放（custom-ui）

- **F3.1 工单详情页**：新增"终端历史" Tab
- **F3.2 时间轴回放**：支持播放/暂停/速度控制（1x/2x/5x）
- **F3.3 操作列表**：表格形式快速浏览所有命令及结果状态
- **F3.4 阶段过滤**：按诊断阶段 S0-S6 筛选相关命令
- **F3.5 关键词搜索**：搜索命令或输出内容
- **F3.6 引用功能**：将历史命令/输出插入当前对话输入框
- **F3.7 导出功能**：导出为纯文本或 JSON

### F4: 管理端查看（admin-ui）

- **F4.1 工单详情页**：新增"终端历史" Tab（管理员视角）
- **F4.2 全工单审计**：查看所有工单的操作记录
- **F4.3 搜索过滤**：按用户、时间范围、关键词搜索
- **F4.4 统计报表**：命令执行频率统计（可选）
- **F4.5 导出审计**：导出审计报告

### F5: 权限控制

- **F5.1 custom-ui**：仅查看自己参与的工单
- **F5.2 admin-ui**：查看所有工单

## 非功能需求

### 性能要求

- 命令录制不影响终端实时性（异步批量上传）
- 回放加载时间 < 3秒（100条操作）
- 搜索响应 < 500ms

### 存储要求

- 单工单预期操作数：50-200条
- 单条输出平均大小：2KB-10KB
- 存储策略：与工单同生命周期

### 可观测性

- 录制成功率指标
- 上传延迟指标
- 存储容量监控

---

## 验收标准

- [ ] 用户执行命令后，能在数据库中查到对应的 input/output 记录
- [ ] custom-ui 工单详情页能显示"终端历史" Tab 并正确回放操作序列
- [ ] admin-ui 工单管理页能查看所有工单的终端操作记录
- [ ] 回放支持时间轴播放和列表查看两种模式
- [ ] 搜索功能能定位包含关键词的命令或输出
- [ ] 导出功能能生成纯文本/JSON格式文件
- [ ] 录制不影响终端实时交互体验

---

## 约束条件

### 技术约束

- 前端采用 Bridge 模式，录制逻辑在浏览器层实现
- 数据库使用 PostgreSQL（现有基础设施）
- 前端框架使用 Vue 3 + Element Plus + xterm.js

### 依赖约束

- 依赖现有的 SSH WebSocket 连接机制
- 依赖现有的工单/会话数据模型

## 风险与假设

### 已知风险

- 大输出可能占用大量存储空间 → 设置单条上限并压缩
- 实时上传可能影响性能 → 采用批量上传或 debounced 写入

### 假设条件

- 工单生命周期内操作记录有意义（工单关闭后仍可查看）
- 终端回放无需精确还原 ANSI 渲染效果（保留原始数据，回放时渲染）
