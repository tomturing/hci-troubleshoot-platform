# SOP 发布 500 错误 + 管理台按钮布局修复需求

## 变更历史
| 日期 | 版本 | 变更内容 |
|------|------|---------|
| 2026-04-14 | v1.0 | 初版 |

## 背景与问题
- **问题1**：SOP 管理页面点击「发布」后提示 `发布失败：HTTP 500`，但报错信息不具体，无法判断根因
- **问题2**：SOP 管理和 KBD 管理页面在浏览器缩放后，「搜索」和「重置」两个按钮不在同一高度
- **问题3**：KBD 管理操作列宽度不足，导致详情/编辑/通过/拒绝四个按钮在缩放时换行

## 需求描述
1. SOP 发布失败时前端应展示具体原因（而非 `HTTP 500`），后端须返回可读的错误消息
2. 过滤栏中「搜索」「重置」按钮在任何缩放比例下需保持同一行同一高度
3. KBD 操作列需保证 4 个按钮始终在同一行不换行

## 功能需求
1. 后端 `approve_sop_document` 需用 try-except 包裹，将未捕获异常转化为可读 HTTPException
2. 后端将 embedding 生成移出长事务，避免 DB 连接超时
3. API Gateway SOP 发布代理的超时从 60s 提升为 600s，并捕获 TimeoutException
4. 过滤栏按钮容器使用 flex 布局防止换行
5. KBD 操作列宽度增加至 260px，并加 `white-space: nowrap` 防止换行

## 验收标准
- [ ] SOP 发布失败时弹出消息包含具体原因（非 `HTTP 500`）
- [ ] SOP/KBD 管理页在浏览器缩放 80%~150% 范围内，「搜索」「重置」始终同行同高
- [ ] KBD 管理操作列 4 个按钮在正常及缩放状态下始终同行显示
