## 3. 发布后观察（30 分钟）

### 3.1 执行命令

```bash
NAMESPACE=hci RELEASE_NAME=hci-platform WINDOW_MINUTES=30 \
  bash scripts/release-observe.sh
```

### 3.2 输出内容

1. 基线快照（deploy/pods/helm history）。
2. 周期采样（默认每 60 秒）。
3. Warning 事件尾部快照。
4. 结束快照。

### 3.3 验收标准

1. 观察窗口内无持续异常 Pending/CrashLoop。
2. 无持续新增 Warning 事件。
3. 服务状态在窗口结束时稳定。

---

## 4. 回滚演练

### 4.1 演练模式（默认，不执行真实回滚）

```bash
NAMESPACE=hci RELEASE_NAME=hci-platform \
  bash scripts/rollback-drill.sh
```

### 4.2 执行模式（真实回滚）

```bash
NAMESPACE=hci RELEASE_NAME=hci-platform EXECUTE=true TARGET_REVISION=<REV> \
  bash scripts/rollback-drill.sh
```

### 4.3 验收标准

1. 可正确识别可回滚 revision。
2. 演练模式可生成报告且不改集群状态。
3. 执行模式回滚后 deployment/pod 恢复健康。

## 6. 常见问题

1. 问题：kubectl 权限不足。
- 处理：确认当前 kubeconfig 与命名空间 RBAC 权限。

2. 问题：helm history 无数据。
- 处理：确认 release_name 与 namespace 是否正确。

3. 问题：演练目标 revision 推导失败。
- 处理：手动传入 `TARGET_REVISION`。
