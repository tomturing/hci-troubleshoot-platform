## 1. 测试环境准备

### 1.1 前提条件

| 组件 | 要求 |
|------|------|
| Docker Engine | ≥ 29.x |
| Docker Compose | ≥ v5.x (plugin) |
| OpenClaw Gateway | 运行中，端口 18790 |
| Python | ≥ 3.12（单元测试） |

### 1.2 一键启动全部服务

```bash
cd /mnt/d/hci-troubleshoot-platform

# 1. 确认 OpenClaw 网关运行中（宿主机 systemd 用户服务）
systemctl --user status openclaw-gateway
# 应显示 active (running)

# 2. 启动所有 Docker 服务
docker compose up -d

# 3. 等待健康检查通过后验证
docker compose ps
```

### 1.3 服务端口一览

| 服务 | 端口 | 说明 |
|------|------|------|
| customer-ui | 3001 | 客户端 Web 界面 |
| admin-ui | 3002 | 管理后台 Web 界面 |
| api-gateway | 8000 | API 统一网关 |
| case-service | 8001 | 工单微服务 |
| conversation-service | 8002 | 对话微服务 |
| scheduler-service | 8003 | 调度微服务 |
| PostgreSQL | 5432 | 数据库 |
| Redis | 6379 | 缓存/Session |
| OpenClaw Gateway | 18790 | AI 网关（宿主机） |

### 1.4 停止所有服务

```bash
docker compose down          # 停止并删除容器（保留数据卷）
docker compose down -v       # 停止并清除所有数据（含数据库、Redis）
```

## 2. 服务健康检查

### 2.1 批量检查（一键脚本）

```bash
echo "=== 后端服务 ==="
for svc in "api-gateway:8000" "case-service:8001" "conversation-service:8002" "scheduler-service:8003"; do
  name=${svc%%:*}; port=${svc##*:}
  code=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:$port/health)
  echo "  $name(:$port) -> HTTP $code"
done

echo "=== 前端服务 ==="
for ui in "customer-ui:3001" "admin-ui:3002"; do
  name=${ui%%:*}; port=${ui##*:}
  code=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:$port/)
  echo "  $name(:$port) -> HTTP $code"
done
```

**通过标准**：所有服务返回 `200`。

### 2.2 OpenClaw 网关连通性验证

```bash
# 宿主机直接调用
curl -s -X POST http://localhost:18790/v1/chat/completions \
  -H "Authorization: Bearer hci-dev-openclaw-token" \
  -H "Content-Type: application/json" \
  -d '{"model":"zai/glm-4.7","messages":[{"role":"user","content":"hi"}],"max_tokens":10}' \
  | python3 -m json.tool

# 从容器内调用（验证 host.docker.internal 映射）
docker exec hci-conversation-service python3 -c "
import urllib.request, json
req = urllib.request.Request('http://host.docker.internal:18790/v1/chat/completions',
    data=json.dumps({'model':'zai/glm-4.7','messages':[{'role':'user','content':'ping'}],'max_tokens':5}).encode(),
    headers={'Authorization':'Bearer hci-dev-openclaw-token','Content-Type':'application/json'},
    method='POST')
resp = urllib.request.urlopen(req, timeout=30)
print('Status:', resp.status)
data = json.loads(resp.read())
print('AI:', data['choices'][0]['message']['content'])
"
```

**通过标准**：宿主机和容器内均返回 AI 文本响应。

## 4. API 接口 curl 验证

> **说明**：所有 API 统一经过 API Gateway（端口 8000）访问，前缀为 `/api/`。

### 4.1 工单完整生命周期

```bash
# === 变量设置 ===
CLIENT_ID="curl-test-001"
GATEWAY="http://localhost:8000"

# === 1. 创建工单 ===
echo "=== 1. 创建工单 ==="
CASE=$(curl -s -X POST "$GATEWAY/api/cases/" \
  -H "Content-Type: application/json" \
  -H "X-Client-ID: $CLIENT_ID" \
  -d '{
    "client_id": "'$CLIENT_ID'",
    "title": "CPU 使用率过高",
    "description": "生产环境服务器 CPU 持续 95% 以上",
    "priority": "high"
  }')
echo "$CASE" | python3 -m json.tool
CASE_ID=$(echo "$CASE" | python3 -c "import sys,json; print(json.load(sys.stdin)['case_id'])")
echo "工单号: $CASE_ID"

# 通过标准：返回 201，包含 case_id、status=created

# === 2. 查询工单详情 ===
echo ""
echo "=== 2. 查询工单详情 ==="
curl -s "$GATEWAY/api/cases/$CASE_ID" | python3 -m json.tool

# 通过标准：返回工单完整信息

# === 3. 查询客户端工单列表 ===
echo ""
echo "=== 3. 查询客户端工单列表 ==="
curl -s "$GATEWAY/api/cases/?client_id=$CLIENT_ID" | python3 -m json.tool

# 通过标准：返回数组，包含该客户端的所有工单

# === 4. 确认工单 ===
echo ""
echo "=== 4. 确认工单 ==="
curl -s -X PUT "$GATEWAY/api/cases/$CASE_ID/confirm" \
  -H "X-Client-ID: $CLIENT_ID" | python3 -m json.tool

# 通过标准：status 变为 confirmed

# === 5. 创建对话 ===
echo ""
echo "=== 5. 创建对话 ==="
CONV=$(curl -s -X POST "$GATEWAY/api/conversations/?case_id=$CASE_ID" \
  -H "X-Client-ID: $CLIENT_ID")
echo "$CONV" | python3 -m json.tool
CONV_ID=$(echo "$CONV" | python3 -c "import sys,json; print(json.load(sys.stdin)['conversation_id'])")
echo "会话ID: $CONV_ID"

# 通过标准：返回 201，包含 conversation_id

# === 6. 发送消息（SSE 流式 AI 响应） ===
echo ""
echo "=== 6. 发送消息 ==="
curl -s -N -X POST "$GATEWAY/api/conversations/$CONV_ID/message" \
  -H "Content-Type: application/json" \
  -H "X-Client-ID: $CLIENT_ID" \
  -d "{
    \"content\": \"CPU使用率95%以上怎么排查\",
    \"case_id\": \"$CASE_ID\",
    \"role\": \"user\"
  }" 2>&1 | head -40

# 通过标准：
#   - 返回 SSE 流：data: <AI文本片段>  （逐行输出）
#   - 最后一行：data: [DONE]
#   - AI 内容为真实排查建议

# === 7. 查询消息历史 ===
echo ""
echo "=== 7. 查询消息历史 ==="
curl -s "$GATEWAY/api/conversations/$CONV_ID/messages" | python3 -m json.tool

# 通过标准：包含 user 和 assistant 两条消息

# === 8. 关闭工单 ===
echo ""
echo "=== 8. 关闭工单 ==="
curl -s -X PUT "$GATEWAY/api/cases/$CASE_ID/close" \
  -H "Content-Type: application/json" \
  -H "X-Client-ID: $CLIENT_ID" \
  -d '{"resolution": "已提供排查步骤"}' | python3 -m json.tool

# 通过标准：status=closed，closed_at 有时间戳
```

## 7. 常见问题排查

### 7.1 前端报 404 `/api/api/cases/`（双前缀）

**原因**：API 方法路径中包含了多余的 `/api` 前缀，与 Axios `baseURL('/api')` 叠加。

**验证**：浏览器控制台查看请求 URL，确认没有 `/api/api/` 双前缀。

**解决**：确保 `frontend/shared/src/api.ts` 中所有 API 路径以 `/cases/`、`/conversations/` 开头（不带 `/api`），因为 `baseURL` 已经是 `/api`。

### 7.2 OpenClaw 网关连接失败（容器内 Connection refused）

**可能原因**：
1. OpenClaw 绑定在 `127.0.0.1` 而非 `0.0.0.0`
2. 端口被幽灵进程占用（WSL2 常见）
3. `extra_hosts` 未配置

**验证和解决**：

```bash
# 检查 OpenClaw 绑定地址
ss -tlnp | grep 18790
# 应为 0.0.0.0:18790

# 检查 docker-compose.yml 中 conversation-service 配置
# 必须包含：
#   OPENCLAW_BASE_URL: http://host.docker.internal:18790
#   extra_hosts: ["host.docker.internal:host-gateway"]

# 从容器内验证
docker exec hci-conversation-service getent hosts host.docker.internal
# 应返回 172.17.0.1
```

### 7.3 SSE 流式响应报错

**错误**：`Attempted to call a sync iterator on an async stream`

**验证**：直接调用 conversation-service（绕过 gateway）：
```bash
curl -s -N -X POST http://localhost:8002/api/conversations/<conv_id>/message \
  -H "Content-Type: application/json" \
  -d '{"content":"test","case_id":"<case_id>","role":"user"}'
```

### 7.4 数据库连接失败

```bash
# 检查 PostgreSQL 容器状态
docker compose ps postgres
# 应显示 healthy

# 手动测试连接
docker exec hci-postgres pg_isready -U hci_admin -d hci_troubleshoot
```

### 7.5 端口冲突

```bash
# 检查端口占用
ss -tlnp | grep -E "5432|6379|8000|8001|8002|8003|3001|3002|18790"

# 如有冲突，停止占用进程后重启
docker compose down && docker compose up -d
```

### 7.6 清理测试数据

```bash
# 方法 1：删除测试工单（保留其他数据）
docker exec hci-postgres psql -U hci_admin -d hci_troubleshoot \
  -c "DELETE FROM messages WHERE case_id IN (SELECT case_id FROM cases WHERE client_id LIKE 'curl-test-%' OR client_id LIKE 'e2e-test-%');"
docker exec hci-postgres psql -U hci_admin -d hci_troubleshoot \
  -c "DELETE FROM conversations WHERE case_id IN (SELECT case_id FROM cases WHERE client_id LIKE 'curl-test-%' OR client_id LIKE 'e2e-test-%');"
docker exec hci-postgres psql -U hci_admin -d hci_troubleshoot \
  -c "DELETE FROM cases WHERE client_id LIKE 'curl-test-%' OR client_id LIKE 'e2e-test-%';"

# 方法 2：完全重置数据库（所有数据清空）
docker compose down -v && docker compose up -d
```
