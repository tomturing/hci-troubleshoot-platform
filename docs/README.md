## 3. 文档状态定义

1. active
- 当前有效，日常执行应以此为准。

2. reference
- 仍有参考价值，但不是当前唯一标准。

3. archive
- 已归档，仅用于历史追溯，不作为当前执行标准。

> 每个文档的当前状态见：
- [26_文档状态索引.md](26_文档状态索引.md)

历史归档目录：
- [archive](archive)

## 4. 维护规则（轻量）

1. 流程变更必须同步更新主链路文档（21/24/25）。
2. 新增文档必须在本页登记入口。
3. 主题重复时，保留一个主文档，其余转为 reference 或 archive。
4. 每两周做一次索引清理，更新状态与下一步动作。
