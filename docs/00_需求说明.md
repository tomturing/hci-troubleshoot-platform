# HCI 智能排障平台 - 需求规格说明书

---

## 1. 项目背景

### 1.1 业务场景

HCI（超融合基础设施）是一种将计算、存储和网络资源集成在一起的 IT 架构。运维人员在日常工作中需要处理各种复杂的技术问题，如：

- 虚拟机启动失败
- 存储性能下降
- 网络连接异常
- 集群节点离线
- 备份任务失败

传统排障方式依赖运维人员的经验和知识，效率低下且容易遗漏关键步骤。

### 1.2 解决方案

开发一个**智能排障平台**，采用云端Server为核心、多客户端接入的架构。

- 通过自然语言对话描述问题
- AI结合环境信息提供排障建议
- 支持分步指导和命令建议
- 记录排障过程形成知识库
- 全链路追踪和可观测性

## 2. 需求目标

### 2.1 核心目标

| 目标 | 描述 |
|------|------|
| **即时排障** | 用户用自然语言描述问题，平台即时打通客户端和AI引擎通路 |
| **智能调度** | 根据API Gateway的Session智能调度拉起和关闭AI引擎POD |
| **知识积累** | 记录排障过程（Conversation信息落库），形成可复用的知识库 |
| **系统对接** | 支持对接第三方系统，如工单系统、知识库系统等 |

### 2.2 功能需求

#### MVP阶段功能

**Web端功能**（MVP测试验证工具）:
1. **case管理**: 查询case、创建case、确认case、关闭case
2. **session建立**: 与gateway建立session
3. **对话交互**: AI响应case请求，与用户对话
4. **命令展示**: 显示AI建议的命令

**云端Server核心功能**:
1. **case管理**: 根据web端唯一标识ClientID查询Case、创建Case、更新Case、关闭Case
2. **session管理**: session创建、关闭、状态维护
3. **对话管理**: 所有对话信息落库，同一CaseID，所有对话信息落库
4. **容器管理**: 根据CaseID，拉起或关闭AI引擎POD，基于OpenClaw + Zhipu API的智能对话理
5. **可观测性**: 全链路TraceID追踪、指标采集、结构化日志

#### 可观测性设计

**监控范围说明**: 可观测性系统用于监控**排障平台自身**（云端Server），确保排障平台的稳定性和可维护性，而不是监控用户的HCI集群环境。

**TraceID全链路追踪**:
- 格式: `hci-{timestamp}-{random}`
- 贯穿: 网关 → 服务 → 数据库 → OpenClaw调用 → 日志
- 用途: 完整追踪每个请求在排障平台内的调用路径

**监控对象**（排障平台自身）:
- ✅ API Gateway: 请求路由、TraceID生成、响应时间
- ✅ 工单服务: 工单CRUD操作、数据库操作
- ✅ 对话服务: OpenClaw调用成功率、响应时间
- ✅ 数据采集服务: 环境信息写入性能
- ✅ PostgreSQL: 连接池、慢查询、trace_id索引查询
- ✅ Redis: 缓存命中率、内存使用

**不监控的对象**:
- ❌ HCI集群: 这是被排障的对象，不是监控对象
- ❌ OpenClaw内部: 独立服务，有自己的可观测性（我们只监控调用它的成功率和响应时间）
- ❌ Zhipu API: 第三方服务（我们只监控调用成功率）

**观测点示例**:
- API Gateway: HTTP请求统计、TraceID生成、路由转发耗时
- 微服务: 业务逻辑耗时、数据库操作耗时、OpenClaw API调用耗时
- 数据库: SQL查询性能、trace_id索引查询性能
- 日志: 结构化日志（JSON格式），所有日志包含trace_id字段

> **重要说明**: Web前端仅为MVP阶段的测试验证工具，用于验证云端Server功能完整性，不是正式客户端功能。正式客户端为Windows客户端。

#### 未来功能（不在MVP范围）
- 用户认证和授权系统
- 环境信息采集
- 知识库系统

## 4. 用户故事

### US-001: 打开排障助手
**作为**临时或者认证用户
**我想要**打开排障助手聊天页面
**以便**获得工单状态

**验收标准**:
- 通过ClientID自动获取到工单信息
- 如果没有工单，提示新建工单；如果有工单，确认是继续排障还是关闭工单
- 关闭工单后提示新建工单；系统生成唯一工单号 (QYYYYMMDDXXXXX)
- 新建工单后，自动连接热备AI引擎POD，并建立session

### US-002: 对话排障助手
**作为**临时或者认证用户
**我想要**创建工单或者选择已有工单后进入聊天页面，描述问题
**以便**获得 AI 的排障建议

**验收标准**:
- AI 根据工单描述信息给出初始响应
- 客户端自动收集用户信息，现场环境信息包括但不限于客户信息、环境信息
- 多次沟通引导客户排障；支持追问和补充信息
- 保留对话历史

### US-003: 获取操作命令
**作为**临时或者认证用户 
**我想要**对话过程中，获取具体的操作命令
**以便**直接执行排障命令

**验收标准**:
- AI 提供可执行的命令
- 命令有适当的说明
- 警告危险操作

### US-004: 关闭工单
**作为**临时或者认证用户 
**我想要**对话结束后，关闭工单
**以便**结束排障

**验收标准**:
- AI 已提供最终解决方案
- 提出关闭工单的建议
- 确认关闭工单

## 5. 约束条件

| 约束 | 说明 |
|------|------|
| 微服务架构 | API Gateway + Case服务 + Conversation服务 + Scheduler服务 |
| AI 引擎 | OpenClaw作为独立服务（代码位于/mnt/d/openclaw） + Zhipu API（智谱AI） |
| 全链路追踪 | 所有请求必须使用TraceID进行追踪 |
| 运行环境 | Linux (支持 WSL2)、Docker、Kubernetes、UV、PNPM |
