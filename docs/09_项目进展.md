### 2.3 核心业务功能联调

- **工单流程**: 创建→查询→确认→关闭，全生命周期 CRUD 验证通过
- **对话流程**: 创建对话→发送消息→SSE流式AI响应→后台异步保存→历史查询
- **跨服务协同**: API Gateway → Case/Conversation Service 代理链路打通
- **E2E验证**: `scripts/docker-e2e-test.sh` 脚本完整覆盖工单创建→对话绑定→SSE打字机回传→消息历史查询

### 2.4 可观测性基建

- **链路追踪**: OTel SDK 自动仪表化（FastAPI / HTTPX / SQLAlchemy），W3C traceparent 跨服务传播
- **结构化日志**: JSON stdout → Promtail → Loki，每条日志自动注入 trace_id / span_id
- **Trace↔Log 双向钻取**: Tempo瀑布图点击Span→Loki日志；Loki日志点击trace_id→Tempo瀑布图
- **一键部署**: `docker-compose-obs.yml` 拉起 Loki + Tempo + Promtail + Grafana
- **自动Provisioning**: Grafana 启动自动配置 Loki/Tempo 数据源，开箱即用

### 2.5 OpenClaw 容器化接入

- **容器化部署**: OpenClaw Gateway 以 Docker 容器运行（端口 18789），纳入业务栈编排
- **真实模型对接**: 通过 `ZAI_API_KEY` 对接 z.ai GLM 模型，SSE流式输出返回真实AI响应

### 2.10 架构 v2.0 升级设计（2026-02-27）

**升级动因**: 识别到四大业务场景需要多助手架构：
1. **多并发** — 云服务场景，0-N 用户同时使用，各需独立 AI 实例

4. **统一性** — 统一知识库 + Prompt，横向对比不同助手的诊断准确率

**设计产出**:
- `01_架构设计.md` 重写为 v2.0 (1200+ 行)，全面泛化 "OpenClaw Pod Pool" → "AI Assistant Pod Pool"

### 5.2 各问题详细分析

#### P4：工单创建 500（DATABASE_URL @ 编码）

**现象：** `POST /api/cases` 返回 HTTP 500，日志 `password authentication failed`

**根因：** Helm 模板用字符串拼接 DATABASE_URL，密码 `aihci@aclient2025` 中的 `@` 使 URL 解析器以为主机段从该位置开始，用户名/密码均截断错误。

**修复（commit `4b30de8`）：**
```yaml
# deploy/helm/hci-platform/templates/*/deployment.yaml
value: {{ printf "postgresql+asyncpg://%s:%s@postgres:5432/%s"
    .Values.config.postgresUser
    (.Values.secrets.postgresPassword | urlquery)
    .Values.config.postgresDb | quote }}
```

**防复发：** 密码禁含 URL 特殊字符，或 Helm 模板一律用 `urlquery` encode。

---

#### P6 + P7：admin-ui 空白页（vite base + nginx 双重故障）

**现象：** `http://<IP>/admin/` 返回 200 但页面全白，JS/CSS 请求 404 或请求到 customer-ui 资源。

**根因一（P6）：** `vite.config.ts` `base: '/'` → 打包后 `index.html` 引用 `/assets/index.js`（绝对路径） → Traefik 路由 `/assets/` 命中 `/`（customer-ui）→ 404。

**根因二（P7）：** 修复 nginx.conf 时 `replace_string_in_file` 操作截断了 `server{}` 块，末尾出现游离的 `location` 块，nginx 启动报配置解析错误。

**修复（commit `4b30de8`）：**
1. `vite.config.ts`: `base: '/admin/'`
2. `router/index.ts`: `createWebHistory('/admin/')`
3. `nginx.conf`: 完整重写，使用 `alias` + `try_files $uri $uri/ /index.html`

**防复发：** 见 PIT-023（三处必须并改）、PIT-025（HTML no-cache）。

---

#### P9：Grafana 路由显示 customer-ui（跨命名空间 Ingress 失效）

**现象：** 访问 `/grafana` 返回 customer-ui 的 HTML，Traefik 日志 `Cannot create service: not found`。

**诊断路径：**
1. `curl /grafana/` 返回 `Server: nginx/1.29.5`（customer-ui），排除浏览器缓存
2. `k3s kubectl get svc grafana -n hci-troubleshoot` → **NotFound**（揭示根因：Ingress 和 Service 不在同一命名空间）
3. 尝试 ExternalName 桥接 → Traefik 报 `externalName services not allowed`

**修复（commit `9deb342`）：**
在 `hci-observability`（grafana Service 所在命名空间）创建独立 Ingress 资源，路由 `/grafana` → `grafana:3000`：
```yaml
# templates/observability/ingress.yaml
namespace: {{ .Values.observability.namespace }}   # hci-observability
rules:
  - http:
      paths:
        - path: /grafana
          pathType: Prefix
          backend:
            service:
              name: grafana
              port: 3000
```

**防复发：** 见 PIT-024（Traefik 全命名空间扫描 Ingress，但同条 Ingress 只能引用本命名空间 Service）。


#### 6.1.2 前端 Dockerfile layer 顺序优化（PIT-029）

**问题：** 原 Dockerfile 先 `COPY` 全量源码再 `npm install`，源码任何改动都导致
依赖层缓存失效，每次构建 8-15 分钟。

**修复：** 拆分 `package.json` COPY → `npm install` 为独立层，置于源码 COPY 之前：
```dockerfile
COPY package.json .npmrc ./
COPY shared/package.json shared/
COPY admin/package.json admin/
RUN npm install          # 仅 package.json 变化时才重跑
COPY shared/ shared/
COPY admin/ admin/
RUN vite build           # 源码变化只重跑这一步（约 20s）
```

#### 6.1.3 OpenClaw Control UI 设备身份报错修复（PIT-026）

**现象：** 打开 OpenClaw WebUI `/control` 页面提示 `requires device identity (use HTTPS or localhost secure context)`

**根因：** K3s 环境通过 HTTP（非 localhost）访问，浏览器 DeviceAPI 被阻断；`openclaw.json` 缺少绕过开关。

**修复：**
```bash
python3 -c "
import json; path='/home/node/.openclaw/openclaw.json'
with open(path) as f: cfg=json.load(f)
cfg['gateway']['controlUi']['dangerouslyDisableDeviceAuth'] = True
with open(path,'w') as f: json.dump(cfg, f, indent=2, ensure_ascii=False)
"
k3s kubectl rollout restart deployment/openclaw -n hci-troubleshoot
```

#### 6.1.4 OpenClaw LLM 超时修复（PIT-027）

**现象：** OpenClaw 对话请求全部 `timed out`

**根因：** 配置的模型 `zai/glm-4.7` 对应域名 `api.zai.chat`，被 Clash TUN 劫持到 `198.18.0.4`，SSL 握手失败。

**修复：** 切换为 `tly/glm-5`（对应 `open.bigmodel.cn`，国内可直连）：
```bash
python3 -c "
import json; path='/home/node/.openclaw/openclaw.json'
with open(path) as f: cfg=json.load(f)
cfg['agents']['defaults']['model'] = {'primary': 'tly/glm-5'}
with open(path,'w') as f: json.dump(cfg, f, indent=2, ensure_ascii=False)
"
```

**验证：** `curl -o /dev/null -w "%{http_code} %{time_total}s" https://open.bigmodel.cn/api/paas/v4/models` → `200 0.25s` ✅

#### 6.1.5 Docker build --network host（PIT-028）

**问题：** 在 Clash TUN 宿主机上 `docker build` 时，容器内 `npm install` 报 `ETIMEDOUT 198.18.x.x`，即使配国内 mirror 也一样。

**根因：** Docker 容器网络独立于宿主机网络，Clash TUN 不覆盖容器内流量，DNS 劫持导致所有出口连接超时。

**修复：** 固定使用 `--network host` 构建前端镜像：
```bash
docker build --network host -t hci-admin-ui:2026.03.07-2 -f frontend/admin/Dockerfile frontend/
```

```bash
from fastapi import FastAPI
from pydantic import BaseModel
from sentence_transformers import SentenceTransformer

app = FastAPI()

class EmbedRequest(BaseModel):
```
