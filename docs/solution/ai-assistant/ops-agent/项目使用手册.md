# Ops Agent 项目使用手册

## 1. 快速开始

### 1.1 环境要求

- **Python**: 3.12 或更高版本
- **依赖管理**: uv (推荐) 或 pip
- **模型服务**: 可用的 LLM API (OpenRouter、OpenAI、Anthropic 等)
- **SOP 数据**: 本地 SOP 知识库文件

### 1.2 安装步骤

#### 方法一：使用 uv (推荐)

```bash
# 1. 克隆或进入项目目录
cd /path/to/ops-agent

# 2. 安装 uv (如果还没有)
pip install uv

# 3. 创建虚拟环境
uv venv --python 3.12

# 4. 激活虚拟环境
source .venv/bin/activate  # Linux/Mac
# 或
.\.venv\Scripts\activate  # Windows PowerShell

# 5. 安装项目依赖
uv pip install -e .
```

#### 方法二：使用 Makefile

```bash
# 安装开发环境
make install-dev

# 运行测试
make test

# 使用 uv 运行测试
make uv-test
```

### 1.3 配置项目

#### 1.3.1 创建配置文件

```bash
# 复制配置文件模板
cp ops_config.yaml.example ops_config.yaml
```

#### 1.3.2 配置模型提供商

编辑 `ops_config.yaml`，配置你的模型提供商：

```yaml
model_providers:
  openrouter-example:
    api_key: "sk-your-api-key-here"  # 或者使用环境变量
    base_url: https://openrouter.ai/api/v1
    provider: openrouter
```

**推荐使用环境变量**（更安全）：

```bash
# 设置 API Key
export OPENROUTER_API_KEY="sk-your-api-key-here"
export OPENROUTER_BASE_URL="https://openrouter.ai/api/v1"

# 如果使用其他提供商
export OPENAI_API_KEY="..."
export ANTHROPIC_API_KEY="..."
```

环境变量命名规则：`{PROVIDER_NAME}_API_KEY` 和 `{PROVIDER_NAME}_BASE_URL`，其中 `PROVIDER_NAME` 是大写的提供商名称。

#### 1.3.3 配置模型

```yaml
models:
  glm47-openrouter:
    model_provider: openrouter-example  # 引用上面的 provider
    model: z-ai/glm-4.7
    max_tokens: 8192
    temperature: 0.8
    top_p: 0.95
    top_k: 0
    max_retries: 3
    parallel_tool_calls: false
    enable_thinking: false
```

#### 1.3.4 配置 Agent

```yaml
agents:
  enable_lakeview: false
  ops_agent:
    model: glm47-openrouter  # 引用上面的 model
    max_steps: 500
    sop_catalog_path: data/case_sop_data/af/sop/node_sops.jsonl
    query_sub_agent:
      model: glm47-openrouter
      max_steps: 80
      tool_call_budget: 40
      budget_warning_threshold: 10
```

**关键配置项说明**：

| 配置项 | 说明 |
|-------|------|
| `sop_catalog_path` | SOP 知识库文件路径，可以是相对路径或绝对路径 |
| `max_steps` | 主 Agent 最大执行步数 |
| `query_sub_agent.max_steps` | 子 Agent 最大执行步数 |
| `query_sub_agent.tool_call_budget` | 子 Agent 工具调用预算 |

### 1.4 准备 SOP 数据

确保 SOP 知识库文件存在于配置的路径：

```bash
# 检查示例数据
ls -la data/case_sop_data/

# 目录结构示例
data/case_sop_data/
├── af/
│   ├── case/
│   └── sop/
│       ├── node_sops.jsonl
│       ├── case_facts.jsonl
│       └── sop_node_selection.summary.json
└── hci/
    └── ...
```

如果使用自己的 SOP 数据，请确保格式符合要求，然后更新 `sop_catalog_path` 配置。

## 2. CLI 使用指南

### 2.1 基本命令

#### 查看帮助

```bash
# 查看全局帮助
ops-cli --help

# 查看子命令帮助
ops-cli run --help
ops-cli show-config --help
ops-cli tools --help
```

#### 运行排障任务

```bash
# 方式一：直接传入问题
ops-cli run "排查 VPN 无法连接" --working-dir /absolute/path/to/workdir

# 方式二：从文件读取问题
ops-cli run --file task.txt --working-dir /absolute/path/to/workdir

# 方式三：自动批准所有工具调用（非交互模式）
ops-cli run "排查服务启动失败" --working-dir /absolute/path/to/workdir --auto-approve
```

**常用参数**：

| 参数 | 说明 |
|-----|------|
| `task` | 直接传入任务描述（与 `--file` 二选一） |
| `--file, -f` | 从文件读取任务描述 |
| `--working-dir, -w` | 指定工作目录（必须是绝对路径） |
| `--provider` | 覆盖模型提供商 |
| `--model, -m` | 覆盖模型名称 |
| `--model-base-url` | 覆盖模型 API Base URL |
| `--api-key, -k` | 覆盖 API Key |
| `--max-steps` | 覆盖主 Agent 最大执行步数 |
| `--config-file` | 指定配置文件路径（默认 `ops_config.yaml`） |
| `--trajectory-file, -t` | 指定轨迹输出目录 |
| `--console-type, -ct` | 控制台样式（默认 `compact`） |
| `--auto-approve, -aa` | 自动批准工具调用 |
| `--debug-llm` | 输出原始模型响应（调试用） |

#### 查看当前配置

```bash
ops-cli show-config
```

这会显示当前生效的配置，包括：
- Agent 配置
- 模型配置
- 模型提供商配置
- 解析后的配置值（环境变量覆盖等）

#### 查看可用工具

```bash
ops-cli tools
```

列出所有注册的工具及其描述。

## 4. 配置详解

### 4.1 完整配置文件示例

```yaml
# =============================================================================
# Agent 配置
# =============================================================================
agents:
  # 是否启用 Lakeview 功能
  enable_lakeview: false

  # OpsAgent 主配置
  ops_agent:
    # 使用的模型（引用下面 models 中的配置）
    model: glm47-openrouter

    # 最大执行步数
    max_steps: 500

    # SOP 知识库路径
    sop_catalog_path: data/case_sop_data/af/sop/node_sops.jsonl

    # SOPQuerySubAgent 子 Agent 配置
    query_sub_agent:
      # 子 Agent 使用的模型
      model: glm47-openrouter

      # 子 Agent 最大执行步数
      max_steps: 80

      # 工具调用预算
      tool_call_budget: 40

      # 预算警告阈值
      budget_warning_threshold: 10

# =============================================================================
# Lakeview 配置 (可选)
# =============================================================================
lakeview:
  model: lakeview_model

# =============================================================================
# 模型提供商配置
# =============================================================================
model_providers:
  # OpenRouter 示例
  openrouter-example:
    api_key: ""  # 建议使用环境变量 OPENROUTER_API_KEY
    base_url: https://openrouter.ai/api/v1
    provider: openrouter

  # OpenAI 示例
  openai-example:
    api_key: ""  # 建议使用环境变量 OPENAI_API_KEY
    base_url: https://api.openai.com/v1
    provider: openai

  # Anthropic 示例
  anthropic-example:
    api_key: ""  # 建议使用环境变量 ANTHROPIC_API_KEY
    base_url: https://api.anthropic.com/v1
    provider: anthropic

# =============================================================================
# 模型配置
# =============================================================================
models:
  # GLM-4.7 via OpenRouter
  glm47-openrouter:
    model_provider: openrouter-example
    model: z-ai/glm-4.7
    max_tokens: 8192
    temperature: 0.8
    top_p: 0.95
    top_k: 0
    max_retries: 3
    parallel_tool_calls: false
    enable_thinking: false

  # GPT-4 via OpenAI
  gpt-4-openai:
    model_provider: openai-example
    model: gpt-4
    max_tokens: 4096
    temperature: 0.7
    top_p: 1.0
    top_k: 0
    max_retries: 3
    parallel_tool_calls: true
    enable_thinking: false

  # Claude 3.5 Sonnet via Anthropic
  claude-3-5-sonnet:
    model_provider: anthropic-example
    model: claude-3-5-sonnet-20241022
    max_tokens: 8192
    temperature: 0.7
    top_p: 1.0
    top_k: 0
    max_retries: 3
    parallel_tool_calls: false
    enable_thinking: false

  # Lakeview 模型
  lakeview_model:
    model_provider: openrouter-example
    model: z-ai/glm-4.7
    max_tokens: 4096
    temperature: 0.0
    top_p: 1.0
    top_k: 0
    max_retries: 1
    parallel_tool_calls: false
    enable_thinking: false

# =============================================================================
# MCP 服务器配置 (可选)
# =============================================================================
mcp_servers:
  # 示例：本地文件服务器
  # filesystem:
  #   command: npx
  #   args:
  #     - -y
  #     - @modelcontextprotocol/server-filesystem
  #   env:
  #     PATH: /usr/local/bin:/usr/bin:/bin
  #   cwd: /

# 允许的 MCP 服务器列表
allow_mcp_servers: []
#  - filesystem
```

### 4.2 模型参数调优

#### temperature (温度)

控制输出的随机性：
- **较低值 (0.0-0.3)**：更确定、更聚焦的输出
- **中等值 (0.4-0.7)**：平衡创造性和确定性
- **较高值 (0.8-1.0)**：更多样化、更有创造性的输出

推荐配置：
```yaml
# 主 Agent：需要一定的创造性
temperature: 0.8

# 子 Agent：需要更确定的检索
temperature: 0.7
```

#### top_p 和 top_k

控制 token 采样策略：
- **top_p**：从累计概率达到 top_p 的 token 中采样
- **top_k**：从概率最高的 top_k 个 token 中采样

推荐配置：
```yaml
top_p: 0.95
top_k: 0  # 0 表示禁用
```

#### max_tokens

控制单次响应的最大 token 数：
```yaml
max_tokens: 8192  # 根据模型能力调整
```

## 5. 部署指南

### 5.1 生产环境部署

#### 1. 系统要求

- **操作系统**: Linux (推荐)、macOS、Windows
- **CPU**: 2 核或以上
- **内存**: 4GB 或以上
- **磁盘**: 至少 1GB 可用空间

#### 2. 使用 uv 部署 (推荐)

```bash
# 1. 创建部署目录
mkdir -p /opt/ops-agent
cd /opt/ops-agent

# 2. 克隆或复制代码
git clone <repository-url> .  # 或复制文件

# 3. 创建虚拟环境
uv venv --python 3.12
source .venv/bin/activate

# 4. 安装依赖（不包含开发依赖）
uv pip install .

# 5. 创建配置文件
cp ops_config.yaml.example ops_config.yaml
# 编辑 ops_config.yaml，配置生产环境的 API Key 等

# 6. 准备数据目录
mkdir -p data

# 7. 测试运行
ops-cli show-config
```

#### 3. 使用 systemd 管理服务 (Linux)

创建服务文件 `/etc/systemd/system/ops-agent-web.service`：

```ini
[Unit]
Description=Ops Agent Web Service
After=network.target

[Service]
Type=simple
User=ops-agent
WorkingDirectory=/opt/ops-agent
Environment="PATH=/opt/ops-agent/.venv/bin"
Environment="OPENROUTER_API_KEY=sk-your-api-key"
ExecStart=/opt/ops-agent/.venv/bin/ops-web
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 创建用户
useradd -r -s /bin/false ops-agent
chown -R ops-agent:ops-agent /opt/ops-agent

# 重载 systemd
systemctl daemon-reload

# 启动服务
systemctl start ops-agent-web

# 设置开机自启
systemctl enable ops-agent-web

# 查看状态
systemctl status ops-agent-web

# 查看日志
journalctl -u ops-agent-web -f
```

#### 4. 使用 Docker 部署

创建 `Dockerfile`：

```dockerfile
FROM python:3.12-slim

WORKDIR /app

# 安装 uv
COPY --from=ghcr.io/astral-sh/uv:latest /uv /usr/local/bin/uv

# 复制项目文件
COPY . .

# 安装依赖
RUN uv venv --python 3.12 && \
    .venv/bin/pip install .

# 配置环境
ENV PATH="/app/.venv/bin:$PATH"
ENV OPS_CONFIG_FILE="/app/ops_config.yaml"

# 暴露端口 (Streamlit 默认 8501)
EXPOSE 8501

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
    CMD curl -f http://localhost:8501/_stcore/health || exit 1

# 启动命令
CMD ["ops-web"]
```

构建和运行：

```bash
# 构建镜像
docker build -t ops-agent:latest .

# 运行容器
docker run -d \
  --name ops-agent \
  -p 8501:8501 \
  -v /path/to/ops_config.yaml:/app/ops_config.yaml:ro \
  -v /path/to/data:/app/data:ro \
  -e OPENROUTER_API_KEY="sk-your-api-key" \
  ops-agent:latest
```

### 5.2 反向代理配置

#### Nginx 配置示例

```nginx
server {
    listen 80;
    server_name ops-agent.your-domain.com;

    location / {
        proxy_pass http://localhost:8501;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        # 超时配置
        proxy_read_timeout 86400;
        proxy_send_timeout 86400;
    }
}

# HTTPS 配置 (使用 Let's Encrypt)
server {
    listen 443 ssl http2;
    server_name ops-agent.your-domain.com;

    ssl_certificate /etc/letsencrypt/live/ops-agent.your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/ops-agent.your-domain.com/privkey.pem;

    location / {
        proxy_pass http://localhost:8501;
        # ... 同上
    }
}
```

### 5.3 安全建议

1. **不要在配置文件中硬编码 API Key**
   - 使用环境变量
   - 或使用密钥管理服务

2. **限制文件权限**
   ```bash
   chmod 600 ops_config.yaml
   chmod 700 .trajectories/
   ```

3. **使用 HTTPS**
   - 在生产环境中启用 HTTPS
   - 使用 Let's Encrypt 等免费证书

4. **配置防火墙**
   - 只允许必要的端口访问
   - 限制来源 IP

5. **定期更新依赖**
   ```bash
   uv pip list --outdated
   uv pip install --upgrade <package>
   ```

## 6. 诊断与调试

### 6.1 常见问题

#### 问题 1：配置文件解析错误

**错误信息**：
```
ConfigError: Error parsing YAML config: ...
```

**解决方案**：
- 检查 YAML 格式是否正确
- 确保缩进使用空格（不是 tab）
- 使用 YAML 验证工具检查格式

#### 问题 2：模型 API 连接失败

**错误信息**：
```
Error connecting to model provider: ...
```

**解决方案**：
- 检查 API Key 是否正确
- 检查网络连接
- 检查 base_url 是否正确
- 确认环境变量已正确设置

#### 问题 3：SOP 文件找不到

**错误信息**：
```
FileNotFoundError: No such file or directory: ...
```

**解决方案**：
- 检查 `sop_catalog_path` 配置是否正确
- 确认文件存在且有读取权限
- 使用绝对路径

#### 问题 4：Agent 陷入无限循环

**现象**：Agent 重复调用相同的工具

**解决方案**：
- 检查是否触发了重复调用检测
- 调整 `max_steps` 限制
- 使用 `--debug-llm` 查看详细信息

### 6.2 调试技巧

#### 1. 启用 LLM 调试输出

```bash

```

#### 2. 查看当前配置

- 工具调用和结果
- 时间信息

可以使用 JSONL 查看工具分析轨迹文件。

```bash

```

确保代码正常工作。
