# ADR-003：AI 层与 RAG 层对接方式（三阶段演进）

## 状态

**已采纳**（2026-03-09），当前处于阶段一

---

## 背景

项目 AI 层使用 openclaw 作为 Agent Runtime，知识库通过 KB Service 提供 RAG 检索能力。需要决定：ProductionClaw 应主动拉取知识，还是由 conversation-service 负责注入？

### 发现的问题

当前架构将 openclaw 当作被动 LLM 使用：

```
conversation-service → POST /api/kb/sop/match
                     → POST /api/kb/search
                     → 拼装 4-Tier Prompt → POST /v1/chat/completions
                                                    ↓
                                             openclaw（被当成 LLM）
```

这违背了 openclaw 的真实能力定位（完整 tool-calling Agent loop），导致：
- openclaw 不知道 KB Service 存在，无法自主决策何时检索
- 无法实现多跳检索
- `productionclaw/TOOLS.md` 中定义的 KB 调用规范形同虚设

---

## 决策问题

> ProductionClaw 应该**主动拉取**知识（Agent Tool），还是**被动接收**注入（Prompt 预烘焙）？

---

## 三阶段演进方案

> 三个阶段是线性递进关系，不是并列可选。

### 阶段一：修复现状（外挂 RAG，最小改动）

**当前处于此阶段**

**核心变化**：部署 kb-service，让 4-Tier Prompt 注入机制实际工作。

```
用户消息
  → conversation-service（RAG 编排）
      ├── POST /api/kb/sop/match  → SOP 全文注入（Tier-2）
      ├── POST /api/kb/search     → 语义检索 top-k（Tier-3）
      └── 拼装完整 system prompt → openclaw（接收预注入知识）
```

**优势**：改动最小，可快速验证 RAG 效果
**劣势**：openclaw 的 Agent 能力未充分利用；无法多跳检索

### 阶段二：ProductionClaw 主动检索（Agent Tool 模式）

**目标态过渡阶段**

ProductionClaw 通过 Tool Calling 主动决策何时、检索什么：

```
用户消息 → ProductionClaw（Agent Loop）
                ↓ 当需要知识时
          调用 search_knowledge(query)
                ↓
          KB Service → 返回检索结果
                ↓
          ProductionClaw 决定是否需要再次检索（多跳）
                ↓
          生成最终回复
```

**conversation-service 职责降级**：从 RAG 编排者 → 纯消息路由 + 历史记录

### 阶段三：LearningClaw ↔ ProductionClaw 自主知识闭环

ProductionClaw 处理工单 → LearningClaw 提炼案例摘要 → 写入 KB → 供未来 ProductionClaw 检索

---

## 结论

- **阶段一（当前）**：kbService.enabled = true，部署 KB Service，4-Tier Prompt 实际生效
- **阶段二（下一阶段）**：adapters/ 下为 ProductionClaw 注册 KB Tool，conversation-service 解除 RAG 编排职责  
- **阶段三（中长期）**：LearningClaw 自动提炼工单知识并写回 KB

---

## 后果

**阶段一**：
- 需要在 K3s 中启用 kb-service（当前 `values.yaml kbService.enabled: false`）
- 现有 4-Tier Prompt 架构不变

**阶段二转换时**：
- conversation-service 的 RAG 注入逻辑需要下线
- openclaw TOOLS.md 需要注册 KB 相关工具
- 需要评估对话质量变化（可能需要 A/B 测试）
