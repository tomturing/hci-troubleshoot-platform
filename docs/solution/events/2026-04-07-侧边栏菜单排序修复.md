# 侧边栏菜单排序修复方案

## 背景与需求

Admin 管理台侧边栏菜单顺序显示错误，用户看到的是：
```
知识审核 → KBD审核 → 仪表盘 → 工单管理 → 分类基线 → 用户管理 → 系统监控
```

正确顺序应该是：
```
仪表盘 → 工单管理 → 分类基线 → 知识审核 → KBD审核 → 用户管理 → 系统监控
```

## 方案（WHAT）

在路由 meta 中添加 `order` 字段，App.vue 按 `order` 排序后渲染菜单。

### 代码改动

**frontend/admin/src/router/index.ts**：
```typescript
meta: { title: '仪表盘', icon: 'Odometer', order: 1 },
meta: { title: '工单管理', icon: 'Tickets', order: 2 },
meta: { title: '分类基线', icon: 'Histogram', order: 3 },
meta: { title: '知识审核', icon: 'Reading', order: 4 },
meta: { title: 'KBD 审核', icon: 'Document', order: 5 },
meta: { title: '用户管理', icon: 'User', order: 6 },
meta: { title: '系统监控', icon: 'Monitor', order: 7 },
```

**frontend/admin/src/App.vue**：
```typescript
const menuItems = router.getRoutes()
  .filter((r) => r.meta?.icon && !r.meta?.hidden)
  .sort((a, b) => (a.meta?.order as number) - (b.meta?.order as number))
  .map((r) => ({...}))
```

## 决策依据（WHY）

### 为什么选此方案

Vue Router 的 `getRoutes()` 返回顺序不保证与定义顺序一致，对动态导入的路由顺序是任意的。通过显式的 `order` 字段可以精确控制菜单顺序。

### 为什么不选其他方案

**方案A：修改路由定义顺序**：无效，`getRoutes()` 不保证返回顺序。

**方案B：硬编码菜单数组**：可行但维护成本高，路由定义和菜单分离容易不同步。
