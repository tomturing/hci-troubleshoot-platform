# KBD 截图识别重构方案

**日期**：2026-04-13  
**状态**：已完成  
**分支**：feature/kbd-screenshot-ocr-redesign

---

## 架构变更

### 废弃 PaddleOCR

原因：PaddleOCR 3.x 在 WSL2/Python 3.13 环境下 OneDNN 框架层抛出 `NotImplementedError: ConvertPirAttribute2RuntimeAttribute`，无法绕过，彻底移除。

### 新架构：Vision LLM + 分析 LLM 双模型

```
截图(PNG/JPG)
    │
    ▼ Step 1: Vision LLM (qwen3.5-plus)
  full_text（全量文字照录）
    │
    ▼ Step 2: Pillow 背景色采样
  background（白色/黑色/其他）
    │
    ▼ Step 3: 分析 LLM (qwen3-max-2026-01-23)
  {type, key_items, tips}
    │
    ▼ Step 4: 组装 v2 desc.txt
  BACKGROUND/TYPE/FULL_TEXT/KEY/TIPS
```

## v2 desc.txt 格式说明

```
BACKGROUND: 白色|黑色|其他
TYPE: 任务截图|日志截图|配置截图|其他截图
FULL_TEXT:
- 照录的文字行1
KEY:
- 关键条目，时间戳
TIPS:
- LLM 分析建议
```

- `FULL_TEXT` 为 Vision LLM 无损照录，不过滤
- `KEY` 为分析 LLM 提取的关键操作/错误条目
- `TIPS` 为分析 LLM 给出的故障诊断建议
- 超时/空文字时降级：TYPE=其他截图，FULL_TEXT=（无文字），KEY/TIPS=无

## 前端展示逻辑

- `任务截图`：FULL_TEXT 尝试解析为"状态/名称/开始/结束"四列表格，KEY 列表显示，TIPS 蓝色建议框
- `日志截图`：FULL_TEXT 以等宽代码块显示
- `配置截图`：FULL_TEXT 以代码块显示
- `其他截图`：FULL_TEXT 原文显示，无 KEY/TIPS

## 降级与容错

| 场景 | 处理方式 |
|------|---------|
| Vision LLM 超时 | 降级为 TYPE=其他截图，full_text 为空 |
| Vision LLM 返回空文字 | 警告日志 + 默认结果 |
| 速率限制 429 | openai client 自动退避重试（最多 2 次） |
| 图片无法解码 | 跳过，记录日志 |
