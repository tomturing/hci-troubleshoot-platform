# 终端操作接口与交互优化方案

**日期**：2026-04-25  
**关联需求**：`docs/requirement/events/2026-04-25-终端操作接口与交互优化需求.md`

## 修复方案

### R1 — api-gateway DATABASE_URL 注入缺失

**根因**：`backend/api-gateway/app/services/terminal.py` 的 `create_operation` / `list_operations` 使用 `self.db.async_session_factory()`，但 `deploy/helm/hci-platform/templates/api-gateway/deployment.yaml` 的 `env:` 块缺少 `DATABASE_URL` 注入，导致 `DatabaseManager` 初始化时连接串为默认的 Docker 内部地址（K8s 不可达）而 500。

**修复**：在 `api-gateway` Deployment `env:` 块中添加与其他服务相同的 `DATABASE_URL` Helm 渲染模板（使用 `urlquery` 对密码做 URL 编码，对应 PIT-022）。

### R2 — 集群信息"未知"字段隐藏

**根因**：`EnvironmentSummary.vue` 仅用 `v-if="value"` 判断，`"未知"` 是非空字符串，始终展示。

**修复**：改为 `v-if="value && value !== '未知'"` 双条件，同时在所有字段均为"未知"时渲染灰色斜体提示文本"集群数据暂未采集"。折叠摘要行同步优化，不再硬拼"未知"字符串。

### R3 — SSH localStorage 填充逻辑修复

**根因**：`SshFormSection.vue` 中 `if (!localForm.value.port && config.port)` 的 `!port` 在默认值 `'22'` 时为 `false`，导致 port 和 username 的 truthy 初始值阻止了 saved 值写入。

**修复**：去掉 `!localForm.value.xxx` 前置判断，改为直接赋值（有 saved 值就覆盖）。安全无副作用：该函数仅在 `onMounted` 等时机调用一次。

### R4 — 圆圈数字选项渲染为可点击按钮

**设计**：
- 识别条件：消息中存在 ≥2 个 `①②③` 开头的行，且包含 `请回复|请选择|请确认|请告知` 等交互引导语。
- 按钮渲染在消息气泡底部，不破坏原有 Markdown 内容区域。
- 点击后调用 `chatStore.sendMessage(choice.label)`（发送序号如"①"），与流式响应共存（`isStreaming` 时不渲染按钮）。
- 已选状态：点击后按钮高亮（`primary`）并 disabled，防止重复发送。
