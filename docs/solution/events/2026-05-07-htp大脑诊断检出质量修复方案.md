# htp 大脑诊断检出质量修复方案

> 关联需求：[需求文档](../../requirement/events/2026-05-07-htp大脑诊断检出质量修复需求.md)  
> ops-agent 参考实现：`ops-agent/feature-hci` 分支

## 一、P0 修复方案

### N-1：中文 Jaccard 改为 bigram 分词

**问题**：`str.split()` 按空格分词，中文无空格导致永远只有 1 个 token，相似度永远 0.

**方案**：改用字符级 bigram — 将字符串拆成连续两字符的集合，天然支持中英文混合。

```python
# 修复前
sa, sb = set(a.lower().split()), set(b.lower().split())

# 修复后
def _tokenize(s: str) -> set[str]:
    s = s.lower().strip()
    if len(s) <= 1:
        return set(s)
    return {s[i:i+2] for i in range(len(s) - 1)}  # bigram 字符级分词
```

> **参考**：业界对中文短文本相似度常用 bigram/trigram，n-gram 比 Jaccard+词袋 在短文本上效果更稳定。

### N-3：env_context 传递修复

**问题**：`context_info` 在步骤 2.6 已获取，但步骤 5 调用 `brain_router.process()` 时硬编码 `env_context=None`。

**方案**：将 `context_info` 提前构建（不再区分阶段），统一传给 BrainRouter。htp 大脑的 `HTPBrainAdapter` 会忽略（已在 system_prompt 中），ops-agent 的 `OpsAgentBrainAdapter` 会通过 `hci_context` 传递。

```python
# 修复：直接传入 context_info
async for brain_event in self._brain_router.process(
    ...
    env_context=context_info,   # 原来是 None
)
```

### D-3：ReactExecutor 阶段同步

**问题**：`send_message_react_stream` 调用 `_build_system_prompt(content, case_id)` 未传 `diagnostic_stage`，默认使用 S0 Prompt。

**方案**：读取当前 `diagnostic_stage` 后传入。

```python
# 修复：获取当前阶段后传入
current_stage = await self._get_current_stage(conversation_id)
system_prompt, _audit_meta = await self._build_system_prompt(
    content, case_id, diagnostic_stage=current_stage
)
```

## 二、P1 修复方案

### N-2：KB 检索分类锁定（S1+ 跳过 classify_intent）

**问题**：`KnowledgeRetriever.retrieve()` 每次都用当前消息调用 `classify_intent`，忽略 S0 已确认的分类。

**方案**：`retrieve()` 增加 `confirmed_category_code` 可选参数，非空时直接用该 code 进行 `route_by_category`，跳过意图识别。

```python
# knowledge_retriever.py
async def retrieve(
    self,
    query: str,
    case_id: str,
    diagnostic_stage: str = "S0",
    confirmed_category_code: str | None = None,  # 新增：S0 确认的分类编码
) -> tuple[str, dict]:
    ...
    # S1+ 且有确认分类时，直接走三轨路由，跳过 classify_intent
    if confirmed_category_code and diagnostic_stage != "S0":
        category_code = confirmed_category_code
        category_score = 1.0  # 用户确认，置信度视为 1.0
    else:
        intent_result = await self._kb_client.classify_intent(query, top_n=3)
        ...
```

调用侧 `_build_system_prompt` 从 `conversation.category_id` 读取：

```python
# conversation_service.py
# 从 conversation 读取 S0 确认的分类（如有）
_confirmed_category: str | None = None
if _conv and getattr(_conv, "category_id", None):
    _confirmed_category = _conv.category_id

return await self._knowledge_retriever.retrieve(
    query=query,
    case_id=case_id,
    diagnostic_stage=diagnostic_stage,
    confirmed_category_code=_confirmed_category,  # 新增传入
)
```

### N-4：ops-agent 路径状态机隔离

**问题**：BrainRouter 路径结束后，代码对 `full_reply`（来自 ops-agent 的输出）继续执行 `detect_stage_transition_with_category()`，可能误触发 htp 阶段推进。

**方案**：判断当前路由到的是否为 ops-agent，若是则跳过后验检测。

```python
# 判断是否走了 ops-agent 路径
_is_ops_agent_path = (
    self._brain_router is not None
    and resolved_assistant_type == "ops-agent"
)

if full_reply and not _is_ops_agent_path:
    # 只有 htp 大脑路径才做阶段转换检测
    new_stage, category_info = self._conversation_manager.detect_stage_transition_with_category(...)
```

## 三、P2 方案：S3 acli 命令自动提取执行

**目标**：当 LLM（htp 大脑，S3 阶段）的回复中包含 `acli ...` 命令建议时，自动提取并执行，将结果追加到回复流。

**设计**：

```python
# conversation_service.py：S3 阶段后处理
if current_stage == "S3" and full_reply and self.tool_router:
    acli_cmds = _extract_acli_commands(full_reply)
    for cmd in acli_cmds:
        risk = _classify_acli_risk(cmd)  # 只读 get/list → risk=1；其余 → 确认
        if risk >= 2 and self.confirm_service:
            # 通过 confirm_service 等待用户确认（已有机制）
            ...
        else:
            result = await self.tool_router.execute("acli_run", {"command": cmd})
            yield f"\n\n**[自动执行] `{cmd}`**\n```\n{result}\n```\n"
```

**命令提取正则**：

```python
_ACLI_CMD_PATTERN = re.compile(
    r"```(?:bash|shell)?\s*(acli\s+[^\n]+)\s*```|`(acli\s+[^`]+)`",
    re.MULTILINE
)
```

**风险分类**：

```python
def _classify_acli_risk(cmd: str) -> int:
    """1=只读自动执行, 2=写操作需确认"""
    _readonly_verbs = {"get", "list", "show", "describe", "status", "info", "check"}
    parts = cmd.strip().split()
    # acli <resource> <verb> ... 或 acli <verb> ...
    verb = parts[2] if len(parts) > 2 else (parts[1] if len(parts) > 1 else "")
    return 1 if verb.lower() in _readonly_verbs else 2
```

## 四、P3 稳定性修复

### N-5：原子计数器（PostgreSQL jsonb_set）

用 PostgreSQL 原生操作避免 read-modify-write 竞态：

```python
await session.execute(
    sa_text("""
        UPDATE conversation
        SET metadata_ = jsonb_set(
            COALESCE(metadata_, '{}'),
            '{s0_candidate_rounds}',
            (COALESCE((COALESCE(metadata_, '{}')->>'s0_candidate_rounds')::int, 0) + 1)::text::jsonb
        )
        WHERE conversation_id = :cid
    """),
    {"cid": str(conversation_id)}
)
```

### N-6：httpx 客户端生命周期管理

在 `OpsAgentBrainAdapter` 增加 `aclose()` 方法，在 FastAPI lifespan `shutdown` 阶段调用：

```python
# ops_agent_brain_adapter.py
async def aclose(self) -> None:
    """关闭底层 httpx 连接池，应在应用 shutdown 时调用。"""
    await self._client.aclose()
```

```python
# main.py / dependencies.py — FastAPI lifespan
@asynccontextmanager
async def lifespan(app: FastAPI):
    yield
    if ops_agent_adapter:
        await ops_agent_adapter.aclose()
```
