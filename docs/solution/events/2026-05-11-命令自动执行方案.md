# 命令自动执行方案

> 详细设计见 [docs/solution/custom-ui/命令自动执行设计.md](../custom-ui/命令自动执行设计.md)

## 背景与需求

关联需求：[docs/requirement/events/2026-05-11-命令自动执行需求.md](../../requirement/events/2026-05-11-命令自动执行需求.md)

当前命令执行有两个手动授权点，连续诊断场景中摩擦成本高。目标：新增自动执行模式，AI 命令自动通过 SSH bridge 执行，结果回填对话上下文。

## 方案概述

### 核心状态机

```
autoExecuteMode: 'off' | 'safe-only' | 'aggressive'
```

`danger` 级命令在任何模式下均强制手动确认。

### 风险决策矩阵

| riskLevel | off | safe-only | aggressive |
|-----------|-----|-----------|------------|
| none/readonly | 手动 | **自动** | **自动** |
| caution | 手动 | 拦截（手动确认）| **自动** |
| danger | 手动 | 强制确认弹框 | 强制确认弹框 |

### 执行流程

```
AI 输出命令块（第一条）
    → 前提条件检查（SSH连接/串行锁/工单有效）
    → 风险矩阵决策
    → 3s取消倒计时
    → 获取串行锁
    → buildBridgeCommandPayload(cmd, marker) → sshWebSocket.send()
    → 等待 parseBridgeCommandResult(marker)
    → 输出截断（>200行）
    → 追加 role:user 消息 → streamAIResponse（AI继续分析）
    → 释放串行锁
```

### 关键决策

**Q1 结果回填**：B — 自动注入对话 + 触发 AI 继续分析（与 Copilot Agent `tool_result` 一致）

**Q2 多命令顺序**：严格串行（业界共识：Claude Code / Copilot Agent 均选串行）；一次 AI 回复仅自动执行第一条命令块，AI 看到结果后决定后续步骤

**Q3 caution处理**：safe-only 模式下拦截（更保守，生产环境优先安全）

## 安全设计

- 默认关闭，需主动开启
- danger 级强制拦截（不可绕过）
- 3s 软取消窗口
- 单会话 10 条上限 + 连续 3 次失败熔断
- 命令队列单消费者（Phase 2）
- 页面后台暂停（Phase 2）
