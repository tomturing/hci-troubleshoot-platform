# 助手选择器可用性判断 Bug 修复 + ops-agent 注册

## 问题背景

用户反馈 custom-ui 的助手选择器只显示直连模式模型，看不到 productionclaw、openclaw 和 ops-agent。

经第一性原理分析，发现 4 个问题：

| 问题 | 根因 | 影响 |
|------|------|------|
| BUG-1 | `scheduler_service.py` 使用 `stats.get("idle_count")`，但 `pod_pool.py` 返回 `"idle"` | Pod 模式助手永远显示为不可用 |
| BUG-2 | ops-agent 未注册到 scheduler-service 的 ASSISTANT_REGISTRY_JSON | 前端选择器不显示 ops-agent |
| BUG-3 | conversation-service 和 scheduler-service 的 assistant_registry 配置源不同 | 配置不一致 |
| BUG-4 | 直连模式判断未检查 base_url | 可能误判 |

## 修复方案

### 修复 1：Key 名称不匹配

**文件**：`backend/scheduler-service/app/services/scheduler_service.py`

**修改**：
```python
# 修复前（错误）
available = is_direct_mode or stats.get("idle_count", 0) > 0

# 修复后
available = is_direct_mode or stats.get("idle", 0) > 0
```

### 修复 2：ops-agent 注册

**文件**：`deploy/helm/hci-platform/templates/configmap.yaml`

**修改**：在 `$defaultAssistantRegistry` 中添加 ops-agent：
```json
"ops-agent": {
  "warm_pool_size": 0,
  "max_pool_size": 0,
  "base_url": "http://ops-agent-service:8006",
  "display_name": "Ops-Agent (GLM-5)",
  "description": "基于 SOP 知识库的智能排障助手",
  "enabled": true,
  "is_default": false
}
```

**设计决策**：
- ops-agent 使用直连模式（warm_pool_size=0）
- 因为 ops-agent 是独立 Deployment，不需要 scheduler 启动 Pod
- base_url 指向 ops-agent-service ClusterIP

### 修复 3：配置统一

**文件**：`backend/conversation-service/app/config.py`

**修改**：assistant_registry 属性优先从 ASSISTANT_REGISTRY_JSON 环境变量读取，与 scheduler-service 配置统一。

```python
# 优先使用环境变量注入的统一配置（来自 hci-common-config ASSISTANT_REGISTRY_JSON）
try:
    registry = json.loads(self.ASSISTANT_REGISTRY_JSON or "{}")
    if isinstance(registry, dict) and registry:
        # 验证至少有一个有效的助手配置
        for _atype, cfg in registry.items():
            if isinstance(cfg, dict) and cfg.get("enabled", True):
                return registry  # 使用统一配置
except json.JSONDecodeError:
    pass

# 降级：默认只有 openclaw（向后兼容）
```

### 修复 4：直连模式判断改进

**文件**：`backend/scheduler-service/app/services/scheduler_service.py`

**修改**：
```python
# 修复前（不完整）
is_direct_mode = config.get("warm_pool_size", 0) == 0 and config.get("max_pool_size", 0) == 0

# 修复后（增加 base_url 检查）
is_direct_mode = (
    config.get("warm_pool_size", 0) == 0
    and config.get("max_pool_size", 0) == 0
    and bool(config.get("base_url"))  # 必须有 API 地址才算直连模式
)
```

## 配置唯一性确认

修复后，助手配置统一为**单一来源**：

| 配置源 | 文件路径 | 作用 |
|--------|----------|------|
| **唯一权威** | `deploy/helm/hci-platform/templates/configmap.yaml` | 定义 ASSISTANT_REGISTRY_JSON |
| 注入到 | scheduler-service | envFrom: hci-common-config |
| 注入到 | conversation-service | envFrom: hci-common-config |

两个服务使用同一个 ConfigMap，配置统一。

## 影响范围

- scheduler-service：修复可用性判断 + ops-agent 注册
- conversation-service：改用统一配置源
- Helm configmap.yaml：添加 ops-agent 到默认注册表
