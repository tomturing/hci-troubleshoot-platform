### 关键技术点

#### 1. DESCRIPTION 段落解析问题
**根因**：`parseContentMd` 检测截图块结束时，把 DESCRIPTION 后的纯文字行误判为"截图块结束"。

**修复**：
```typescript
// 新增 inDescription 状态跟踪
let inDescription = false

// 检测进入 DESCRIPTION section
if (/^>\s*DESCRIPTION:$/.test(trimmed)) {
  inDescription = true
}

// DESCRIPTION section 结束：遇到下一个 section 关键词
if (inDescription && /^>\s*(BACKGROUND|TYPE|FULL_TEXT|KEY|TIPS):/.test(trimmed)) {
  inDescription = false
}

// v3: DESCRIPTION section 内的纯文字行属于截图块
const isDescriptionText = inDescription && !isBlank && !isFieldLine && !isBulletLine
```

#### 2. 图片 URL 提取公共函数
**设计原则**：
- 单次遍历 `<img>` 标签，同时产出 URL 和位置信息
- 使用原始 src 字符串定位（而非 abs_url），避免相对路径/query/文件名重复问题
- `extract_image_urls()` 是 `extract_image_urls_with_positions()` 的薄封装

**核心代码**：
```python
def extract_image_urls_with_positions(html: str, base_url: str) -> list[tuple[str, int]]:
    soup = BeautifulSoup(html, "lxml")
    seen: set[str] = set()
    result: list[tuple[str, int]] = []

    for img in soup.find_all("img"):
        src = img.get("src") or img.get("data-src") or ""
        if not src or src.startswith("data:"):
            continue

        abs_url = urljoin(base_url, src)
        if abs_url in seen:
            continue
        seen.add(abs_url)

        # 使用原始 src 定位（而非 abs_url）
        escaped_src = re.escape(src)
        m = re.search(escaped_src, html)
        pos = m.start() if m else 0
        result.append((abs_url, pos))

    return result

def extract_image_urls(html: str, base_url: str) -> list[str]:
    return [url for url, _pos in extract_image_urls_with_positions(html, base_url)]
```

## 决策依据 (WHY)

### 方案选择：公共模块位置
| 方案 | 优点 | 缺点 | 评分 |
|------|------|------|------|
| 方案A：新建 `html_utils.py`（选中）| 单一权威来源，职责清晰 | 需要两处导入修改 | ★★★★☆ |
| 方案B：放在 `fetcher.py` | 减少文件 | image_proc 需跨模块导入，职责模糊 | ★★★☆☆ |
| 方案C：放在 `image_proc.py` | 同上 | fetcher 需跨模块导入 | ★★☆☆☆ |

### 为什么选择方案A？
- 符合"单一职责原则"，图片 URL 提取是独立功能
- 两处调用地位平等，不应依附于任一侧
- 便于后续其他模块复用（如 converter）

### 为什么使用原始 src 定位？
- 相对路径在 HTML 中不包含 base_url，搜索 abs_url 会失败
- 带 query 的 src（如 `/img.png?v=123`）在 abs_url 中可能被规范化
- 文件名重复（`/dir1/img.png` vs `/dir2/img.png`）时搜索文件名会匹配错误位置

### 权衡与妥协
- 保留 `fetcher._extract_image_urls()` 向后兼容封装，避免破坏现有测试
