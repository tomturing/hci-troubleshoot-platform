# ops-agent 刷新后上下文丢失 — 根因分析与修复方案

> 关联需求：[docs/requirement/events/2026-05-13-ops-agent刷新后上下文丢失修复需求.md](../../requirement/events/2026-05-13-ops-agent刷新后上下文丢失修复需求.md)

---

## 1. 根因分析（第一性原理）

### 1.1 内存态 vs 磁盘态的本质矛盾

ops-agent 的 `ACPServer` 将所有会话状态（`ACPSession`）存储在进程内存字典 `_sessions` 中。
每个 `ACPSession.llm_message_history` 包含完整的 LLM 对话上下文（system prompt + 多轮 user/assistant 消息）。

```
进程重启前（内存）:
  _sessions = {
    "conv-uuid-123": ACPSession(
      llm_message_history=[...42条消息，包含完整诊断上下文...]
    )
  }

进程重启后（内存清空）:
  _sessions = {}
```

当 hci-platform 在重启后调用 `POST /acp/sessions`（`_ensure_acp_session`）重建 session 时，
`session_new` 执行：

```python
llm_message_history = copy.deepcopy(
    load_latest_message_history(self._trajectory_file) or []
)
```

而 REST server 的 `ACPServer(...)` **没有传入 `trajectory_file`**：

```python
# ops_agent/server/main.py（修复前）
acp_server = ACPServer(
    config_file=settings.OPS_CONFIG_PATH,
    auto_approve=settings.OPS_AGENT_AUTO_APPROVE,
    debug_llm=settings.OPS_AGENT_DEBUG_LLM,
    # ❌ 无 trajectory_file → self._trajectory_file = None
)
```

`load_latest_message_history(None)` 返回 `None`，fallback 为 `[]`。
**结果：session 永远以空历史初始化，ops-agent 永远从步骤 1 开始。**

---

### 1.2 ops-web 为什么不受影响

ops-web（Streamlit 界面）在创建 `ACPServer` 时传入了 per-chat trajectory 路径：

```python
# ops_web/streamlit_app.py（简化）
server = ACPServer(
    config_file=...,
    trajectory_file=f"~/.ops_agent/chats/{chat_id}/",  # ← per-chat 目录
)
```

每轮 `session_prompt` 完成后，`trajectory_recorder` 将 LLM 消息历史写入：

```
~/.ops_agent/chats/{chat_id}/original_trajectory__*.json
```

重建 session 时，`load_latest_message_history` 从该目录读取最新文件，历史完整恢复。

**对比差异**：

| 维度 | ops-web | hci REST server（修复前） |
|------|---------|--------------------------|
| `ACPServer` 构造 | `trajectory_file=per_chat_dir` ✅ | 无 `trajectory_file` ❌ |
| `session_new` 加载历史 | `load_latest_message_history(traj_dir)` ✅ | `load_latest_message_history(None) → []` ❌ |
| `_create_agent` trajectory 路径 | `self._trajectory_file`（已设置） ✅ | `self._trajectory_file = None` ❌ |
| 进程重启后历史恢复 | ✅ 完整恢复 | ❌ 从空历史开始 |
| per-session 隔离 | ✅（不同 chat_id 目录不同） | N/A |

---

### 1.3 为什么必须改 ops-agent（不能只改 hci-platform）

| 需要解决的问题 | 能否在 hci-platform 侧解决 |
|----------------|--------------------------|
| `session_new` 从磁盘读取 trajectory 文件 | ❌ 逻辑完全在 ops-agent 内部 |
| `Agent` 初始化时绑定 trajectory 持久化路径 | ❌ `trajectory_recorder` 在 Agent 构造时注入，hci-platform 无法干预 |
| per-session trajectory 目录的选择与使用 | ❌ 路径决策和 I/O 都在 ops-agent 内部 |
| 向 ops-agent 传递 `trajectory_dir` 参数 | ✅ 通过 `POST /acp/sessions` 请求体传入 |

**结论**：hci-platform 唯一能做的是在 `POST /acp/sessions` 时传入 `trajectory_dir` 字段。
但这要求 ops-agent 的 `session_new` 接口支持 per-session `trajectory_dir`——**当前不支持**，必须在 ops-agent 侧实现。

---

## 2. 技术方案

### 2.1 方案概述

**核心思路**：在 `session_new` 接口增加 `trajectory_dir` 可选参数，
允许调用方为每个 session 指定独立的 trajectory 目录。
hci-platform 在调用 `POST /acp/sessions` 时传入 `trajectory_dir=/data/ops-agent-trajectories/{conversation_id}`。

这样每个 hci conversation 对应一个独立的 trajectory 目录，既实现了跨进程重启的历史恢复，也保证了多租户隔离。

### 2.2 方案比较

| 方案 | 描述 | 优点 | 缺点 | 评分 |
|------|------|------|------|------|
| **A（选中）per-session trajectory_dir** | `session_new` 新增参数，调用方指定目录 | 最小改动；per-session 隔离；向后兼容 | 需要 ops-agent + hci-platform 两端配合 | ★★★★★ |
| B 全局 trajectory_file 配置 | 在 `main.py` 给 `ACPServer` 传全局路径 | 一行改动 | 所有 session 共享同一目录，加载时会读错 session 的历史 | ★★☆☆☆ |
| C 在 hci-platform 侧回放历史 | hci-platform 在每次 prompt 时把完整历史作为 messages 传入 | 不改 ops-agent | ops-agent 侧已有多轮 context，会重复注入；消耗大量 token | ★★☆☆☆ |

**选 A 的理由**：
- ops-web 已有 per-chat trajectory 机制，方案 A 是将该机制扩展到 REST 模式，设计一致
- 最小侵入：不改变 `ACPServer.__init__` 签名，不影响 ops-web 路径
- per-session 目录自然隔离，不存在跨 session 污染风险

### 2.3 完整数据流（修复后）

```
hci-platform: _ensure_acp_session("conv-uuid-123")
  │
  └─► POST /acp/sessions
      body: {
        "session_id": "conv-uuid-123",
        "trajectory_dir": "/data/ops-agent-trajectories/conv-uuid-123"
      }
          │
          ▼ ops-agent: ACPServer.session_new(trajectory_dir=...)
              traj_source = "/data/ops-agent-trajectories/conv-uuid-123"
              history = load_latest_message_history(traj_source)
              # → 若文件存在：[...42条消息...]
              # → 若文件不存在（新会话）：[]
              ACPSession(
                llm_message_history=[...42条...],  ← 历史已恢复
                trajectory_dir="/data/.../conv-uuid-123"
              )
          │
          ▼ ops-agent: _create_agent(session=session)
              traj_path = session.trajectory_dir  # "/data/.../conv-uuid-123"
              Agent("ops_agent", config, traj_path, console, ...)
              # trajectory_recorder 绑定到 traj_path → 每轮完成后自动写入
```

## 3. 改动清单（最小 16 行）

### 3.1 `ops-agent/ops_agent/acp/server.py`（+12 行）

**改动1**：`ACPSession` 新增 `trajectory_dir` 字段

```python
# 修复前
llm_message_history: list[Any] = field(default_factory=list)
lock: threading.Lock = field(default_factory=threading.Lock)

# 修复后
llm_message_history: list[Any] = field(default_factory=list)
trajectory_dir: str | None = None  # per-session trajectory 目录（覆盖全局 _trajectory_file）
lock: threading.Lock = field(default_factory=threading.Lock)
```

**改动2**：`session_new` 接受 `trajectory_dir` 参数

```python
# 修复后
def session_new(self, *, ..., trajectory_dir: str | None = None) -> JsonDict:
    traj_source = trajectory_dir or self._trajectory_file
    session = ACPSession(
        ...,
        llm_message_history=copy.deepcopy(load_latest_message_history(traj_source) or []),
        trajectory_dir=trajectory_dir,  # 存下来给 _create_agent 用
    )
```

**改动3**：`_create_agent` 优先使用 per-session `trajectory_dir`

```python
# 修复后
traj_path = (session.trajectory_dir if session is not None else None) or self._trajectory_file
return Agent("ops_agent", config, traj_path, console, **agent_kwargs)
```

### 3.2 `ops-agent/ops_agent/server/acp_routes.py`（+2 行）

`POST /acp/sessions` 路由透传 `trajectory_dir` 请求体字段：

```python
# 修复后
result = server.session_new(
    cwd=body.get("cwd"),
    session_id=body.get("session_id"),
    trajectory_dir=body.get("trajectory_dir"),  # 新增
)
```

### 3.3 `hci-platform/backend/conversation-service/app/adapters/ops_agent_brain_adapter.py`（+7 行）

`__init__` 新增 `_trajectory_base_dir`；`_ensure_acp_session` 传入 per-conversation `trajectory_dir`：

```python
# __init__ 新增
self._trajectory_base_dir = os.environ.get(
    "OPS_AGENT_TRAJECTORY_DIR", "/data/ops-agent-trajectories"
)

# _ensure_acp_session 修复后
trajectory_dir = f"{self._trajectory_base_dir}/{session_id}"
payload = {"session_id": session_id, "trajectory_dir": trajectory_dir}
```

## 4. 部署要求

### 4.1 必须：为 ops-agent 挂载持久化存储

ops-agent pod 需要将 `/data/ops-agent-trajectories` 挂载到 **PVC**，
否则 pod 重调度后 trajectory 文件丢失，效果等同于未修复。

```yaml
# Helm values 示例（需在 deploy/ 目录配置）
volumes:
  - name: trajectory-storage
    persistentVolumeClaim:
      claimName: ops-agent-trajectories-pvc
volumeMounts:
  - name: trajectory-storage
    mountPath: /data/ops-agent-trajectories
```

dev 环境可使用 `hostPath` 或 `emptyDir`（pod 重启后仍会丢失，但进程重启不影响）。

### 4.2 可选：环境变量覆盖

```yaml
env:
  - name: OPS_AGENT_TRAJECTORY_DIR
    value: /data/ops-agent-trajectories  # 默认值，可覆盖
```
