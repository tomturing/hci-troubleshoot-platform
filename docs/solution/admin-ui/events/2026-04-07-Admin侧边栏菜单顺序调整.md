## 背景与需求

Admin 管理台侧边栏菜单当前顺序不符合业务优先级逻辑，需要重新调整以提高运维效率和用户体验。

### 当前顺序

1. 仪表盘
2. 工单管理
3. 用户管理
4. 系统监控
5. 知识审核
6. KBD 审核
7. 分类基线

### 问题分析

- "分类基线"是诊断流程的核心配置，应优先展示
- "用户管理"和"系统监控"是低频操作，应放在后面
- 知识相关功能（知识审核、KBD审核）应相邻放置

---

## 方案（WHAT）

### 新顺序

```
1. 仪表盘      ← 高频入口
2. 工单管理    ← 核心业务
3. 分类基线    ← 核心配置（前置）
4. 知识审核    ← 知识管理
5. KBD 审核    ← 知识管理
6. 用户管理    ← 低频运维
7. 系统监控    ← 低频运维
```

### 实现方式

修改 `frontend/admin/src/router/index.ts` 中路由定义的顺序，侧边栏组件会自动按照路由顺序渲染菜单项。

---

## 决策依据（WHY）

### 为什么选择此顺序

1. **业务优先级驱动**：高频操作前置，低频操作后置
2. **功能分组清晰**：
   - 第一组：监控入口（仪表盘）
   - 第二组：核心业务（工单管理、分类基线）
   - 第三组：知识管理（知识审核、KBD审核）
   - 第四组：系统运维（用户管理、系统监控）
3. **最小改动原则**：仅调整路由顺序，无需修改组件代码

### 为什么不选其他方案

| 备选方案 | 优点 | 缺点 | 结论 |
|---------|------|------|------|
| 按拼音排序 | 简单机械 | 无业务逻辑，效率低 | ❌ 不采用 |
| 按使用频率动态排序 | 个性化体验 | 实现复杂，需要埋点统计 | ❌ 过度设计 |
| 可配置顺序 | 灵活性高 | 增加配置复杂度，当前无此需求 | ❌ 过度设计 |
