## 一、项目概述

**SOP Agent** 是一个面向运维历史案例的离线 Python 流水线，用于将 Excel 案例表沉淀为结构化、可复用的节点级排障 SOP。

**核心价值**：
- 将非结构化的运维案例文档转化为标准化、可执行的排障流程
- 支持多场景（HCI 云计算排障、AF 等）的差异化处理
- 提供 HTML/Word 等多格式渲染输出

**技术特点**：
- 全异步并发处理，支持大规模案例处理
- 断点续跑机制，中断后可无缝恢复
- LLM 驱动的智能清洗、分类、聚合
- 强本地校验确保产物可执行

### 节点与案例的对应关系（1:N）

节点与案例是 **一对多** 关系：一个节点对应多个案例。

| 数据结构 | 数量 | 说明 |
|---------|------|------|
| selected_nodes.csv | N 个节点 | 选中的 SOP 聚合节点 |
| case_assignments.csv | M 个案例 | 每个案例归属到唯一节点 |
| case_facts.jsonl | M 行 | 每个案例一行事实卡 |
| node_sops.jsonl | N 行 | 每个节点一行 SOP |

**关系链**：

```
节点 : 案例 = 1 : N  （一个节点对应多个案例）
案例 : 事实卡 = 1 : 1 （一个案例一张事实卡）
节点 : SOP = 1 : 1   （一个节点一个 SOP）
```

**可追溯性**：节点 SOP 里的 `source_case_ids` 可以反查回单 case 结果。

## 四、核心设计模式

### 1. 异步并发 + 限流重试

所有脚本统一采用此模式，确保大规模数据处理时的稳定性和效率。

```python
# 统一配置参数
MAX_CONCURRENCY = 40-500  # 并发上限
MAX_RETRIES = 5-10        # 最大重试次数
RETRY_BASE_DELAY = 2-3    # 指数退避基础延迟

# Worker Queue 模式实现
semaphore = asyncio.Semaphore(worker_count)
row_queue = asyncio.Queue()
for row_data in pending_rows:
    row_queue.put_nowait(row_data)

workers = [
    asyncio.create_task(worker_loop(queue, client, ...))
    for _ in range(worker_count)
]
await row_queue.join()
```

**关键点**：
- 使用信号量控制并发数，避免 API 过载
- 指数退避重试：`delay = BASE_DELAY * (2 ** (attempt - 1))`
- Worker 数量可配置，适应不同负载场景

### 2. 断点续跑机制

所有脚本扫描已有结果列/文件，自动跳过已完成行。

**Excel 断点续跑**：
```python
def scan_rows(ws, col_source: int, col_result: int):
    pending = []
    skipped = 0
    for row_idx in range(2, ws.max_row + 1):
        source = ws.cell(row=row_idx, column=col_source).value
        result = ws.cell(row=row_idx, column=col_result).value
        if result and str(result).strip() != "":
            skipped += 1  # 已完成，跳过
            continue
        pending.append((row_idx, source))
    return pending, skipped
```

**JSONL 断点续跑**：
```python
def load_processed_ids(output_path: Path) -> set[str]:
```

**原子写盘**：

## 六、核心脚本详解

### clean.py - 案例清洗

**功能**：将原始案例的"合并"列清洗为规范 Markdown。

**核心流程**：
1. 加载场景对应的清洗提示词
2. 打开 Excel，定位/创建"合并-clean"列
3. 扫描待处理行（跳过已有结果）
4. 异步并发调用 LLM API
5. 从响应中提取 `<CLEANED_CONTENT>` 标签内的内容
6. 原子写盘，按 `SAVE_INTERVAL` 批量保存

**提示词模板结构**：
```
<RAW_CONTENT>
{{ content }}
</RAW_CONTENT>

期望输出：
<CLEANED_CONTENT>
清洗后的 Markdown 内容
</CLEANED_CONTENT>
```

### rewrite.py - 视觉重写

**功能**：结合图片信息重写案例内容，生成更稳定的正文。

**特殊处理**：
- 图片 URL 白名单过滤（`ACCESSIBLE_IMAGE_DOMAINS`）
- 图片下载 + base64 编码
- curl fallback（绕过 Python TLS 兼容性问题）
- 多模态消息构建（文本 + 图片）

**图片处理流程**：

```
1. 图片 URL 提取与分类
   ├─ 提取 ![截图](URL) 格式的图片链接
   ├─ URL 有效性校验（排除含中文、无图片后缀的脏数据）
   └─ 域名白名单过滤

2. 图片下载策略
   ├─ Python aiohttp 下载（默认）
   ├─ curl fallback（特定域名，绕过 TLS 兼容性问题）
   └─ 指数退避重试

3. 多模态消息构建
   ├─ 文本部分：含 [IMAGE_N] 占位符的源内容
   ├─ 图片部分：base64 编码的 data URI
   └─ 失败图片：提示"无法加载，请仅基于正文改写"

4. 视觉模型调用
   ├─ OpenAI 兼容格式多模态 API
   └─ 从 <REWRITTEN_CONTENT> 标签提取结果
```

**行分类逻辑**：
- **无图片**：直接复制 `合并-clean` 内容到 `合并-text-rewrite`
- **含不可访问图片**：跳过，写入 `[跳过: 含不可访问的图片URL]` 标记
- **仅有可访问图片**：进入重写流程

**curl Fallback**：某些站点（如 `support.sangfor.com.cn`）与 Python TLS 栈存在兼容性问题，脚本会自动切换到 curl。

### classify.py - 案例分类

详细分析参见 [案例分类.md](./案例分类.md)

**分类树索引构建**：classify.py 加载 YAML 后构建三个关键索引：
- `child_options`：每个节点的合法子节点集合
- `leaf_paths`：所有叶子节点的路径集合
- `path_lookup`：从路径后缀到完整路径的映射

**路径修复机制**：模型输出的路径可能不完整或格式不规范，通过 `path_lookup` 进行修复。

### select_sop_nodes.py - 节点选择与聚类

详细分析参见 [节点选择与聚类.md](./节点选择与聚类.md)

### extract_case_facts.py - 事实卡抽取

详细分析参见 [事实提取.md](./事实提取.md)

**输出结构**：

### rewrite Prompt 设计要点

**核心目标**：将图片中的关键信息转写为纯文本，使读者即使看不到图片也能独立理解。

**红线规则**：

| 规则 | 说明 |
|------|------|
| 禁止无依据补写 | 不得根据经验、常识猜测图片中未能可靠识别的内容 |
| 禁止删除实质信息 | 原文已有的有效文字、代码、日志、命令都必须保留 |
| 禁止保留指代性废话 | "如图所示"、"见下图"等离开图片就无法理解的表达必须改写 |
| 禁止模板化占位 | 不得新增"此处省略"等原文没有的抽象占位符 |
| 禁止保留图片占位符 | 重写后的正文中不得出现 `[IMAGE_N]` 或 `![截图](URL)` |

### 聚合阶段 Prompt 设计要点

**检查步骤设计规则**：每个 check 都必须只判断一件事。

**命令保真规则**：
1. `command_line_executable=true` 时，`command_or_path` 必须是自洽的单行 bash 命令
2. 不要只写裸路径
3. 不要把关键命令只写进 action 却让 command_or_path 为空
4. 不要发明输入中不存在的命令

**分支编号规则**：按 Excel 列名风格递增（branch-A 到 branch-Z，超过 26 个用 branch-AA、branch-AB）。

**允许值枚举**：

| 字段 | 允许值 |
|------|--------|
| branch_type | runbook · info_collection · escalation |
| source_type | explicit · inferred · mixed |
| field_source_types | explicit · inferred · mixed · empty |
| exit_reason | not_match · resolved · fallback_parent · manual_review · escalate |

## 八、关键设计决策

| 决策 | 理由 |
|------|------|
| 离线流水线而非服务 | 一次性批处理场景，无需实时响应；便于人工复核中间产物 |
| Excel 作为中间产物载体 | 清洗/重写/分类结果写回原始 Excel，便于人工复核 |
| JSONL 作为结构化产物 | 便于增量追加、流式处理、断点续跑 |
| 场景化提示词分离 | HCI 和 AF 业务差异大，提示词需要针对性设计 |
| 分层模型配置 | 批量任务用快模型，关键决策用强模型，平衡成本和效果 |
| 强本地校验 | 确保产物可执行，避免生成不可用的 SOP |

## 十、使用方式

### 完整流水线

```bash
# 1. 环境准备
uv python install 3.12
uv sync
source .venv/bin/activate
cp .env.example .env.local

# 2. 按顺序执行
python scripts/clean.py -i "data/SPT源数据-HCI.xlsx" --scene hci
python scripts/rewrite.py -i "data/SPT源数据-HCI.xlsx" --scene hci
python scripts/add_image.py -i "data/SPT源数据-HCI.xlsx"
python scripts/classify.py -i "data/SPT源数据-HCI.xlsx" -t "output-hci/云计算_tree.yaml" --scene hci
python scripts/select_sop_nodes.py -i "data/SPT源数据-HCI.xlsx" -t "output-hci/云计算_tree.yaml" -o "output-hci/sop_node_selection" --scene hci
python scripts/extract_case_facts.py -i "data/SPT源数据-HCI.xlsx" -n "output-hci/sop_node_selection.selected_nodes.csv" -o "output-hci/case_facts.jsonl" --scene hci
python scripts/aggregate_node_sop.py -i "output-hci/case_facts.jsonl" -o "output-hci/node_sops.jsonl" --scene hci --save-intermediate
python scripts/render_sop_html.py -i "output-hci/node_sops.jsonl" -o "output-hci/web_sops/"
python scripts/render_sop_docx.py -i "output-hci/node_sops.jsonl" -o "output-hci/docx_sops/" --case-facts "output-hci/case_facts.jsonl"
```

### 断点续跑

所有脚本支持断点续跑：中断后重新执行相同命令，会自动跳过已完成的行/节点。

### 单节点调试

```bash
# 抽取指定 case
python scripts/extract_case_facts.py \
  -i "data/SPT源数据-HCI.xlsx" \
  -n "output-hci/sop_node_selection.selected_nodes.csv" \
  -o "output-hci/case_facts.sample.jsonl" \
  --scene hci \
  --case-ids 23068,23069

# 聚合单个节点
python scripts/aggregate_node_sop.py \
  -i "output-hci/case_facts.jsonl" \
  -o "output-hci/node_sop.vm_backup_failed.json" \
  --target-node "云计算-排障-虚拟机-可靠性管理-虚拟机备份-虚拟机备份创建失败" \
  --scene hci
```
