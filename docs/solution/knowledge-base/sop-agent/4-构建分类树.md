# SOP Agent 构建分类树分析

> 本文档详细分析 `sop-agent` 项目中构建分类树阶段的实现设计。
>
> 源代码位置：
> - `/mnt/d/aihci/sop-agent/scripts/build_hci_tree.py`
> - `/mnt/d/aihci/sop-agent/scripts/build_af_tree.py`

---

## 核心特点

- **纯算法解析，无 LLM**：两个脚本均为纯 openpyxl 解析，读取 Excel 中预定义的多级分类标签列，递归构建嵌套 JSON 树。不依赖任何模型调用，执行速度极快。
- **HCI / AF 双脚本架构**：HCI 按列位置隐含层级，AF 按分隔符拆分。
- **树结构标准化**：统一输出嵌套 JSON 树 + 人类可读的 `.txt` 文件。

## 核心问题

案例分类需要一个预定义的分类树作为路由依据。原始数据中分类标签以不同形式存在（HCI 是多列 Excel，AF 是单列含分隔符文本），需要统一转换为标准树结构。

## 核心思路

对于 HCI：从 Excel 逐行读取，每行代表一条分类路径，列 A = 一级、列 B = 二级、列 C = 三级……空单元格处截断，按行构建树。对于 AF：从指定列读取标签文本，按指定分隔符拆分为多级路径，同一路径只保留第一条记录（忽略重复）。

## 核心目标

1. 将外部分类标签表转换为标准嵌套 JSON 树结构
2. 输出机器可读的 `.yaml` 和人类可读的 `.txt` 两种格式
3. 自动去重，处理空行和空单元格
4. 为后续案例分类提供可靠的路由树

## 核心约束

1. 空单元格处截断该路径，不补全缺失层级
2. 重复路径只保留第一条出现的记录
3. 根节点名称通过参数指定（HCI 默认 `云计算-排障`，AF 默认 `AF`）
4. 不支持动态合并或重命名节点

## LLM 使用情况

**使用**：否

此阶段为纯算法解析，依赖预定义的分类标签表（Excel），不调用任何 LLM API。

## 一、概述

构建分类树阶段的目标是将外部维护的**分类树源 Excel 文件**转换为 YAML 格式的分类树配置文件，供后续 `classify.py` 使用。

**核心特点**：
- 纯数据转换，无 LLM 调用
- 通常只需一次性运行，后续由人工维护 YAML 文件
- 同时输出 YAML 和文本树两种格式

---

## 二、输入输出

| 项目 | 内容 |
|------|------|
| **输入文件** | 分类树源 Excel（外部维护，按层级分列） |
| **输出文件** | `*_tree.yaml`（YAML 分类树）+ `*_tree.txt`（文本树） |

**命令**：

```bash
python scripts/build_hci_tree.py \
  --input "<分类树源xlsx>" \
  --sheet "HCI" \
  --yaml-output "output-hci/云计算_tree.yaml" \
  --txt-output "output-hci/云计算_tree.txt"

python scripts/build_af_tree.py \
  --input "<分类树源xlsx>" \
  --sheet "AF" \
  --yaml-output "output-af/云计算_tree.yaml" \
  --txt-output "output-af/云计算_tree.txt"
```

## 三、输入格式

分类树源 Excel 按**层级分列**排列，每行代表一条叶子节点路径：

| 列 A | 列 B | 列 C | 列 D | ... |
|------|------|------|------|------|
| 云计算 | 排障 | 虚拟机 | 开机失败 | |
| 云计算 | 排障 | 虚拟机 | 网络问题 | |
| 云计算 | 排障 | 存储 | 性能异常 | |

脚本从第 2 行开始读取，遇到空单元格则停止该行的路径解析。

## 四、构建逻辑

### 4.1 节点树构建

```python
# build_hci_tree.py 第47-61行
def build_tree(xlsx_path: Path, sheet_name: str) -> Node:
    root = Node(name=sheet_name)
    for row in worksheet.iter_rows(min_row=2, values_only=True):
        path = extract_path(row)   # 提取非空单元格作为路径
        if not path:
            continue
        current = root
        for name in path:
            current = current.get_or_add_child(name)  # 逐层创建/获取子节点
    return root
```

**节点数据结构**：

```python
# build_hci_tree.py 第13-27行
@dataclass
class Node:
    name: str
    children: dict[str, "Node"] = field(default_factory=dict)

    def get_or_add_child(self, name: str) -> "Node":
        if name not in self.children:
            self.children[name] = Node(name=name)
        return self.children[name]
```

### 4.2 路径提取

```python
# build_hci_tree.py 第37-44行
def extract_path(row: tuple[Any, ...]) -> list[str]:
    path: list[str] = []
    for cell in row:
        text = normalize_cell(cell)
        if text is None:
            break    # 遇到空单元格则停止
        path.append(text)
    return path
```

## 五、与 classify.py 的关系

```
分类树源 Excel
      │
      ▼ build_hci_tree.py / build_af_tree.py（一次性）
  *_tree.yaml
      │
      ▼ classify.py（每次运行）
  案例自动分类
```

- `classify.py` 直接加载 YAML 文件，不调用构建脚本
- YAML 文件是**预生成的配置文件**，通常由 CTI 统一维护
- `build_hci_tree.py` 仅在分类树结构调整时运行一次

---

## 六、注意事项

1. **一次性运行**：`build_hci_tree.py` 不需要每次分类时都运行，YAML 生成后由人工维护
2. **叶子节点唯一性**：Excel 中同一叶子路径出现多次不会导致重复，最终只保留一个节点
3. **空行跳过**：读取时遇到第一个空单元格即停止该行，即使后续列有内容也不读取
4. **Sheet 名称**：需通过 `--sheet` 参数指定，默认值为 `HCI`
