# SOP Agent 视觉重写分析

> 本文档详细分析 `sop-agent` 项目中视觉重写阶段的实现设计。
>
> 源代码位置：`/mnt/d/aihci/sop-agent/scripts/rewrite.py`

---

## 一、概述

视觉重写阶段的目标是将清洗后的案例文本**结合图片语义**，重写为语义完整、可独立阅读的 Markdown 正文。

**输入输出**：

| 项目 | 内容 |
|------|------|
| **输入列** | `合并-clean`（清洗后 Markdown，含图片链接） |
| **输出列** | `合并-text-rewrite`（纯文本重写，融入图片信息） |

**核心特点**：
- 使用视觉大模型（支持图片输入的 LLM）进行多模态重写
- 将图片下载并编码为 base64 后发送给模型
- 图片下载失败时不阻断流程，降级为纯文本处理

---

## 二、处理流程

```
合并-clean 文本
     │
     ▼ 提取图片 URL
  分类判断 ──→ 无图片 → 直接复制
     │
     ├── 含不可访问图片 → 写入 "[跳过: 含不可访问的图片URL]"
     │
     └── 仅含可访问图片 → 需要重写
            │
            ▼ 替换图片为占位符
          [IMAGE_1] [IMAGE_2] ...
            │
            ▼ 下载图片并编码为 base64
          多模态消息构建
            │
            ▼ 调用视觉 LLM
          提取 <REWRITTEN_CONTENT>
            │
            ▼ 写入"合并-text-rewrite"列
```

**命令**：

```bash
python scripts/rewrite.py -i "data/SPT源数据-HCI.xlsx" --scene hci
python scripts/rewrite.py -i "data/SPT源数据-AF.xlsx" --scene af
```

## 六、多模态消息构建

```python
# rewrite.py 第605-650行
def build_multimodal_content(text_with_placeholders, images):
    """构建 OpenAI 兼容的多模态 content 数组。"""
    content_parts = []
    # 1. 文本部分（含 [IMAGE_N] 占位符）
    content_parts.append({
        "type": "text",
        "text": f"<SOURCE_CONTENT>\n{text_with_placeholders}\n</SOURCE_CONTENT>",
    })
    # 2. 图片部分，按编号追加
    for i, (b64, mime) in enumerate(images, 1):
        if b64 is not None:
            content_parts.append({"type": "text", "text": f"以下是 [IMAGE_{i}] 对应的截图："})
            content_parts.append({"type": "image_url", "image_url": {"url": f"data:{mime};base64,{b64}"}})
        else:
            content_parts.append({"type": "text", "text": f"[IMAGE_{i}] 的截图无法加载，请仅基于正文信息改写对应部分。"})
```

**关键设计**：
- 占位符 `[IMAGE_N]` 让模型知道图片与文本的对应关系
- 图片下载失败时不阻断流程，改用文字提示让模型基于正文信息改写
- MIME 类型根据 URL 后缀或 Content-Type 自动判断

## 七、数据行分类处理

```
扫描数据行
  ├── already_done（已完成）     → 跳过
  ├── empty_clean（源列为空）   → 跳过
  ├── no_images（无图片）       → 直接复制到 text-rewrite
  ├── has_inaccessible（含不可访问图片）→ 写入跳过标记
  └── need_rewrite（需要重写） → 调用视觉 LLM
```

## 八、容错与可靠性

| 机制 | 说明 |
|------|------|
| 断点续跑 | 检测已有输出列，跳过已完成行 |
| 指数退避重试 | 3s 基础延迟，最多 5 次重试 |
| 原子写盘 | `.tmp` + `os.replace()`，防中断损坏 |
| 图片下载重试 | 3 次，含可重试异常判断 |
| 双通道下载 | aiohttp + curl，绕过 TLS 问题 |
| 降级处理 | 图片下载失败时仍可基于正文重写 |
| 域名白名单 | 避免浪费时间下载不可达图片 |
