## 一、概述

### 1.1 系统目标

为 HCI 智能排障平台构建知识库检索增强生成（RAG）系统，实现：

1. **~7000 KBD 案例知识条目**的结构化存储与智能检索
2. **~10 篇 SOP 排障手册**的精确路由与步骤引导
3. **对话场景下的知识注入**，让 AI 助手基于真实案例和 SOP 生成精准答案

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **KBD（Knowledge Base Document）** | 历史故障案例，陈述性知识（What-happened），从 `root_cause` 提取假设 |
| **SOP（Standard Operating Procedure）** | 排障手册，程序性知识（How-to），按决策树遍历执行 |
| **category_id** | 分类编码（如 `虚拟机-003`），S0 阶段确认，用于精确路由 |

### 1.3 双轨知识架构

```
用户描述故障（S0 意图识别 → category_id 确认）
        │
        ▼ category_id 已知
[第1轨：SOP 直查] sop_document WHERE category_id = X
        │
        ├─ 命中 ──→ 加载 sop_chunk → SOP 步骤引导 → ✅ 返回
        │
        └─ 未命中
                │
                ▼
        [第2轨：KBD 检索] kbd_entry WHERE category_id = X
                │           BM25(tsv) + 向量(embedding) → RRF → top5
                │
                ├─ 命中 ──→ 注入「历史案例参考」
                │
                └─ 未命中 → 人工兜底
```

## 二、数据存储设计

### 2.1 存储分层

| 数据层 | Source of Truth | 说明 |
|--------|----------------|------|
| kbd_entry | DB | KBD 知识条目，整条 embedding，无分块 |
| sop_document | DB | SOP 文档元信息 |
| sop_chunk | DB | SOP 分块，向量 + 全文索引 |
| kb_category | DB | 分类树 |
| sop_skills/ | 文件系统 | SOP 源文件（keywords_map.json + chapters/） |

### 2.2 数据库 Schema

#### kbd_entry — KBD 知识条目

```sql
CREATE TABLE kbd_entry (
    id                  BIGSERIAL PRIMARY KEY,
    support_id          VARCHAR(20) UNIQUE NOT NULL,    -- 深信服案例ID（幂等键）
    support_url         TEXT,                           -- 原始案例 URL
    title               TEXT NOT NULL,                  -- 案例标题
    content_md          TEXT,                           -- 结构化 Markdown
    metadata            JSONB DEFAULT '{}',             -- 补充元数据

    -- 分类字段
    category_id         VARCHAR(32),                    -- 人工确认分类
    ai_category_id      VARCHAR(32),                    -- AI 分类建议
    ai_category_conf    FLOAT,                          -- 分类置信度
    ai_category_reason  TEXT,                           -- 分类理由

    -- 检索字段（published 时生成）
    embedding           vector(1536),                   -- 语义向量
    tsv                 tsvector,                       -- BM25 全文索引

    -- 状态机
    status              VARCHAR(20) NOT NULL DEFAULT 'draft',  -- draft/published/archived/rejected
    reviewer_id         INTEGER,
    reviewed_at         TIMESTAMP WITH TIME ZONE,
    review_note         TEXT,
    published_at        TIMESTAMP WITH TIME ZONE,

    created_at          TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at          TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX idx_kbd_entry_status ON kbd_entry(status);
CREATE INDEX idx_kbd_entry_category ON kbd_entry(category_id);
CREATE INDEX idx_kbd_entry_embedding ON kbd_entry USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
CREATE INDEX idx_kbd_entry_tsv ON kbd_entry USING GIN (tsv);
```

#### sop_document — SOP 文档

```sql
CREATE TABLE sop_document (
    id              SERIAL PRIMARY KEY,
    source_id       VARCHAR(100) UNIQUE,            -- 幂等键，如 sop-vm-start-failure
    category_id     VARCHAR(32),                    -- 关联 KB 分类
    title           VARCHAR(500) NOT NULL,          -- SOP 标题
    content_md      TEXT NOT NULL,                  -- 完整 SOP Markdown
    docx_hash       VARCHAR(64),                    -- 源文件哈希（幂等去重）
    status          VARCHAR(20) DEFAULT 'draft',    -- draft/published/archived
    reviewer_id     INTEGER,
    reviewed_at     TIMESTAMP WITH TIME ZONE,
    published_at    TIMESTAMP WITH TIME ZONE,
    created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX idx_sop_document_category ON sop_document(category_id);
```

#### sop_chunk — SOP 分块检索

```sql
CREATE TABLE sop_chunk (
    id              SERIAL PRIMARY KEY,
    document_id     INTEGER REFERENCES sop_document(id) ON DELETE CASCADE,
    chunk_index     SMALLINT NOT NULL,              -- 分块序号（0-based）
    chapter_title   VARCHAR(200),                   -- 章节标题
    content         TEXT NOT NULL,                  -- 分块内容
    embedding       vector(1536),                   -- 语义向量（审核时生成）
    tsv             tsvector,                       -- 全文检索向量
    created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX idx_sop_chunk_document ON sop_chunk(document_id);
CREATE INDEX idx_sop_chunk_embedding ON sop_chunk USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);
CREATE INDEX idx_sop_chunk_tsv ON sop_chunk USING GIN (tsv);
```

#### kb_category — 分类树

```sql
CREATE TABLE kb_category (
    id              SERIAL PRIMARY KEY,
    parent_id       INTEGER REFERENCES kb_category(id),
    code            VARCHAR(32) UNIQUE NOT NULL,    -- 分类编码，如 "虚拟机-003"
    name            VARCHAR(100) NOT NULL,          -- 分类名称
    level           SMALLINT NOT NULL,              -- 1=L1, 2=L2, 3=L3, 4=L4
    keywords        TEXT[],                         -- 关键字
    created_at      TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX idx_kb_category_parent ON kb_category(parent_id);
CREATE INDEX idx_kb_category_code ON kb_category(code);
```

## 三、SOP 代入方法

### 3.1 SOP 存储规范

| 存储单元 | 说明 |
|---------|------|
| `sop_document` | SOP 文档元信息 + 完整内容 |
| `sop_chunk` | 按章节拆分，支持向量 + 全文检索 |
| `sop_skills/` | 文件系统源文件（keywords_map.json + chapters/） |

### 3.2 SOP 路由机制

**两种路由方式：**

#### 方式一：category_id 直查（O(1)，S1+ 推荐）

```python
# S0 阶段已确认 category_id，S1+ 直接查 SOP
async def get_sop_by_category(category_id: str) -> SopDocument | None:
    async with session() as s:
        result = await s.execute(
            select(SopDocument).where(
                SopDocument.category_id == category_id,
                SopDocument.status == "published"
            )
        )
        return result.scalar_one_or_none()
```

#### 方式二：关键字精确匹配（S0 阶段）

```python
# 从 sop_skills/keywords_map.json 加载关键字索引
matcher = SopMatcher(sop_skills_dir="/app/sop_skills")
await matcher.load()

# 匹配用户问题中的关键字
result = matcher.match(user_query)
# 返回: SopMatchResult(skill_id, node_name, matched_keyword, content, file_path)
```

### 3.3 SOP 注入标签

```
【SOP 排障流程 | 来源: {sop_document.title}】
{章节内容}
```

## 四、KBD 代入方法

### 4.1 KBD 存储规范

- **存储单元**：`kbd_entry`（整条 embedding，无分块）
- **检索字段**：`embedding`（向量）+ `tsv`（BM25），RRF 融合
- **必须字段**：`title`、`content_md`、`category_id`、`embedding`、`tsv`

### 4.2 KBD 检索流程

```
POST /api/kb/search
{
  "query": "虚拟机开机失败",
  "top_n": 5,
  "category_l1": "虚拟机",        # 可选，分类过滤
  "category_l2": "开机失败"       # 可选
}
```

**检索流程：**

1. **BM25 检索**：jieba 分词 → tsquery → tsvector 全文匹配 → top-20
2. **向量检索**：embed(query) → pgvector 余弦相似度 → top-20
3. **RRF 融合**：score = Σ 1/(k + rank_i)，k=60
4. **返回 top-N**

### 4.3 KBD 注入标签

```
【历史案例参考 | 案例 #{support_id}】
{案例内容}
```

## 五、知识注入机制

### 5.1 四层 Prompt 组装

Conversation Service 在每轮对话时组装 messages[]：

```python
messages = [
    # Tier 1: 系统角色
    {"role": "system", "content": AGENTS_MD_CONTENT},

    # Tier 2: SOP 知识（可选，命中时注入）
    {"role": "system", "content": f"【SOP 排障流程】\n{sop_content}"},

    # Tier 3: KBD 知识（可选，检索结果）
    {"role": "system", "content": f"【历史案例参考】\n{kbd_chunks}"},

    # Tier 4: 工单上下文
    {"role": "system", "content": f"工单ID: {case_id}\n工程师: {engineer}"},

    # 历史对话（最近 20 条）
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."},

    # 当前消息
    {"role": "user", "content": current_message}
]
```

### 5.2 知识注入时序

```
S0 阶段（意图识别）:
  1. 用户描述故障
  2. 调用 POST /api/kb/classify → 确认 category_id
  3. 调用 POST /api/kb/sop/match → SOP 关键字匹配
  4. 不触发 KBD 检索（S0 专职分类）

S1+ 阶段（诊断执行）:
  1. 用 category_id 直查 SOP → 命中则注入 Tier 2
  2. 用 category_id 过滤 + query 检索 KBD → top-5 注入 Tier 3
  3. 组装完整 messages[] → 发给 LLM
```

## 六、KB Service API

### 6.1 检索接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/kb/search` | POST | 混合检索（SOP + KBD） |
| `/api/kb/sop/match` | POST | 仅 SOP 关键字匹配 |
| `/api/kb/classify` | POST | LLM 分类器，返回 category_id |
| `/api/kb/classify/intent` | POST | 对话意图识别（conversation-service 调用） |

> **注**：v3.3 删除了 classify.py 中的 `GET /categories` 端点（零调用者，与 categories.py 路由冲突）。分类树查询统一由 `/api/kb/categories` GET（categories.py）提供。

### 6.2 摄入接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/kb/kbd/ingest` | POST | 导入 KBD 知识条目（深信服案例） |
| `/api/sop/ingest` | POST | 导入 SOP 文档（自动分块） |
| `/api/kb/sop/import` | POST | 批量导入 SOP 节点（kb_sop_node 表） |
| `/api/kb/categories` | GET/POST | 分类树管理 |

### 6.3 管理接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/kb/documents` | GET | 文档列表（Admin） |
| `/api/kb/documents/{id}` | PATCH | 审核/发布/下线 |

## 七、知识导入方法

### 7.1 KBD 导入方法

#### 方式一：后台脚本导入（推荐）

```bash
# 单条导入
python -m scripts.kbd.run import --ids 12345

# 批量导入
python -m scripts.kbd.run import --ids 12345,12346,12347

# 从文件列表导入
python -m scripts.kbd.run import --file support_ids.txt
```

**导入流程：**

```
scripts/kbd/importer.py
        │
        ▼ 从 cache/{support_id}/raw.json 读取
    ┌─────────────────────────────┐
    │ 1. converter 生成 content_md │
    │ 2. 调用 API 写入 kbd_entry  │
    │ 3. 状态默认 draft           │
    └─────────────────────────────┘
        │
        ▼ 返回 created/idempotent/error
```

#### 方式二：API 接口导入

```
POST /api/kb/kbd/ingest
Authorization: Bearer {INTERNAL_API_TOKEN}
```

```json
{
  "support_id": "12345",
  "title": "虚拟机开机失败排查案例",
  "content_md": "# 问题描述\n...",
  "metadata": {"product": "HCI", "version": "6.8.1"},
  "ai_category_id": "虚拟机-003",
  "ai_category_conf": 0.92
}
```

**幂等机制**：`support_id` 唯一性校验，已存在则跳过。

#### 状态流转

```
draft ──(Admin 审核)──► published ──(触发 embedding 生成)──► 可检索
```

### 7.2 SOP 导入方法

#### 方式一：后台脚本导入（推荐）

```bash
# 单文件导入
python -m scripts.kbd.import_sop --file /path/to/sop.docx --category-id "虚拟机-003"
```

**导入流程：**

```
scripts/kbd/import_sop.py
        │
        ▼ 解析 .docx 文件
    ┌─────────────────────────────┐
    │ 1. python-docx 解析章节     │
    │ 2. 生成 Markdown 内容       │
    │ 3. 计算 docx_hash 幂等键   │
    │ 4. 调用 API 写入 DB        │
    └─────────────────────────────┘
        │
        ▼ 自动分块写入 sop_chunk
```

#### 方式二：API 接口导入

```
POST /api/sop/ingest
Authorization: Bearer {INTERNAL_API_TOKEN}
```

```json
{
  "source_id": "sop-vm-start-failure",
  "title": "虚拟机开机失败排障手册",
  "content_md": "# 概述\n...\n## 问题诊断\n...",
  "category_id": "虚拟机-003",
  "docx_hash": "a1b2c3d4..."
}
```

**自动分块规则**：按 `##` 或 `###` 标题自动拆分为 `sop_chunk`。

## 九、分类树结构

### 9.1 数据来源

分类基准文件：`backend/kb-service/config/category_baseline.yaml`

- **来源**：云计算-结构图-AI-全.mm
- **总计**：198 条叶节点（L2-L4）
- **L1 域节点**：5 个（虚拟机/网络/存储/硬件/平台）

### 9.2 分类编码规范

```
格式：<DOMAIN>-<SEQ>

示例：
  虚拟机-001  → 虚拟机创建失败
  虚拟机-003  → 虚拟机开机失败
  网络-015    → 分布式防火墙策略不生效
  存储-008    → 存储卷扩容失败
```

### 9.3 层级结构示例

```yaml
# L2 叶节点（直接属于 L1）
- id: 虚拟机-003
  domain: 虚拟机
  label: 虚拟机开机失败
  path:
    - 虚拟机
    - 虚拟机开机失败

# L3 叶节点（中间有分组）
- id: 虚拟机-001
  domain: 虚拟机
  label: 虚拟机创建失败
  path:
    - 虚拟机
    - 虚拟机创建
    - 虚拟机创建失败

# L4 叶节点（多层分组）
- id: 硬盘-001
  domain: 硬件
  label: 单盘故障排查
  path:
    - 硬件
    - 硬盘
    - 硬盘排障
    - 单盘故障排查
```

### 9.4 五大技术域

| L1 域 | 说明 | 典型分类 |
|-------|------|---------|
| 虚拟机 | 虚拟机生命周期管理 | 创建/删除/开机/关机/迁移/克隆 |
| 网络 | 物理网络 + 虚拟网络 | 网口/VLAN/分布式防火墙/负载均衡 |
| 存储 | 虚拟存储 + 外部存储 | 创建/扩容/快照/备份/恢复 |
| 硬件 | 物理硬件故障 | 风扇/电源/CPU/内存/硬盘 |
| 平台 | 集群管理/系统运维 | 授权/升级/监控/日志 |
