# 27_21到22迁移周度看板模板

> 用途：将 23 文档中的迁移执行清单直接映射为周度看板卡片，支持 VK 或任意看板工具。
> 
> 使用方式：
> 1. 每周复制对应章节到看板。
> 2. 每张卡片关联负责人、截止时间、验收标准。
> 3. 周会仅讨论未完成项、阻塞项、风险项。

---

## Week 1：21 基线固化（稳定底座）

### W1-CARD-01 分支保护与 PR 模板落地

- 目标：确保主分支不可绕过审查与门禁。
- 负责人：Platform Owner
- 交付物：分支保护规则、PR 模板生效
- 验收标准：main/release 无法直推；PR 必填风险和回滚字段

### W1-CARD-02 CI 门禁闭环

- 目标：后端、前端、Helm、安全测试门禁可执行。
- 负责人：Backend/Frontend Owner
- 交付物：CI 任务全绿记录
- 验收标准：至少 1 个真实 PR 全量门禁通过

### W1-CARD-03 发布与回滚 SOP 演练

- 目标：验证发布和回滚流程可执行。
- 负责人：Release Manager
- 交付物：演练记录与问题清单
- 验收标准：完成 1 次 staging 发布和 1 次回滚演练

### W1-CARD-04 环境配置分层确认

- 目标：明确 dev/staging/prod 差异与边界。
- 负责人：Platform Owner
- 交付物：环境配置差异表
- 验收标准：发布责任人可按表执行，不依赖口头沟通

---

## Week 2：接入 22 前半段（研发自动化）

### W2-CARD-01 Orchestrator Agent 任务编排

- 目标：自动拆解任务并分配执行顺序。
- 负责人：AI Workflow Owner
- 交付物：任务依赖图、执行计划
- 验收标准：至少 1 个需求自动拆分并可并行执行

### W2-CARD-02 Coder/Reviewer 交叉审查闭环

- 目标：实现自动编码、自动审查、自动回流修复。
- 负责人：Tech Leads
- 交付物：结构化审查报告
- 验收标准：阻断项清零后可自动进入 QA

### W2-CARD-03 QA Agent 统一门禁报告

- 目标：门禁结果标准化输出。
- 负责人：QA Owner
- 交付物：JSON + Markdown 门禁报告模板
- 验收标准：失败项可被自动定位并触发修复

### W2-CARD-04 Agent 权限边界校验

- 目标：防止越权改动与越权发布。
- 负责人：Platform Owner
- 交付物：权限矩阵与审计记录
- 验收标准：Agent 越权改动 = 0

---

## Week 3：接入 22 发布自动化（dev/staging）

### W3-CARD-01 Release Agent 更新环境声明

- 目标：自动提交环境声明变更。
- 负责人：Platform Owner
- 交付物：自动变更 PR 记录
- 验收标准：dev/staging 自动生成并合并发布声明

### W3-CARD-02 Argo 同步与发布可观测

- 目标：自动部署可观测、可追踪。
- 负责人：SRE Owner
- 交付物：发布执行日志、状态快照
- 验收标准：连续 3 次 dev/staging 发布成功

### W3-CARD-03 异常自动降级开关

- 目标：发布失败时自动切回 21 稳定链路。
- 负责人：Release Manager
- 交付物：降级策略与触发条件
- 验收标准：演练中可自动降级并恢复服务

### W3-CARD-04 30 分钟观察报告自动化

- 目标：发布后自动生成观察报告。
- 负责人：SRE Owner
- 交付物：观察报告模板与样例
- 验收标准：每次发布后自动产出报告

---

## Week 4：prod 执行面迁移（审批保留人工）

### W4-CARD-01 审批令牌机制

- 目标：无审批不发布生产。
- 负责人：Release Approver
- 交付物：审批令牌策略文档
- 验收标准：无审批生产发布事件为 0

### W4-CARD-02 Release Agent 执行 prod 发布

- 目标：生产发布从人工执行过渡到 Agent 执行。
- 负责人：Release Agent Owner
- 交付物：生产发布轨迹记录
- 验收标准：1 次演练发布全链路通过

### W4-CARD-03 Rollback Agent 自动回滚

- 目标：故障触发后快速自动回滚。
- 负责人：Incident Commander
- 交付物：回滚报告与恢复验证
- 验收标准：回滚时间达到目标阈值

### W4-CARD-04 迁移复盘与稳定态确认

- 目标：确认进入“人工审批 + Agent 执行”稳定态。
- 负责人：Platform Owner
- 交付物：迁移复盘报告
- 验收标准：M1-M4 里程碑全部达成

## 每周例会检查清单

1. 上周未完成卡片是否有阻塞和明确 owner。
2. 本周卡片是否都有验收标准。
3. 阶段闸门指标是否达标（Go/No-Go）。
4. 是否触发降级或回退策略。
5. 是否需要调整下一周并行度与资源配置。
