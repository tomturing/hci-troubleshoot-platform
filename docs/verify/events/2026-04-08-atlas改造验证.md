### 2.2 Helm 渲染验证

```bash
helm template hci-platform deploy/helm/hci-platform \
  -f deploy/helm/hci-platform/values.yaml \
  | grep -A 30 "name: db-migrate"
```

## 三、K8s 环境验证

### 3.1 Job 状态检查

```bash
# 查看 db-migrate Job 列表
kubectl get job -n <namespace> | grep db-migrate

# 预期：STATUS=Complete，COMPLETIONS=1/1
```

### 3.2 Job 日志检查

```bash
kubectl logs job/db-migrate-<revision> -n <namespace>
```

**成功标志（已有 DB 场景）**：
```
Atlas: Migrating your database <url>
-- starting migration --
-- ok, no migrations to execute
```

**成功标志（全新 DB 场景）**：
```
Atlas: Migrating your database <url>
-- applying migration 20260408000000_baseline.sql
-- ok (500ms)
```

### 3.3 版本跟踪表验证

```bash
kubectl exec -n <namespace> <postgres-pod> -- \
  psql -U hci_admin hci_troubleshoot \
  -c "SELECT version, applied_at, execution_time FROM atlas_schema_revisions ORDER BY applied_at;"
```

**预期**：
- `atlas_schema_revisions` 表存在
- 记录数 ≥ 1（baseline 记录）

### 3.4 业务数据完整性验证

```bash
kubectl exec -n <namespace> <postgres-pod> -- \
  psql -U hci_admin hci_troubleshoot -c "
SELECT tablename, schemaname
FROM pg_tables
WHERE schemaname = 'public'
AND tablename NOT IN ('atlas_schema_revisions')
ORDER BY tablename;"
```

**预期**：17 张业务表全部存在

## 四、开发者工作流验证

**目标**：验证声明式开发工作流可以走通（新增字段 → 生成迁移 → PR 验证）。

### 4.1 本地测试新增迁移

```bash
# 1. 准备本地 dev DB
docker run -d --name pg-atlas-test \
  -e POSTGRES_USER=atlas -e POSTGRES_PASSWORD=test -e POSTGRES_DB=hci_test \
  -p 15432:5432 pgvector/pgvector:pg15
sleep 3

# 2. 设置环境变量
export DATABASE_URL="postgres://atlas:test@localhost:15432/hci_test?sslmode=disable"

# 3. 安装 Atlas（本地开发用）
curl -sSf https://atlasgo.sh | sh -s -- -y

# 4. 执行当前所有迁移（建立 baseline）
atlas migrate apply --env local

# 5. 模拟新增字段到 desired_schema.sql
# 比如在 user 表添加一个 phone 字段
# 编辑 database/desired_schema.sql：
#   ADD `phone varchar(20)` 到 user 表

# 6. 生成迁移文件
atlas migrate diff --env local add_user_phone

# 7. 查看生成的文件
cat database/atlas-migrations/$(ls -t database/atlas-migrations/*.sql | head -1)
# 预期输出：ALTER TABLE "user" ADD COLUMN "phone" character varying(20);

# 清理测试
docker rm -f pg-atlas-test
```

**验收**：Atlas 能正确 diff 并生成符合预期的迁移 SQL。

## 五、负面验证（lint 应能拦截危险操作）

```bash
# 创建一个包含危险操作的测试迁移文件
cat > /tmp/test_dangerous.sql << 'EOF'
-- migrate:up
ALTER TABLE "user" ADD COLUMN dangerous_test varchar(100) NOT NULL;
-- 无默认值的 NOT NULL 列，Atlas lint 应该告警
EOF

# 验证 lint 能检测到
atlas migrate lint --dir file://database/atlas-migrations 2>&1 | grep -i "warning\|error\|MF101"
# 预期输出：包含关于无默认值的警告
```
