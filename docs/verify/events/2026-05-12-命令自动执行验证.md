
**完成标准**：控制台输出 `'off'`，不出现非法模式值

### 3.2 shell 元字符拦截（V4）

1. 切换模式为「安全模式（safe-only）」
2. 让 AI 建议包含管道的命令，如：
   ```bash
   cat /etc/os-release | grep VERSION
   ```
3. 观察 CommandBlock 是否触发倒计时

**完成标准**：含 `|` 的命令不启动倒计时（因 inferRiskLevel 返回 `caution`，safe-only 下不自动执行）

### 3.3 pageVisible 暂停（V5）

1. 切换到「安全/激进」模式
2. 触发一条 acli 只读命令被 AI 建议
3. 倒计时开始后，切换到其他浏览器 Tab（页面隐藏）
4. 等待 3 秒后切回

**完成标准**：倒计时停止而非在后台执行命令

### 3.4 模式开关 UI（V6）

1. 确保 SSH 已连接（终端面板显示已连接状态）
2. 观察终端面板头部

**完成标准**：头部左侧显示「🔴 关闭自动执行」下拉框，可切换为「🟢 安全模式」「🟠 激进模式 ⚠️」

### 3.5 端到端自动执行流程

1. 切换到「安全模式」
2. 在对话框中发送：`帮我查看一下系统磁盘使用情况`
3. 等待 AI 回复包含 `df -h` 命令

**完成标准**：
- CommandBlock 显示 3 秒倒计时
- 倒计时结束后自动执行，显示 `✓ 已执行 (exit 0)`
- 对话框中出现执行结果消息
- AI 继续基于结果分析

### 3.6 会话上限熔断

1. 将 `AUTO_EXEC_MAX` 临时改为 2（开发时验证）
2. 执行 3 次自动执行

**完成标准**：第 3 次时系统消息提示「已累计自动执行 N 条命令，已自动关闭」

## 四、已知设计决策说明

### acli 硬编码问题

`commandRisk.ts` 中 `READONLY_PREFIXES`、`DANGER_PREFIXES`、`CAUTION_PREFIXES` 均为硬编码。

**设计决策**：这是有意为之，原因：
1. HCI 平台的 `acli` CLI 是稳定的产品化工具，命令集变化频率极低
2. 避免引入 API 请求增加启动延迟
3. 风险推断逻辑纯客户端、无网络依赖，适合离线场景

**潜在改进**（非当前迭代）：未来可通过后端 `/api/assistants/command-risk-rules` 接口动态下发规则，支持运维人员配置白名单。
