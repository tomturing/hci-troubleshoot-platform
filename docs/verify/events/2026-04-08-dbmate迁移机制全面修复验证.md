# dbmate 迁移机制全面修复验证

## 验证目标

确认三层修复全部生效，无副作用。

## 完成标准（客观、唯一、可执行）

### C1：无重复 version 号

```bash
make db-check
# 期望：输出 "✅ 所有文件 version 号唯一"，退出码 0
```

### C2：ConfigMap 与迁移文件同步

```bash
make db-check
# 期望：输出 "✅ 所有迁移文件已同步到 ConfigMap，version 号唯一"
```

### C3：sop_document / sop_chunk 表已创建

```bash
# 在目标环境 psql 中执行
\dt sop_*
# 期望输出：
#  Schema |     Name     | Type  |  Owner
# --------+--------------+-------+----------
#  public | sop_chunk    | table | hci_admin
#  public | sop_document | table | hci_admin
```

### C4：迁移执行记录正确

```bash
SELECT version FROM schema_migrations WHERE version >= '20260407' ORDER BY version;
# 期望输出（共 4 条）：
#  20260407001
#  20260407002
#  20260407003
#  20260408001
```

### C5：废弃表不存在

```bash
SELECT tablename FROM pg_tables
WHERE schemaname = 'public'
  AND tablename IN (
    'kb_document','kb_chunk','kb_sop_node','kb_synonym',
    'raw_cases','knowledge_atoms','prompt_audit','tool_audit_log'
  );
# 期望输出：0 rows
```

### C6：总表数为 17

```bash
SELECT COUNT(*) FROM pg_tables
WHERE schemaname = 'public'
  AND tablename NOT IN ('schema_migrations','alembic_version');
# 期望输出：17
```

### C7：Alembic Job 不运行

```bash
kubectl get job -n hci | grep alembic
# 期望输出：无任何行（Job 未被创建）
```

### C8：CI check-db-migrations-sync job 通过

- GitHub Actions PR check `检查数据库迁移同步` 显示绿色
- 日志中可见 "✅ 所有文件 version 号唯一" + "✅ 所有迁移文件已同步到 ConfigMap"
