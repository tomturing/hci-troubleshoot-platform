# 数据库迁移同步自动化验证

## 验证概述

验证数据库迁移同步自动化机制正常工作。

## 验证清单

### 1. 同步脚本验证

- [ ] 脚本可执行
- [ ] 运行后 ConfigMap 包含所有迁移
- [ ] 脚本幂等，重复执行无副作用

```bash
# 运行同步脚本
scripts/ci/sync-db-migrations.sh

# 验证输出包含所有迁移
grep -E "^  [0-9]" deploy/helm/hci-platform/templates/hooks/db-migrations-configmap.yaml | wc -l
# 期望：与 database/migrations/*.sql 数量一致
```

### 2. 检查脚本验证

- [ ] 所有迁移已同步时返回 0
- [ ] 存在未同步迁移时返回 1 并提示

```bash
# 全部同步时应成功
scripts/ci/check-db-migrations-sync.sh
echo $?  # 期望：0

# 模拟未同步（删除 ConfigMap 中一个迁移后检查）
# ...
```

### 3. CI 检查验证

- [ ] PR 中新增迁移但未同步 ConfigMap 时 CI 失败
- [ ] 同步后 CI 通过

### 4. 数据库验证

- [ ] ArgoCD Sync 后迁移 Job 执行成功
- [ ] 数据库表数量正确（17 张）

```bash
# 检查迁移 Job
kubectl logs -n hci-dev job/db-migrate-<revision> --tail=20

# 验证表数量
kubectl exec -n hci-dev deploy/case-service -- \
  psql "$DATABASE_URL" -c "SELECT COUNT(*) FROM information_schema.tables WHERE table_schema = 'public' AND table_type = 'BASE TABLE'"
# 期望：17
```

### 5. 废弃表清理验证

- [ ] 以下表不存在：
  - kb_document
  - kb_chunk
  - kb_sop_node
  - kb_synonym
  - raw_cases
  - knowledge_atoms
  - prompt_audit

```bash
kubectl exec -n hci-dev deploy/case-service -- \
  psql "$DATABASE_URL" -c "SELECT tablename FROM pg_tables WHERE schemaname = 'public' ORDER BY tablename"
```
