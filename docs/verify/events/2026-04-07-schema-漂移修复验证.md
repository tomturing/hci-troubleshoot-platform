# Schema 漂移修复 — 验证方案

## 验证目标

确认修复迁移 `20260407001_schema_repair.sql` 在各环境执行后，DB schema 符合 `数据库设计.md` v6.2 规范，所有依赖服务正常工作。

## 一、Schema 完整性验证

### 1.1 表数量检查

```sql
SELECT COUNT(*) AS table_count
FROM information_schema.tables
WHERE table_schema = 'public' AND table_type = 'BASE TABLE';
```

**预期**：23（17 业务表 + 6 系统/扩展表）

### 1.2 业务表存在性检查

```sql
SELECT table_name
```

| # | 表名 | 所属模块 |

### 1.3 关键列检查

```sql
-- case 表新增列
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'case' AND column_name IN ('customer_id', 'close_reason', 'closed_at');

-- conversation 表新增列
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'conversation'
  AND column_name IN ('repeat_question_count', 'hypothesis', 'diagnostic_stage', 'stage_updated_at');
```

**预期**：每个查询均返回对应列，数据类型与 `数据库设计.md` 一致。

## 三、服务功能验证

### 3.1 case-service 工单创建

```bash
kubectl exec -n <namespace> deploy/case-service -- python3 -c "
import urllib.request, json
req = urllib.request.Request('http://localhost:8001/api/cases/',
    data=json.dumps({
        'title': 'schema验证-$(date +%H%M)',
        'description': '修复验证',
        'assistant_type': 'openclaw',
        'client_id': 'verify-client'
    }).encode(),
    headers={'Content-Type': 'application/json'}, method='POST')
resp = urllib.request.urlopen(req)
print(f'HTTP {resp.status}: {json.loads(resp.read())[\"case_id\"]}')
"
```

**预期**：HTTP 201 + 返回有效 case_id。

### 3.2 conversation-service 会话创建

创建工单后，对该 case 发起会话，验证 `conversation` 表新增列可正常写入。

### 3.3 知识库查询

验证 `kbd_entry`、`sop_document`、`sop_chunk` 表可正常读写。

## 四、幂等性验证

重复执行修复迁移，确认无报错且数据不变：

```bash
kubectl cp database/migrations/20260407001_schema_repair.sql <namespace>/postgres-0:/tmp/
kubectl exec postgres-0 -n <namespace> -- psql -U hci_admin -d hci_troubleshoot \
  -f /tmp/schema_repair.sql
```

**预期**：全部 `IF NOT EXISTS` 分支静默通过，无 ERROR 输出。
