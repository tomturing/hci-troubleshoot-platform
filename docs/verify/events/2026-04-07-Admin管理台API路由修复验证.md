# Admin 管理台 API 路由修复验证

## 验证环境

- 浏览器访问 `http://hci.local:4888/admin/`
- 后端服务：api-gateway(8000)、kb-service(8004)
- 数据库：`kbd_entry` 表中需有 status=draft 的测试数据

## 验证用例

### TC-1：分类基线页面加载

| 项目 | 内容 |
|------|------|
| 前置条件 | `kb_category` 表有活跃分类数据 |
| 操作 | 访问 `/admin/category` |
| 预期结果 | 页面正常加载，左侧按域分组显示分类列表，无报错 |
| 验证命令 | `curl -s -H "Authorization: Bearer hci-dev-internal-token" http://localhost:8000/api/kb/categories?grouped=true \| jq .domains \| head` |
| 通过标准 | 返回 200，`domains` 字段为非空 dict |

### TC-2：知识审核菜单已移除

| 项目 | 内容 |
|------|------|
| 操作 | 查看侧边栏菜单 |
| 预期结果 | 无"知识审核"菜单项，菜单顺序为：仪表盘、工单管理、分类基线、KBD 审核、用户管理、系统监控 |
| 通过标准 | 菜单仅显示 6 项，不含"知识审核" |

### TC-3：KBD 审核列表加载

| 项目 | 内容 |
|------|------|
| 前置条件 | `kbd_entry` 表有 status=draft 的条目 |
| 操作 | 访问 `/admin/knowledge/kbd-review` |
| 预期结果 | 列表正常加载，显示案例 ID、标题、AI 分类、置信度等列 |
| 验证命令 | `curl -s -H "Authorization: Bearer hci-dev-internal-token" "http://localhost:8000/api/v1/kbd/pending?page=1&page_size=5&status=draft" \| jq '{total: .total, count: (.entries \| length)}'` |
| 通过标准 | 返回 200，entries 为数组，total 为整数 |

### TC-4：KBD 审核通过

| 项目 | 内容 |
|------|------|
| 前置条件 | TC-3 通过，有 draft 状态条目 |
| 操作 | 点击某条目"通过"按钮，确认弹窗 |
| 预期结果 | 提示"审核通过，KBD 条目已发布"，条目从列表消失 |
| 通过标准 | 条目 status 变为 published |

### TC-5：KBD 审核拒绝

| 项目 | 内容 |
|------|------|
| 前置条件 | TC-3 通过，有 draft 状态条目 |
| 操作 | 点击"拒绝" → 填写拒绝原因 → 确认 |
| 预期结果 | 提示"已拒绝，状态更新为 rejected"，条目从列表消失 |
| 验证命令 | `curl -s -X PATCH -H "Authorization: Bearer hci-dev-internal-token" -H "Content-Type: application/json" -d '{"reviewer_id":"admin","review_note":"测试拒绝"}' "http://localhost:8000/api/v1/kbd/{kbd_id}/reject" \| jq .` |
| 通过标准 | 返回 `{"success": true, "status": "rejected"}` |

### TC-6：分类基线 YAML 导入/导出

| 项目 | 内容 |
|------|------|
| 操作 | 点击"导入 YAML"按钮，选择文件上传 |
| 预期结果 | dry_run 阶段显示差异预览 |
| 通过标准 | 不崩溃，multipart 文件正确透传到 kb-service |
