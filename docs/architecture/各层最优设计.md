## Layer 1：基础设施层

### 最优设计

**PostgreSQL 15 + pgvector**（知识向量存储）
**Redis 7**（会话缓存 + 人工确认等待队列 + ReAct状态中间缓存）

关键表设计原则：
- 每张表必须有 `trace_id`（W3C traceparent 格式）
- 工具调用独立一张表（`tool_audit_log`），不可绕过写入
- 诊断状态单独字段，不混在 JSON blob 里

**核心表**：

```sql
-- 对话会话（扩展现有，加诊断状态字段）
```

## Layer 2：LLM 接入层

### 最优设计：GLM 专用适配器

不直接用 LangChain 的 `ChatOpenAI`，而是封装一个 `GLMClient`，处理 GLM 的特有行为：

```python
from openai import AsyncOpenAI
from pydantic import BaseModel

class LLMResponse(BaseModel):
    content: str | None
    finish_reason: str          # stop | tool_calls | length
    tool_calls: list[ToolCall]
    usage: dict

class GLMClient:
    """GLM 专用客户端，处理与标准 OpenAI 的格式差异"""

    def __init__(self, base_url: str, api_key: str, model: str):
        self.client = AsyncOpenAI(base_url=base_url, api_key=api_key)
        self.model = model

    async def chat(
        self,
        messages: list[dict],
        tools: list[dict] | None = None,
        stream: bool = True
    ) -> LLMResponse:
        params = {
            "model": self.model,
            "messages": messages,
            "stream": stream,
            "temperature": 0.1,          # 排障场景要求确定性
            "max_tokens": 4096,
        }
        if tools:
            params["tools"] = tools
            params["tool_choice"] = "auto"

        if stream:
            return self._stream_chat(params)
        else:
            resp = await self.client.chat.completions.create(**params)
            return self._parse_response(resp)

    def _parse_response(self, resp) -> LLMResponse:
        choice = resp.choices[0]
        tool_calls = []
        if choice.message.tool_calls:
            for tc in choice.message.tool_calls:
                try:
                    args = json.loads(tc.function.arguments)
                except json.JSONDecodeError:
                    # GLM 偶尔返回非标准 JSON，需要容错
                    args = self._repair_json(tc.function.arguments)
                tool_calls.append(ToolCall(
                    id=tc.id,
                    name=tc.function.name,
                    args=args
                ))
        return LLMResponse(
            content=choice.message.content,
            finish_reason=choice.finish_reason,
            tool_calls=tool_calls,
            usage=resp.usage.model_dump()
        )

    def _repair_json(self, raw: str) -> dict:
        """修复 GLM 偶尔输出的非标准 JSON（如缺少引号、尾随逗号）"""
        import re
        # 移除尾随逗号
        raw = re.sub(r',\s*}', '}', raw)
        raw = re.sub(r',\s*]', ']', raw)
        try:
            return json.loads(raw)
        except Exception:
            return {"_raw": raw}    # 降级：保留原始字符串
```

### Prompt 组装：5段式结构

```python
def build_system_prompt(
    knowledge_chunks: list[dict],
    diagnostic_stage: str,
    case_context: dict
) -> str:
    """
    5段式 Prompt 构建：
    Segment 1: 专家身份
    Segment 2: 诊断方法论
    Segment 3: 机制知识锚（领域硬编码知识）
    Segment 4: 参考资料（动态注入 RAG 结果）
    Segment 5: 当前任务上下文
    """
    segments = [
        _segment_identity(),
        _segment_methodology(diagnostic_stage),
        _segment_mechanism_knowledge(),
        _segment_reference_materials(knowledge_chunks),
        _segment_task_context(case_context, diagnostic_stage),
    ]
    return "\n\n---\n\n".join(segments)

def _segment_identity() -> str:
    return """## 你的角色
你是一位资深 HCI（超融合基础设施）系统排障专家，拥有超过10年的实战经验。
你精通 AHV 虚拟化、分布式存储（DSF/NDFS）、网络配置、硬件诊断。
你现在正在帮助客户诊断 HCI 系统故障。"""

def _segment_methodology(stage: str) -> str:
    stage_guidance = {
        "S0": "当前处于症状采集阶段。你的首要任务是精确理解故障现象，不要过早下结论。",
        "S1": "当前处于分类定位阶段。根据已知症状，确定故障属于哪个技术域（VM/存储/网络/硬件/平台）。",
        "S2": "当前处于假设生成阶段。提出3个最可能的根因假设，按概率排序，说明推理依据。",
        "S3": "当前处于假设验证阶段。选择验证成本最低的假设优先验证，使用工具收集证据。",
        "S4": "当前处于根因确认阶段。根据验证结果收敛到最终根因，说明置信度。",
        "S5": "当前处于方案输出阶段。给出具体的修复步骤，区分临时止血和永久修复。",
        "S6": "当前处于关单总结阶段。确认问题解决，生成可复用的知识摘要。",
    }
    return f"""## 工作方法论
遵循以下诊断原则：
1. 先问"是什么"（症状），再问"为什么"（根因），最后问"怎么办"（方案）
2. 每个假设必须有验证路径，不发表无法验证的猜测
3. 使用工具收集证据时，说明期望得到什么信息、为什么需要它
4. 保持不确定性诚实：说"可能"时给出概率，说"确定"时给出依据
5. 区分"现象"和"根因"：告警是现象，导致告警的机制问题才是根因

**当前诊断阶段**：{stage_guidance.get(stage, '')}"""

def _segment_mechanism_knowledge() -> str:
    return """## HCI 系统机制知识（内置，无需参考资料）

### 虚拟机故障常见根因树
- 无法开机 → 资源不足（CPU/内存超分），磁盘损坏，网络隔离，快照链损坏
- 性能劣化 → 存储延迟高（检查 IOPS/BW/latency），CPU 争抢（查 ready time），内存气球
- 网络不通 → VLAN 配置，vSwitch 设置，物理链路，IP 冲突

### 存储故障根因树
- 磁盘离线 → 驱动报错，SMART 异常，SAS/SATA 控制器问题
- 存储延迟高 → 磁盘填满（>85% 触发告警），冷热分层异常，网络带宽争抢
- 数据不一致 → 强一致写失败，节点宕机未完成副本同步

### 告警处理优先级原则
- 多个告警同时存在时，硬件告警（HW）> 存储告警（Storage）> 网络告警（Network）> 虚拟机告警（VM）
- 相关告警往往有因果链，找出根源告警比逐一处理更高效"""

def _segment_reference_materials(chunks: list[dict]) -> str:
    if not chunks:
        return """## 参考资料
当前故障类型暂无匹配的 SOP 或历史案例。
请基于你的 HCI 领域机制知识进行推理，不要因为没有参考资料就拒绝诊断。"""

    content = "## 参考资料（已按相关性排序）\n\n"
    for i, chunk in enumerate(chunks[:5], 1):      # 最多注入5条
        content += f"### 参考 {i}：{chunk['source_type']} — {chunk.get('title', '')}\n"
        content += chunk['content'] + "\n\n"
    content += "**使用规则**：参考资料是辅助，当资料与你的机制判断矛盾时，说明差异，不要盲目服从参考资料。"
    return content
```

## Layer 3：知识层

### 最优设计：三级知识体系

```
知识层
  ├── L1：机制知识（Prompt 硬编码）
  │     → 不依赖检索，LLM 随时可用
  │     → 内容：故障根因树、组件关系、指标阈值
  │
  ├── L2：SOP 知识库（BM25 + Vector 混合检索）
  │     → 诊断到 S1（分类确定）后触发
  │     → 内容：标准化排障步骤，按 category_l1/l2 索引
  │
  └── L3：案例知识库（相似案例检索）
        → 在 S2（假设生成）时触发
        → 内容：历史工单，结构化为：症状+根因+解法
```

### 检索策略

```python
class KnowledgeRetriever:
    """三级知识检索，按需触发"""

    async def retrieve(
        self,
        stage: str,
        category: str | None,
        query: str,
        symptoms_embedding: list[float]
    ) -> list[dict]:

        chunks = []

        # L2：SOP 检索（分类确定后）
        if stage in ("S1", "S2", "S3") and category:
            sop_results = await self._hybrid_search(
                query=query,
                filter={"category_l1": category, "source_type": "sop"},
                top_k=3
            )
            chunks.extend(sop_results)

        # L3：案例检索（假设生成时）
        if stage in ("S2", "S3"):
            case_results = await self._vector_search(
                embedding=symptoms_embedding,
                filter={"source_type": "case"},
                top_k=3
            )
            chunks.extend(case_results)

        # RRF 融合去重
        return self._rrf_merge(chunks)

    async def _hybrid_search(self, query, filter, top_k) -> list[dict]:
        """BM25（关键词）+ 向量（语义）混合，RRF 合并"""
        bm25_results = await self._bm25_search(query, filter, top_k * 2)
        vec_results = await self._vector_search_by_text(query, filter, top_k * 2)
        return self._rrf_merge(bm25_results + vec_results)[:top_k]

    def _rrf_merge(self, results: list[dict], k=60) -> list[dict]:
        """Reciprocal Rank Fusion"""
        scores = {}
        for rank, item in enumerate(results):
            doc_id = item["id"]
            scores[doc_id] = scores.get(doc_id, 0) + 1 / (k + rank + 1)
        sorted_ids = sorted(scores, key=scores.get, reverse=True)
        id_to_item = {r["id"]: r for r in results}
        return [id_to_item[i] for i in sorted_ids if i in id_to_item]
```

#### 4.2 SCP Adapter（新增微服务）

```python
import httpx
from tenacity import retry, stop_after_attempt, wait_exponential

class SCPAdapter:
    """SCP（Service Control Platform）的统一适配器"""

    def __init__(self, base_url: str, auth_token: str):
        self.client = httpx.AsyncClient(
            base_url=base_url,
            headers={"Authorization": f"Bearer {auth_token}"},
            timeout=30.0
        )

    @retry(stop=stop_after_attempt(3), wait=wait_exponential(min=1, max=10))
    async def get_active_alerts(self, severity: str = "all", since_minutes: int = 60) -> dict:
        resp = await self.client.get("/api/v1/alerts", params={
            "severity": severity,
            "since": f"{since_minutes}m"
        })
        resp.raise_for_status()
        return resp.json()

    async def get_failed_tasks(self, task_type: str | None = None, hours: int = 24) -> dict:
        params = {"status": "FAILED", "hours": hours}
        if task_type:
            params["type"] = task_type
        resp = await self.client.get("/api/v1/tasks", params=params)
        resp.raise_for_status()
        return resp.json()

    async def execute_acli(self, command: str, args: dict) -> dict:
        """通过 SCP 调用 acli（安全容器隔离，安全底线由平台保证）"""
        resp = await self.client.post("/api/v1/acli/execute", json={
            "command": command,
            "args": args
        })
        resp.raise_for_status()
        return resp.json()
```

## Layer 5：ReAct 执行层

### 最优设计：有状态 ReAct 执行器

这是整个系统的核心，需要处理：推理循环、流式输出、人工确认中断、审计记录。

```python
import asyncio
from typing import AsyncGenerator

class ReactExecutor:
    """
    HCI 排障专用 ReAct 执行器
    设计参考 LangGraph 概念，但为现有 FastAPI + SSE 架构定制
    """

    MAX_STEPS = 15              # 防止无限循环
    CONFIRM_TIMEOUT = 1800      # 人工确认超时 30 分钟

    def __init__(self, llm: GLMClient, tools: list[ToolDefinition], scp: SCPAdapter, kb: KnowledgeRetriever):
        self.llm = llm
        self.tool_map = {t.name: t for t in tools}
        self.scp = scp
        self.kb = kb

    async def run(
        self,
        session: ConversationSession,
        user_input: str
    ) -> AsyncGenerator[dict, None]:
        """
        主循环，返回异步生成器（直接对接 SSE）
        yield 的事件类型：
          {"type": "token", "content": "..."}           → LLM 生成的文字，实时推送
          {"type": "tool_start", "tool": "...", ...}    → 工具开始执行
          {"type": "tool_result", "tool": "...", ...}   → 工具执行完成
          {"type": "confirm_request", "tool": ..., ...} → 需要用户确认
          {"type": "stage_change", "from": "S2", "to": "S3"} → 诊断阶段变化
          {"type": "final", "content": "..."}           → 最终回复
        """
        # 构建 Prompt（注入当前诊断阶段的引导）
        knowledge = await self.kb.retrieve(
            stage=session.diagnostic_stage,
            category=session.hypothesis.get("category") if session.hypothesis else None,
            query=user_input,
            symptoms_embedding=await self.llm.embed(user_input)
        )
        system_prompt = build_system_prompt(knowledge, session.diagnostic_stage, session.case_context)

        # 组装消息历史
        messages = [
            {"role": "system", "content": system_prompt},
            *session.messages,
            {"role": "user", "content": user_input}
        ]

        for step in range(self.MAX_STEPS):
            # ── 推理 ──
            full_content = ""
            tool_calls_buffer = []

            async for event in self.llm.stream_chat(messages, tools=self._to_openai_tools()):
                if event.type == "content_delta":
                    full_content += event.content
                    yield {"type": "token", "content": event.content}
                elif event.type == "tool_call_delta":
                    tool_calls_buffer = event.tool_calls   # 工具调用在完成后整体返回

            # ── 判断退出 ──
            if not tool_calls_buffer:
                # LLM 给出了最终答案
                await self._save_turn(session, user_input, full_content, messages)
                yield {"type": "final", "content": full_content}
                return

            # 把 LLM 的 assistant 回复加入历史
            messages.append({
                "role": "assistant",
                "content": full_content or None,
                "tool_calls": [tc.to_openai_format() for tc in tool_calls_buffer]
            })

            # ── 执行工具 ──
            for tool_call in tool_calls_buffer:
                tool_def = self.tool_map.get(tool_call.name)
                if not tool_def:
                    result = {"error": f"未知工具: {tool_call.name}"}
                    messages.append(self._tool_result_msg(tool_call, result))
                    continue

                yield {"type": "tool_start", "tool": tool_call.name, "args": tool_call.args}

                # 授权检查（框架强制，与 LLM 无关）
                if tool_def.policy == RiskPolicy.CONFIRM:
                    yield {
                        "type": "confirm_request",
                        "tool": tool_call.name,
                        "args": tool_call.args,
                        "description": tool_def.description,
                        "confirm_key": f"confirm:{session.id}"
                    }
                    # 阻塞等待用户确认（或超时）
                    confirmed = await self._wait_confirmation(session.id)
                    if not confirmed:
                        result = {"status": "cancelled", "message": "用户取消了此操作"}
                        await self._audit(session.id, tool_call, tool_def, result, authorized_by=None)
                        messages.append(self._tool_result_msg(tool_call, result))
                        yield {"type": "tool_result", "tool": tool_call.name, "result": result}
                        continue

                # 执行工具
                try:
                    result = await self._dispatch(tool_call, tool_def)
                    await self._audit(session.id, tool_call, tool_def, result, authorized_by=session.user_id)
                except Exception as e:
                    result = {"error": str(e)}
                    await self._audit(session.id, tool_call, tool_def, result, authorized_by=None)

                messages.append(self._tool_result_msg(tool_call, result))
                yield {"type": "tool_result", "tool": tool_call.name, "result": result}

                # 检测诊断阶段是否应该推进
                new_stage = self._infer_stage_transition(session, messages)
                if new_stage != session.diagnostic_stage:
                    yield {"type": "stage_change", "from": session.diagnostic_stage, "to": new_stage}
                    session.diagnostic_stage = new_stage

        # 超出最大步数
        yield {"type": "final", "content": "诊断步骤超出限制，请提供更多具体信息。"}

    async def _wait_confirmation(self, session_id: str) -> bool:
        """等待用户确认（Redis BRPOP，超时返回 False）"""
        redis_key = f"confirm:{session_id}"
        result = await redis.brpop(redis_key, timeout=self.CONFIRM_TIMEOUT)
        if result is None:
            return False
        return json.loads(result[1]).get("confirmed", False)

    async def _audit(self, session_id, tool_call, tool_def, result, authorized_by):
        """写入审计日志，不可绕过"""
        await db.execute(
            INSERT_TOOL_AUDIT_LOG,
            session_id=session_id,
            tool_name=tool_call.name,
            tool_args=tool_call.args,
            risk_level=tool_def.risk_level,
            policy=tool_def.policy,
            authorized_by=authorized_by,
            result=result
        )
```

## Layer 6：对话管理层

### 最优设计：有状态会话管理（参考 LangGraph Thread 概念）

```python
class ConversationSession(BaseModel):
    """对话会话状态，ReAct 执行器的状态容器"""
    id: UUID
    case_id: UUID
    user_id: str
    trace_id: str

    # 诊断状态
    diagnostic_stage: str = "S0"           # S0-S6
    category: str | None = None            # 确定后的故障分类
    hypothesis: list[dict] = []            # 当前假设列表（按概率排序）

    # 消息历史（完整 OpenAI 格式）
    messages: list[dict] = []

    # 执行元数据
    step_count: int = 0
    pending_confirm: dict | None = None    # 待确认的工具调用
    created_at: datetime
    updated_at: datetime

    # 案例上下文（从 Case Service 获取）
    case_context: dict = {}


class ConversationManager:
    """会话管理：创建、加载、持久化、上下文压缩"""

    MAX_MESSAGE_TOKENS = 8000       # 超过此值触发上下文压缩
    COMPRESSION_TARGET = 4000

    async def get_or_create(self, case_id: UUID, user_id: str) -> ConversationSession:
        existing = await db.fetchrow(SELECT_SESSION_BY_CASE, case_id)
        if existing:
            return ConversationSession(**existing)
        return await self._create_session(case_id, user_id)

    async def save(self, session: ConversationSession):
        await db.execute(UPSERT_SESSION, **session.dict())

    async def compress_if_needed(self, session: ConversationSession) -> ConversationSession:
        """上下文压缩：保留系统消息 + 最近N轮 + 中间摘要"""
        token_count = count_tokens(session.messages)
        if token_count <= self.MAX_MESSAGE_TOKENS:
            return session

        # 保留：最近 6 轮对话（确保连贯性）
        recent = session.messages[-12:]
        older = session.messages[:-12]

        # 压缩较旧的对话为摘要
        summary = await self.llm.summarize(older,
            prompt="用简洁的中文总结以下排障对话的关键发现，包括：已确认的症状、排除的假设、当前假设。")

        session.messages = [
            {"role": "system", "content": f"[对话摘要] {summary}"},
            *recent
        ]
        return session
```

## 各层设计原则汇总

| 层 | 核心原则 | 关键决策 |
|----|---------|---------|
| Layer 1 基础设施 | 审计不可绕过 | `tool_audit_log` 独立表，强制写入 |
| Layer 2 LLM接入 | 容错优先 | GLM JSON 修复，温度 0.1 保确定性 |
| Layer 3 知识 | 检索时机延迟 | 分类确定后再检索，减少噪声 |
| Layer 4 工具 | 风险静态声明 | 工具定义里写 `policy`，框架强制 |
| Layer 5 ReAct | 透明可审计 | 每步 yield 事件，每次调用写审计 |
| Layer 6 对话 | 状态机驱动 | S0-S6 明确阶段，Prompt 跟随阶段 |
| Layer 7 前端 | 风险可视化 | 风险级别颜色区分，确认前展示参数 |
