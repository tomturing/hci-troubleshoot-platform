## 待实现任务

### T1：实现叶优先解析器 `sop_parser.py`

**描述**：
实现 Markdown → SOPNode 树的状态机解析器，采用叶优先策略。

**核心逻辑**：
```python
# 关键词等效表（解析器层唯一来源）
DIAGNOSIS_KEYWORDS: frozenset[str] = frozenset([
    "判断方法", "判断依据", "排查方法", "排查步骤", "识别方法", "确认方法", "诊断方法",
])
SOLUTION_KEYWORDS: frozenset[str] = frozenset([
    "解决方案", "解决方法", "处理方法", "处理步骤", "修复方法", "修复步骤", "解决步骤",
])

def classify_heading(text: str) -> Literal["diagnosis", "solution", "node", "unknown"]:
    """标题分类：关键词匹配，与 H 层级无关"""
    for kw in DIAGNOSIS_KEYWORDS:
        if kw in text:
            return "diagnosis"
    for kw in SOLUTION_KEYWORDS:
        if kw in text:
            return "solution"
    return "node"  # 普通节点标题
```

**解析流程（状态机）**：
1. 扫描所有 Heading，不截断层级（`#` 数量即为 level，不限制到 H3）
2. 对每个 Heading 执行 `classify_heading(text)`
3. `"diagnosis"` / `"solution"` → 收集内容到当前叶节点候选的 `DiagnosisDetail` / `SolutionDetail`，记录 `source_heading=原始文本`
4. `"node"` → 按相对层级构建树结构（子节点挂到最近的更高层父节点下）
5. 扫描完成后，对 children=[] 且无 diagnosis/solution 的节点生成 `ValidationIssue(level="error", ...)`
6. 对 source_heading 不在标准词（"判断方法"/"解决方案"）中的节点生成 `ValidationIssue(level="warning", ...)`

**中文编号列表支持**：
- `^\d+[、.]\s*(.+)` → 数字+顿号/点 格式
- `^[-*]\s+(.+)` → Markdown 无序列表
- `^\d+\.\s+(.+)` → Markdown 有序列表

**文件变更**：
- 新建：`backend/kb-service/app/services/sop_parser.py`

**验收标准**：
- [ ] 任意深度的 SOP 文档均能正确识别根和叶
- [ ] H1→H2（只有两层）的 SOP 正确解析叶节点
- [ ] source_heading 正确记录原始标题文本
- [ ] 单测：`tests/unit/test_sop_parser.py` 覆盖至少 5 个典型结构

### T3：实现 `SOPValidationResult` 构建逻辑

**描述**：
在 `sop_parser.py` 中实现完整的 `parse_sop_markdown()` 函数，返回 `SOPValidationResult`。

**函数签名**：
```python
def parse_sop_markdown(content_md: str) -> SOPValidationResult:
    """
    Markdown → SOPNode 决策树 + 校验结果

    返回：
      is_valid=True   → tree 非 None，errors 为空（warnings 可有）
      is_valid=False  → tree 为 None（有 error 级别问题）
    """
```

**校验规则**：
| 规则 | 级别 | 触发条件 |
|------|------|---------|
| 叶节点缺 diagnosis | error | children=[] 且 diagnosis is None |
| 叶节点缺 solution | error | children=[] 且 solution is None |
| source_heading 非标准词 | warning | source_heading not in {"判断方法", "解决方案"} |
| 路由节点带内容段落 | warning | children 非空且 diagnosis/solution 非 None |
| 未识别段落 | warning | classify_heading() 返回 "unknown" |

**验收标准**：
- [ ] `errors` 列表包含精确的节点路径（`location` 字段）
- [ ] `warnings` 正确覆盖非标准话术
- [ ] 整树有 error 时 `tree=None`、`is_valid=False`
- [ ] 整树只有 warning 时 `tree` 非 None、`is_valid=True`

**依赖**：T1
