# Task 04：知识原子数据库设计与迁移（P1）

```
你是一名负责 hci-troubleshoot-platform 数据库架构的 agent。

【仓库】
git clone https://github.com/tomturing/hci-troubleshoot-platform.git
cd hci-troubleshoot-platform

【背景】
当前知识库使用 knowledge_chunks 表存储切块后的文本向量，缺乏结构化元数据，
导致 AI Agent 无法区分"诊断步骤"、"解决方案"、"判断条件"等不同类型的知识，
检索精度受限，且无法追溯知识来源和版本适用范围。

本任务建立「知识原子」数据模型，是 Phase 1 知识库重建的基础。

知识原子设计原则：
  - 每个知识原子有明确的 type（知识类型：diagnostic_step/fix_action/decision_gate/
    background/escalation）
  - 关联 category_baseline.yaml 的叶节点（198 个分类）
  - 含触发条件（task_error_keywords），支持精确匹配路由
  - 含版本范围（applicable_version_min），避免错用旧方案
  - 含来源追踪（source_type/source_ref）和置信度

同时需要新增：
  - error_code_index 表：HCI 错误码精确匹配索引（从截图/文本中提取的错误码，
    是排障 AI 的最强锚点）
  - raw_cases 表：案例原始数据存储（Phase 4 数据管道使用）

【任务目标】
1. 创建 Alembic 迁移 002_knowledge_atoms.py，定义 knowledge_atoms 表
2. 创建 error_code_index 表和 raw_cases 表
3. 在 kb-service 的 SQLAlchemy 模型中添加对应 ORM 类
⚠️ 检索接口路由不在本任务范围内，由 Task 06 统一实现，避免并行冲突。

【涉及服务 / 文件范围】
允许修改：
  - backend/shared/migrations/versions/（新增 002 迁移文件）
  - backend/shared/models/（新增知识原子 ORM 模型）
禁止修改（职责边界）：
  - backend/kb-service/app/api/（API 路由由 Task 06 负责，Task 04 不接触）
只读参考：
  - docs/architecture/知识库重建设计方案.md § 六、存储层设计
  - data-pipeline/config/category_baseline.yaml（198 个分类 ID）
  - backend/shared/migrations/versions/0001_initial_schema.py（参考格式）
禁止：
  - 修改 database/init_schema.sql（生产数据安全）
  - 修改已有迁移文件

【详细实现步骤】

Step 2：添加 SQLAlchemy ORM 模型

在 backend/shared/models/ 下新建 knowledge_atom.py：

```python
"""知识原子 ORM 模型"""
from sqlalchemy import Column, String, JSON, Boolean, Float, Integer, Text, TIMESTAMP
from sqlalchemy.sql import func
from pgvector.sqlalchemy import Vector
from .base import Base  # 使用现有 Base


class KnowledgeAtom(Base):
    """知识原子——知识库的最小单元"""
    __tablename__ = "knowledge_atoms"

    id = Column(String(64), primary_key=True, comment="知识原子 ID")
    type = Column(String(32), nullable=False, comment="知识类型")
    category_id = Column(String(32), index=True, comment="分类 ID")
    category_path = Column(JSON, comment="分类路径")
    knowledge_domain = Column(String(16), comment="知识来源")
    trigger = Column(JSON, comment="触发条件")
    content = Column(JSON, nullable=False, comment="知识内容")
    applicable_version_min = Column(String(20), comment="最低适用版本")
    applicable_version_max = Column(String(20), comment="最高适用版本")
    source_type = Column(String(16), comment="来源类型")
    source_ref = Column(Text, comment="来源引用")
    confidence = Column(Float, default=0.8, comment="置信度")
    verified = Column(Boolean, default=False, comment="人工验证")
    embedding = Column(Vector(1536), comment="语义向量")
    usage_count = Column(Integer, default=0, comment="被引用次数")
    feedback_positive = Column(Integer, default=0)
    feedback_negative = Column(Integer, default=0)
    trace_id = Column(String(55), comment="写入时的 trace ID")
    created_at = Column(TIMESTAMP(timezone=True), server_default=func.now())
    updated_at = Column(TIMESTAMP(timezone=True), server_default=func.now(),
                        onupdate=func.now())
```
