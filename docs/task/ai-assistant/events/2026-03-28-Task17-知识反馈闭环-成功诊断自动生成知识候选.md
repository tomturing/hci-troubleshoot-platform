# Task 17：知识反馈闭环——成功诊断自动生成知识候选（P2）

```
你是一名负责 hci-troubleshoot-platform 知识闭环的 agent。

【仓库】
git clone https://github.com/tomturing/hci-troubleshoot-platform.git
cd hci-troubleshoot-platform

【背景】
当用户完成一次成功的故障排障（对话到达 S6 阶段），系统记录了完整的：
  - 故障类型（category_id）
  - 使用的诊断工具和参数（tool_audit_log）
  - 工具输出结果（在消息历史中）
  - 根因分析内容（在消息历史中）
  - 解决方案（在消息历史中）

这些信息可以自动提炼为新的知识原子，充实知识库：
  - 复现路径（trigger conditions）
  - 诊断命令序列
  - 根因模式
  - 推荐解决方案

注意：自动生成的知识原子不直接入库，需要人工（管理员）审核后才标记 verified=true

前置条件：
  - Task 04（knowledge_atoms 表已建立）
  - Task 07（tool_audit_log 已记录）
  - Task 08（对话状态机，S6 阶段）

【任务目标】
1. 在对话进入 S6（验证闭环）阶段时，触发知识提炼
2. 调用 GLM 对本次对话进行结构化总结，生成知识原子候选
3. 写入 knowledge_atoms 表（verified=false，需人工审核）
4. 在管理控制台（admin 前端）展示待审核知识原子列表

【涉及服务 / 文件范围】
允许新建/修改：
  - backend/conversation-service/app/services/knowledge_extractor.py（新建）
  - backend/kb-service/app/api/review.py（新建：审核 API）
  - frontend/admin/src/views/KnowledgeReviewView.vue（新建：审核页面）
  - backend/conversation-service/app/services/conversation_service.py（触发逻辑）

【详细实现步骤】

Step 1：KnowledgeExtractor 服务

```python
# backend/conversation-service/app/services/knowledge_extractor.py
"""从完成的诊断对话中提炼知识原子候选"""
import logging
from datetime import datetime, timezone
import uuid
from ..core.glm_client import GLMClient

logger = logging.getLogger(__name__)

EXTRACTION_PROMPT = """
你是一位 HCI（超融合基础设施）知识管理专家。
以下是一次完整的故障排障对话记录（JSON 格式）。
请从中提炼出可复用的诊断知识，输出 JSON 格式的知识原子列表。

每个知识原子格式：
{
  "type": "diagnostic_step|fix_action|decision_gate",
  "trigger": {
    "stage": "S2|S3|S4",
    "category_id": "（故障分类 ID）",
    "conditions": ["触发该步骤的条件描述"]
  },
  "content": {
    "full_text": "完整诊断步骤描述",
    "commands": ["相关 acli/系统命令列表"]
  },
  "source_ref": "本次对话 session_id"
}

对话记录：
{conversation_summary}

仅输出 JSON 数组，不要有其他文字。
"""

class KnowledgeExtractor:
    def __init__(self, glm_client: GLMClient, kb_service_url: str):
        self.glm = glm_client
        self.kb_url = kb_service_url

    @classmethod
    def from_env(cls, glm_client: GLMClient) -> "KnowledgeExtractor":
        """从环境变量创建实例。KB_SERVICE_URL 指向 kb-service 内部地址。"""
        import os
        return cls(
            glm_client=glm_client,
            kb_service_url=os.environ["KB_SERVICE_URL"],    # 如 http://kb-service:8003
        )

    async def extract_from_session(
        self,
        session_id: str,
        conversation_messages: list[dict],
        tool_audit_logs: list[dict],
        category_id: str,
    ) -> list[str]:
        """
        从会话中提炼知识原子候选，返回创建的原子 ID 列表。
        只在 S6 阶段（对话成功闭环）触发。
        """
        # 构建对话摘要（含工具调用结果）
        summary = self._build_summary(conversation_messages, tool_audit_logs)

        prompt = EXTRACTION_PROMPT.format(
            conversation_summary=summary
        )

        response = await self.glm.chat([{"role": "user", "content": prompt}])
        candidates = self.glm._safe_parse_json(response.content or "[]", "extractor")

        if not isinstance(candidates, list):
            logger.warning(f"知识提炼结果非列表: {response.content[:100]}")
            return []

        # 写入知识库服务（verified=false，待审核）
        created_ids = []
        import httpx
        for candidate in candidates[:5]:   # 最多提炼 5 条
            try:
                atom_id = f"ka-{uuid.uuid4().hex[:12]}"
                # 调用 KB Service API 写入
                async with httpx.AsyncClient() as client:
                    resp = await client.post(
                        f"{self.kb_url}/api/v1/atoms",
                        json={
                            "id": atom_id,
                            "type": candidate.get("type", "diagnostic_step"),
                            "category_id": category_id,
                            "trigger": candidate.get("trigger", {}),
                            "content": candidate.get("content", {}),
                            "source_type": "session",
                            "source_ref": session_id,
                            "verified": False,   # 待人工审核
                            "confidence": 0.7,   # 机器生成，初始置信度 0.7
                        }
                    )
                    if resp.status_code == 201:
                        created_ids.append(atom_id)
            except Exception as e:
                logger.error(f"知识原子写入失败: {e}")

        logger.info(f"会话 {session_id} 提炼了 {len(created_ids)} 个知识候选")
        return created_ids

    def _build_summary(self, messages: list[dict], audit_logs: list[dict]) -> str:
        """构建对话摘要（仅保留关键信息）"""
        parts = []
        for msg in messages[-20:]:    # 只取最近 20 条消息
            role = msg.get("role", "")
            content = str(msg.get("content", ""))[:300]   # 截断长内容
            parts.append(f"[{role}] {content}")

        if audit_logs:
            parts.append("\n--- 工具调用记录 ---")
            for log in audit_logs[:10]:
                parts.append(
                    f"工具: {log.get('tool_name')} | "
                    f"参数: {str(log.get('tool_args', {}))[:100]} | "
                    f"结果: {str(log.get('result', ''))[:200]}"
                )
        return "\n".join(parts)
```

【约束】
- 知识提炼必须异步执行，不可阻塞（异步 create_task）
- 自动生成的原子 verified 字段初始为 false，管理员审核后才能被检索系统使用
- 单次提炼最多生成 5 个候选（防止 LLM 过度生成）
- 提炼失败不影响对话正常结束

- [ ] S6 阶段成功触发知识提炼，knowledge_atoms 表出现 verified=false 的记录

- [ ] "通过"操作将 verified 改为 true，此后知识检索能返回该原子

- [ ] uv run pytest backend/kb-service/tests/test_review_api.py -v 通过
