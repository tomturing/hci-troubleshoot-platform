## 任务清单

### T1：ops-agent `ACPSession` 新增 `trajectory_dir` 字段

- **描述**：在 `ACPSession` dataclass 中新增 `trajectory_dir: str | None = None` 字段，用于存储 per-session trajectory 目录路径
- **文件变更**：`ops-agent/ops_agent/acp/server.py`
- **验收标准**：`ACPSession` 可以携带 `trajectory_dir`，不影响已有字段
- **依赖**：无
- **状态**：✅ 已完成

### T2：ops-agent `session_new` 支持 `trajectory_dir` 参数

- **描述**：`session_new` 接受可选 `trajectory_dir` 参数；用 `traj_source = trajectory_dir or self._trajectory_file` 加载历史；将 `trajectory_dir` 写入 `ACPSession`
- **文件变更**：`ops-agent/ops_agent/acp/server.py`
- **验收标准**：传入有效目录路径时，session 的 `llm_message_history` 从磁盘文件恢复；不传时行为与修复前一致
- **依赖**：T1
- **状态**：✅ 已完成

### T3：ops-agent `_create_agent` 优先使用 per-session trajectory_dir

- **描述**：`_create_agent` 中用 `(session.trajectory_dir if session else None) or self._trajectory_file` 选择 trajectory 路径，确保 `trajectory_recorder` 写入正确目录
- **文件变更**：`ops-agent/ops_agent/acp/server.py`
- **验收标准**：每轮 `session_prompt` 完成后，trajectory 文件出现在 `session.trajectory_dir` 下
- **依赖**：T1、T2
- **状态**：✅ 已完成

### T4：ops-agent REST 路由透传 `trajectory_dir`

- **描述**：`POST /acp/sessions` 路由从请求体读取 `trajectory_dir` 并透传给 `server.session_new`；更新接口文档注释
- **文件变更**：`ops-agent/ops_agent/server/acp_routes.py`
- **验收标准**：HTTP 请求体中带 `trajectory_dir` 字段时，`session_new` 能收到并使用
- **依赖**：T2
- **状态**：✅ 已完成

### T5：hci-platform `OpsAgentBrainAdapter` 传入 per-session trajectory_dir

- **描述**：`__init__` 从环境变量 `OPS_AGENT_TRAJECTORY_DIR` 读取根目录（默认 `/data/ops-agent-trajectories`）；`_ensure_acp_session` 构造 `trajectory_dir = base_dir / session_id` 并加入 payload
- **文件变更**：`hci-platform/backend/conversation-service/app/adapters/ops_agent_brain_adapter.py`
- **验收标准**：`POST /acp/sessions` 请求体包含 `trajectory_dir` 字段；日志中可见 `trajectory_dir=...`
- **依赖**：T4
- **状态**：✅ 已完成
