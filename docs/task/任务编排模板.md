# 任务编排模板

> 本模板用于规范任务文档的编写，确保任务目标明确、步骤可执行、验收标准清晰。

---

## 使用说明

1. **文件命名**：`YYYY-MM-DD-{主题}.md` 放入对应模块的 `events/` 目录
2. **模板字段**：保留所有字段，不可省略
3. **Agent Prompt**：如需分发给其他 agent，按 §4 格式编写

---

## 一、任务元信息

| 字段 | 内容 |
|------|------|
| **任务 ID** | `T-XX`（按模块编号，如 T-CONV-01） |
| **创建日期** | YYYY-MM-DD |
| **优先级** | P0（阻塞）/ P1（重要）/ P2（常规） |
| **所属模块** | 架构 / 数据库 / 对话 / 工单 / AI助手 / 知识库 / 客户端 / 部署 |
| **关联方案** | [方案文档链接] |

---

## 二、背景与问题

> 说明触发此任务的原因，尽量量化。

```
【背景】
<描述当前状态，为什么需要这个任务>

【问题】
<具体问题描述，如有 issue/工单号请引用>
```

## 三、任务目标

> 任务完成后应该达到的状态，要求可验证。

1. <目标 1>
2. <目标 2>

```bash

```

**验收点**：
- [ ] <验收条件>

...

## 六、验收标准

> 必须可量化，不可量化则不可验收。

| # | 验收项 | 通过标准 |
|---|--------|---------|
| 1 | 单元测试 | `pytest tests/xxx/` 100% pass |
| 2 | 手动验证 | <具体操作步骤> |
