# SSH UX 全场景重构任务

## 关联方案

[SSH-UX全场景重构方案.md](../../../solution/custom-ui/events/2026-04-23-SSH-UX全场景重构方案.md)

## 背景

PR #203 完成后用户验证发现 3 个问题：
1. **问题1**：环境采集后页面仍显示"未采集环境数据"（状态不刷新）
2. **问题2**：CaseCreateDialog 精简后移除了 SSH 表单，点击"连接SSH并创建工单"跳出第二弹框
3. **问题3**：TerminalPanel 三态按钮逻辑缺失

本次任务基于方案文档完整实现上述修复，同时整合用户讨论中确认的新 UX 设计（Bridge 前置检测、弹框内视图状态机）。

### T-FE-2：CaseCreateDialog 重构（视图状态机）

**描述**：将 CaseCreateDialog 重构为含内部视图状态机的组件，包含 6 个视图，所有 SSH 流程在弹框内完成，不跳出新弹框。

**视图状态定义**：
```typescript
type DialogView =
  | 'bridge-guide'   // 视图A：Bridge 未运行引导
  | 'form'           // 视图B：SSH 表单（正常入口）
  | 'progress'       // 视图B'：进度中
  | 'error-auth'     // 视图C：SSH 认证失败
  | 'error-collect'  // 视图D：采集失败
  | 'no-ssh'         // 视图E：无SSH创建进行中
  | 'success'        // 视图F：全部成功
```

**弹框打开逻辑**：
```typescript
// 调用方（store 或 TerminalPanel）在弹框打开前传入 bridgeStatus
// 由调用方完成前置检测，将结果传给 CaseCreateDialog
props: {
  bridgeStatus: 'running' | 'not-running' | 'checking'
}
// 弹框根据 bridgeStatus 设置初始 viewState：
// 'not-running' → 'bridge-guide'
// 'running'     → 'form'
// 'checking'    → 显示 loading（等待检测完成）
```

**各视图行为**：

| 视图 | 触发条件 | 操作按钮 | 下一视图 |
|------|---------|---------|---------|
| A bridge-guide | bridgeStatus='not-running' | [已运行刷新] → 重检测；[无SSH创建] | 检测成功→B；检测失败→A+Toast |
| B form | bridgeStatus='running' 或 检测成功 | [连接SSH并创建工单]；[无SSH创建]；[取消] | 点击连接→B' |
| B' progress | 点击"连接SSH并创建工单" | [取消] | 成功→F；认证失败→C；采集失败→D |
| C error-auth | SSH 认证失败 | [重试]→回B；[无SSH创建]→E | — |
| D error-collect | 采集失败（SSH已连通）| [重试采集]→保留SSH重走②；[无SSH创建]→E | — |
| E no-ssh | 任意入口点击"无SSH创建" | 无（自动进行）| 完成→关闭弹框 |
| F success | 全部成功 | 无（2s自动关闭）| 关闭→进入对话 |

**业务流程（视图B' 内）**：
```typescript
async function runSshAndCreateCase() {
  viewState.value = 'progress'
  currentStep.value = 0

  // 步骤①：SSH 认证
  try {
    await connectSSH(form.host, form.port, form.username, form.password)
    currentStep.value = 1
  } catch (err) {
    viewState.value = 'error-auth'
    errorMessage.value = err.message
    return
  }

  // 步骤②：采集环境（与认证强绑定，认证成功必须采集）
  let collectedData: CollectedEnvData | null = null
  try {
    collectedData = await collectEnvironment()
    currentStep.value = 2
  } catch (err) {
    viewState.value = 'error-collect'
    errorMessage.value = err.message
    return
  }

  // 步骤③：创建工单
  const caseId = await createCase(form.title, form.description, form.assistant)
  currentStep.value = 3

  // upsert 环境数据
  for (const [envType, envData] of Object.entries(collectedData)) {
    await environmentApi.upsert(caseId, envType, envData)
  }

  viewState.value = 'success'
  setTimeout(() => {
    closeDialog()
    navigateToChat(caseId)
    fetchEnvironments(caseId) // 修复问题1：刷新 environmentSummary
  }, 2000)
}
```

### T-FE-3：TerminalPanel 三态按钮

**描述**：TerminalPanel 根据「是否有当前工单」和「SSH 是否已连接」显示三种不同状态。

**状态判断逻辑**：
```typescript
// TerminalPanel.vue
const terminalState = computed(() => {
  if (sshConnectionState.value === 'connected') return 'C'  // 已连接
  if (currentCaseId.value) return 'B'                       // 有工单，未连接
  return 'A'                                                  // 无工单，无连接
})
```

**各状态 UI**：

| 状态 | 显示内容 | 点击行为 |
|------|---------|---------|
| A（无工单+无SSH）| 按钮"🖥 连接SSH并创建工单" | `checkAndOpenTerminal()` → 前置检测 → 打开 CaseCreateDialog |
| B（有工单+无SSH）| 按钮"🖥 连接SSH" | `checkAndOpenTerminal()` → 前置检测 → 打开 SshConnectDialog |
| C（已连接）| xterm 终端（无按钮）| 直接操作终端 |

**`checkAndOpenTerminal()` 三态路由**：
```typescript
async function checkAndOpenTerminal() {
  if (sshConnectionState.value === 'connected') return // 状态C：已有终端，无需操作

  // 前置 Bridge 检测
  openCaseCreateDialog_loading.value = true
  const bridgeStatus = await checkBridgeBeforeOpen()
  openCaseCreateDialog_loading.value = false

  if (!currentCaseId.value) {
    // 状态A：打开 CaseCreateDialog
    showCaseCreateDialog.value = true
    caseCreateDialogBridgeStatus.value = bridgeStatus
  } else {
    // 状态B：打开 SshConnectDialog
    showSshConnectDialog.value = true
    sshConnectDialogBridgeStatus.value = bridgeStatus
  }
}
```

**文件变更**：
- `frontend/customer/src/components/TerminalPanel.vue`：三态判断 + 按钮渲染
- `frontend/customer/src/stores/chat.ts`：`checkAndOpenTerminal()` 实现三态路由

**验收标准**：
- [ ] 状态A：显示"连接SSH并创建工单"按钮，点击后触发前置检测（按钮 loading 状态）
- [ ] 状态A：前置检测完成后弹出 CaseCreateDialog，初始视图正确
- [ ] 状态B：显示"连接SSH"按钮，点击后触发前置检测
- [ ] 状态C：显示 xterm 终端，无任何按钮
- [ ] 状态切换实时响应（不需要刷新页面）

**依赖**：T-FE-1、T-FE-2

## 关键约束（实现必读）

1. **密码禁止存入 localStorage**：主机/端口/用户名可缓存，密码每次需重新输入
2. **Bridge 检测超时 3s**：超过 3s 视为未运行
3. **SSH 认证成功 = 必须采集**：不允许跳过采集（仅"无SSH直接创建"可绕过）
4. **重试采集（视图D→重试）**：保留已有 SSH 连接，currentSshSession 不释放，直接重走采集
5. **仅支持 Windows Bridge 下载**：不提供 macOS/Linux 下载入口
6. **弹框关闭后必须刷新**：无论成功还是无SSH创建，关闭后都要 `fetchEnvironments(caseId)`
