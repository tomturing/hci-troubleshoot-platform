# Task 30-A：数据库迁移脚本

```
你是一名负责 hci-troubleshoot-platform 项目数据库迁移的 agent。

【仓库】
git clone https://github.com/tomturing/hci-troubleshoot-platform.git
cd hci-troubleshoot-platform

【任务目标】
新建文件 database/migrate_evaluation_v1.sql，实现以下 DDL 变更：

1. 在 case 表增加列：close_reason VARCHAR(20) CHECK (close_reason IN ('user_command','timeout','abandon','admin_close'))
2. 在 conversation 表增加列：repeat_question_count INTEGER NOT NULL DEFAULT 0
3. 在 assistant_evaluation 表增加列：composite_score SMALLINT、score_breakdown JSONB、close_reason VARCHAR(20)、user_rating SMALLINT（若不存在）、calculated_at TIMESTAMPTZ
4. 新建 prompt_audit 表：
   - audit_id UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - conversation_id UUID NOT NULL（外键 → conversation.id）
   - case_id UUID
   - has_sop BOOLEAN DEFAULT FALSE
   - kb_chunks_count INTEGER DEFAULT 0
   - captured_at TIMESTAMPTZ DEFAULT NOW()
   - payload JSONB  -- 采样时存完整 messages[]
   - trace_id VARCHAR(64)

【设计依据】
详见 docs/16_评分机制与评价系统.md §5（DB Schema 设计），以及 docs/09_项目进展.md §四 Task 30-A 验收标准。

【约束】
- SQL 文件必须幂等（使用 IF NOT EXISTS / ADD COLUMN IF NOT EXISTS）
- 不破坏现有外键和索引
- 每条 ALTER/CREATE 语句后写明注释说明用途
- 文件头部注释：迁移版本 v1、日期、目的

【验收标准】
- psql -c "\d case" 输出包含 close_reason varchar(20) 列
- psql -c "\d conversation" 输出包含 repeat_question_count int 列
- psql -c "\d assistant_evaluation" 输出包含 composite_score smallint、score_breakdown jsonb
- psql -c "\d prompt_audit" 表存在，含 audit_id uuid、has_sop bool、kb_chunks_count int
- 脚本可重复执行，第二次运行无错误

完成后提交 PR，描述中列出所有新增字段与表，等待 Claude 审核。
```
