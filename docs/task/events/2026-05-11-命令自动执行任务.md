## Phase 1 — MVP 核心功能

### T1: 命令风险推断工具（commandRisk.ts）
- **描述**：新建 `frontend/customer/src/utils/commandRisk.ts`，实现 `inferRiskLevel(command)` 函数，基于 acli 命令前缀静态推断 `RiskLevel`（readonly/caution/danger）
- **文件变更**：新增 `frontend/customer/src/utils/commandRisk.ts`
- **验收标准**：`acli platform info get` → `readonly`；`acli vm migrate` → `danger`；未知命令 → `caution`
- **依赖**：无
- **状态**：待开始

### T2: chat.ts 新增自动执行状态与执行方法
- **描述**：在 `chat.ts` 中新增：`autoExecuteMode`（持久化localStorage）、`isExecutingCommand`、`executingCommand`、`autoExecCount`、`consecutiveFailures`，以及核心方法 `executeCommandViaSSH(command, blockId)`（走现有 sshWebSocket，marker 协议，结果注入对话并触发 streamAIResponse）
- **文件变更**：`frontend/customer/src/stores/chat.ts`
- **验收标准**：`executeCommandViaSSH` 可在 SSH 已连接时成功执行命令并将结果追加为 role:user 消息；串行锁保证不并发
- **依赖**：T1
- **状态**：待开始

### T3: CommandBlock.vue 执行状态与取消窗口
- **描述**：CommandBlock 新增自动执行逻辑：1）调用 `inferRiskLevel` 推断风险；2）SSH 已连接 + 开关开启时，展示 3 秒倒计时取消按钮；3）倒计时结束调用 `chatStore.executeCommandViaSSH`；4）执行中/成功/失败三态 UI
- **文件变更**：`frontend/customer/src/components/CommandBlock.vue`
- **验收标准**：执行前 3 秒内可点 × 取消；执行中显示 spinner；exit=0 显示绿色 ✓；exit≠0 显示红色 ✗ + 退出码
- **依赖**：T2
- **状态**：待开始

### T4: MessageBubble.vue 传入 riskLevel
- **描述**：`MessageBubble` 解析命令块时调用 `inferRiskLevel`，将结果作为 `riskLevel` prop 传给 `CommandBlock`；仅对第一个命令块设置 `isFirstBlock=true`（用于自动执行判断）
- **文件变更**：`frontend/customer/src/components/MessageBubble.vue`
- **验收标准**：不同 acli 命令的命令块显示正确的风险标签（只读/谨慎/高危）
- **依赖**：T1, T3
- **状态**：待开始

### T5: TerminalPanel.vue 自动执行开关 UI
- **描述**：在 TerminalPanel 头部状态栏新增三级下拉开关（关闭/安全模式/激进模式）；仅 SSH 已连接时显示；绑定 `chatStore.autoExecuteMode`；安全模式绿色，激进模式橙色 + ⚠️ 提示
- **文件变更**：`frontend/customer/src/components/TerminalPanel.vue`
- **验收标准**：SSH 未连接时开关不可见；切换开关后 CommandBlock 行为符合决策矩阵
- **依赖**：T2
- **状态**：待开始

## Phase 2 — 安全加固

### T6: Agent Loop 防护（计数上限 + 失败熔断）
- **描述**：`chat.ts` 中 `executeCommandViaSSH` 增加：单会话执行计数（上限10条，超限自动关闭并系统提示）+ 连续失败计数（3次熔断自动关闭并系统提示）
- **文件变更**：`frontend/customer/src/stores/chat.ts`
- **验收标准**：执行第11条命令时自动关闭并显示系统消息；连续3次exit≠0后熔断
- **依赖**：T2
- **状态**：待开始

### T7: Page Visibility 暂停队列
- **描述**：在 `chat.ts` 初始化时监听 `visibilitychange` 事件，`document.hidden` 时暂停命令队列；页面恢复可见时继续
- **文件变更**：`frontend/customer/src/stores/chat.ts`
- **验收标准**：切换到其他浏览器标签页时，等待执行的命令不被派发；切回后正常继续
- **依赖**：T2
- **状态**：待开始
