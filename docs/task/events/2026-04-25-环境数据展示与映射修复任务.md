## 任务清单

### T1：重写前端 `parseClusterOutput`（B1 修复）

- **描述**：
  `acli platform info get` 输出为 `[section] + key=value` 的 ini 格式。
  当前实现按 `:` 分割，完全错误；同时包含 DONE marker 行和命令提示符行的污染。
  重写为 ini 风格解析器：
  1. 先剥离 marker 行（`__HCI_DONE_` 开头）
  2. 剥离提示符行（含 `Sangfor:` 或行尾 `# ` 的 shell prompt）
  3. 按 `=` 分割 kv 行，按 `[section]` 分节，展平为一级 dict
  4. 额外提取版本号行（纯版本字符串，如 `6.10.0_R2`）和内核行（`Linux ...`）
  
- **文件变更**：
  - `frontend/customer/src/stores/chat.ts` — 重写 `parseClusterOutput` 函数

- **验收标准**：
  给定真实输出，返回值包含 `{ hci_version: "6.10.0_R2", mcastaddr: "239.208.228.124", zta_enable: "0", ... }`，
  不包含命令行回显 key（如 `acli platform info get; status`）。

- **依赖**：无
- **状态**：待开始

---

### T2：修复后端 `_extract_alert_logs` 字段映射（B2 修复）

- **描述**：
  修正字段名对应关系，并增加 `urgent_type` → `level` 的中英文映射：
  - `urgent_type`（"紧急"/"普通"）→ `level`（"CRITICAL"/"WARNING"，其他 → "INFO"）
  - `description`（截断 200 字符）→ `content`
  - `start` → `time`
  - `target` → `source`
  
- **文件变更**：
  - `backend/case-service/app/services/environment_service.py` — `_extract_alert_logs` 方法

- **验收标准**：
  调用 `_extract_alert_logs({"alerts": [{"urgent_type": "紧急", "description": "...", "start": "2026-04-25", "target": "本地控制器"}]})` 返回
  `[{"level": "CRITICAL", "content": "...", "time": "2026-04-25", "source": "本地控制器"}]`。

- **依赖**：无
- **状态**：待开始

### T3：修复后端 `_extract_task_logs` 字段映射（B3 修复）

- **描述**：
  用 `process` 字符串替代数字 `status` 字段，修正其他字段名：
  - `process`（"失败"/"完成"/"执行中"）→ `status`
  - `type`（"系统备份"等）→ `name`
  - `start` → `time`
  - `description`（截断 300 字符）→ `error`
  
- **文件变更**：
  - `backend/case-service/app/services/environment_service.py` — `_extract_task_logs` 方法

- **验收标准**：
  调用 `_extract_task_logs({"tasks": [{"process": "失败", "type": "系统备份", "start": "2026-04-25 00:59:46", "description": "执行系统备份失败..."}]})` 返回
  `[{"status": "失败", "name": "系统备份", "time": "2026-04-25 00:59:46", "error": "执行系统备份失败..."}]`。

- **依赖**：无
- **状态**：待开始

### T4：同步更新后端 `_extract_env_info` 字段映射

- **描述**：
  `_extract_env_info` 当前期望 `hci_version`/`cluster_name`/`host_count` 等字段，
  而 T1 修复后 cluster env_data 的实际字段为 `hci_version`/`name`/`mcastaddr` 等。
  同步修正映射：
  - `cluster_name` ← `name`（[cluster] section 的 name=）
  - `network_config` ← `mcastaddr`（组播地址，体现网络配置）
  - `hci_version` ← `hci_version`（T1 已提取，直接用）
  - `host_count`/`storage_type` 暂保留"未知"（当前采集命令无此数据，后续扩展）
  
- **文件变更**：
  - `backend/case-service/app/services/environment_service.py` — `_extract_env_info` 方法

- **验收标准**：
  调用时 `cluster_name` 能取到 [cluster] section 中的 name 值；`network_config` 能取到 mcastaddr。

- **依赖**：T1（取决于 T1 产出的 env_data key 结构）
- **状态**：待开始

### T5：前端 SSH 终端：断开重连黑屏 + 半屏黑修复

- **描述**：（已在之前对话中分析，在本次 PR 中一并纳入）
  
  **Bug 5a（重连黑屏）**：`watch(isConnected)` 断开时不销毁 xterm，
  重连后 `initTerminal()` 被 `if (xterm) return` 守卫拦截，终端不显示。
  修复：断开时调用 `disposeTerminal()`。

  **Bug 5b（半屏黑）**：`.terminal-stage` 未建立 flex 链，`.terminal-canvas {height:100%}` 失效，
  回退 `min-height: 230px`（约实际高度一半）；同时 `onMounted` 时 Drawer 动画未完成导致 `fitAddon.fit()` 拿到错误尺寸。
  修复：CSS 改为 flex 链 + double rAF。

  （注：5a/5b 已在前次会话中通过 `multi_replace_string_in_file` 修改了 main 分支的 TerminalPanel.vue，
   需要在 worktree 分支中重新应用相同修改，同时将其纳入本次 PR 统一提交。）

- **文件变更**：
  - `frontend/customer/src/components/TerminalPanel.vue` — watch + CSS

- **验收标准**：
  - 断开 SSH 后重新连接，终端正常显示
  - CaseCreateDialog 流程连接后终端全高显示，不显示半屏黑

- **依赖**：无
- **状态**：待开始（需在 worktree 中应用）
