# SSH 连接架构重构任务

## 关联方案
[SSH 连接架构重构方案](../solution/events/2026-04-21-SSH连接架构重构方案.md)

## 任务清单

### T1: chatStore 全局 SSH 状态管理
- **描述**：在 chatStore 中添加全局 SSH 状态和相关方法
- **文件变更**：`frontend/customer/src/stores/chat.ts`
- **验收标准**：
  - 有 sshWebSocket、sshConnectionState、sshConfig ref
  - 有 connectSSH、disconnectSSH、sendCommand 方法
  - 有 onSshConnected 回调触发智能采集
- **依赖**：无
- **预计耗时**：2小时
- **状态**：待开始

### T2: TerminalPanel 改用全局连接
- **描述**：移除 TerminalPanel 的独立 WebSocket，改用 chatStore
- **文件变更**：`frontend/customer/src/components/TerminalPanel.vue`
- **验收标准**：
  - 无 bridgeSocket ref
  - 使用 chatStore.connectSSH/disconnectSSH
  - 终端输出通过 event 接收
- **依赖**：T1
- **预计耗时**：2小时
- **状态**：待开始

### T3: CaseCreateDialog 改用全局连接
- **描述**：移除 CaseCreateDialog 的独立 WebSocket，改用 chatStore
- **文件变更**：`frontend/customer/src/components/CaseCreateDialog.vue`
- **验收标准**：
  - 无 sshCreationSocket ref
  - 使用 chatStore.connectSSH
  - Bridge 弹框有「重新检测」按钮
  - 连接状态显示分层（蓝色主 + 灰色辅助）
- **依赖**：T1
- **预计耗时**：2小时
- **状态**：待开始

### T4: 智能采集逻辑
- **描述**：SSH 连接成功后自动检查并采集环境数据
- **文件变更**：`frontend/customer/src/stores/chat.ts`
- **验收标准**：
  - 连接成功后检查环境数据状态
  - 无数据或过期(>5min)时自动采集
  - 采集不影响用户终端操作
- **依赖**：T1, T3
- **预计耗时**：1小时
- **状态**：待开始

### T5: 连接日志增强
- **描述**：添加连接各阶段的 console.log，便于排查卡住问题
- **文件变更**：`frontend/customer/src/stores/chat.ts`
- **验收标准**：
  - 连接开始、WebSocket 各阶段、消息收发都有日志
  - 日志格式：[SSH] + 时间戳 + 关键信息
- **依赖**：T1
- **预计耗时**：30分钟
- **状态**：待开始

### T6: EnvironmentSummary 刷新按钮优化
- **描述**：刷新按钮只在 SSH 连接时启用，显示采集状态
- **文件变更**：`frontend/customer/src/components/EnvironmentSummary.vue`
- **验收标准**：
  - SSH 未连接时刷新按钮禁用
  - 显示「需要 SSH 连接才能刷新」提示
- **依赖**：T1
- **预计耗时**：30分钟
- **状态**：待开始

## 测试计划

- **单元测试**：无（前端重构）
- **集成测试**：无
- **人工测试**：
  1. 启动 terminal_bridge.exe
  2. 打开页面，创建工单，填写 SSH 信息
  3. 点击「连接并创建工单」，观察：
     - 状态显示分层（蓝色 + 灰色）
     - 连接成功后自动采集
     - 工单创建成功
  4. 打开终端面板，观察使用同一连接
  5. 关闭 terminal_bridge.exe，弹框显示刷新按钮
