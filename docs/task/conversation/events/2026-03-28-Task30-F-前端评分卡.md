# Task 30-F：前端评分卡（customer 应用）

```
你是一名负责 hci-troubleshoot-platform 前端 customer 应用开发的 agent。

【仓库】
git clone https://github.com/tomturing/hci-troubleshoot-platform.git
cd hci-troubleshoot-platform/frontend

【环境要求】
pnpm install  # 使用 pnpm，不使用 npm/yarn

【前置条件】
Task 30-E（评分 API）已完成，POST /conversations/{id}/evaluate 接口可用。

【任务目标】

1. 新建 frontend/customer/src/components/RatingCard.vue
   - 工单关闭后弹出，非阻塞（不遮挡界面）
   - 展示 1-5 颗星的交互式评分
   - 3 秒无操作后自动消失（设计类似 toast notification）
   - 点击星星 → 调用 POST /conversations/{id}/evaluate → 卡片关闭
   - 展示"跳过"按钮 → 关闭卡片，不发请求

2. 新建 frontend/customer/src/api/evaluate.ts
   - 封装 POST /conversations/{id}/evaluate 的 API 调用
   - 入参：conversationId: string, score: number
   - 处理网络错误（静默失败，不影响用户操作）

3. 修改 frontend/customer/src/stores/chat.ts 或对应 store
   - 在工单关闭事件触发后（case status 变为 closed），设置 showRatingCard = true

4. 修改 frontend/customer/src/components/ChatWindow.vue（或等效组件）
   - 引入并条件渲染 <RatingCard />

参考设计见 docs/16_评分机制与评价系统.md §7。

【约束】
- 使用 Vue 3 Composition API + TypeScript
- 使用 pnpm（不使用 npm/yarn）
- 不引入新的 UI 组件库（使用项目已有组件/CSS）
- 评分卡不阻塞对话窗口内容
- 代码注释使用中文

【验收标准】
- 工单关闭后评分卡显示，3 秒无操作后自动消失（不阻塞后续操作）
- 提交评分 → 卡片关闭，网络请求发出，控制台无报错
- 点击"跳过" → 卡片关闭，无网络请求
- TypeScript 无编译错误（pnpm run type-check 通过）

完成后提交 PR，等待 Claude 审核。
```
