## 2026-04-22 分类管理页面 Bug 修复记录

### 修复说明

`fix/category-manage-view-bugs` 分支，涉及文件：`frontend/admin/src/views/CategoryManageView.vue`、`backend/kb-service/app/repositories/category_repo.py`

| # | 问题 | 根因 | 修复方式 |
|---|------|------|---------|
| 1 | 分类详情中 SOP/KBD 列表始终为空 | 前端直接调用 `/api/admin/sop`、`/api/admin/kbd/pending` 等内部路径，api-gateway 未注册这些路径（404） | 改为走 api-gateway 统一入口：`/api/v1/sop`、`/api/v1/kbd/pending`（4 处路径全部修正） |
| 2 | 分类编码排序错误（如 虚拟机-010 排在 虚拟机-009 前） | SQL `ORDER BY c.code` 按字符串排序，数字部分补零位数不同时顺序错误 | `SPLIT_PART` + `LPAD` 实现自然数字排序，`get_all` 和 `get_all_active` 两处同步修改 |
| 3 | 分类详情标题行（分类详情 / 启用禁用 / 保存修改）三者高度不一致 | `.detail-header` 使用 `justify-content: space-between` 将三项均匀分布，radio 组被推到容器中央 | 改为 `gap: 16px` + 保留 `.detail-actions { margin-left: auto }` 推按钮靠右，标题与 radio 自然左对齐 |
