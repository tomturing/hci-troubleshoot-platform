# Task 07：数据库迁移 003——诊断状态字段与工具审计表（P1）

```
你是一名负责 hci-troubleshoot-platform 数据库迁移的 agent。

【仓库】
git clone https://github.com/tomturing/hci-troubleshoot-platform.git
cd hci-troubleshoot-platform

【背景】
conversation_sessions 表目前只有基础的会话字段，缺乏追踪诊断进度所需的状态字段。
Phase 2 的状态机需要在 DB 层持久化诊断阶段，以支持：
  - 会话在不同轮次中保持诊断进度（S0→S1→S2→...）
  - 断点续接（用户中断后重新进入，状态不丢失）
  - 分析统计（哪个阶段最容易卡住）

同时需要新建工具调用审计表 tool_audit_log，这是生产安全要求：
  - 所有工具调用（包括只读的 acli 查询）都必须记录
  - 高风险操作需要记录授权用户
  - 审计日志不可被绕过（ReAct 执行器强制写入）

注意：Task 04 已创建了迁移 002（knowledge_atoms），本任务是迁移 003。

【任务目标】
1. 创建 Alembic 迁移 003_diagnostic_state.py
2. 扩展 conversation_sessions：加诊断状态字段
3. 新建 tool_audit_log 表
4. 更新 SQLAlchemy ORM 模型
5. 更新 Pydantic Schema（API 响应含 diagnostic_stage）

【涉及服务 / 文件范围】
允许修改：
  - backend/shared/migrations/versions/（新增 003 迁移文件）
  - backend/shared/models/（更新 ConversationSession 模型）
  - backend/conversation-service/app/schemas/（更新响应 schema）
禁止：
  - 修改已有迁移文件（001、002）
  - 修改 database/init_schema.sql

【详细实现步骤】

Step 1：创建迁移文件

新建 backend/shared/migrations/versions/0003_diagnostic_state.py：
